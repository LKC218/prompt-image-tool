# 外部图标本地化接入规范

> 适用范围：从 iconfont、开源图标库或其他免费图标网站引入图标，并在本项目中本地化使用。  
> 核心原则：不依赖运行时外链、不混用风格、不忽略授权。

## 二、选型规则

1. 优先使用同一图标库的一套图标，避免线宽、圆角、透视和填充风格混乱。
2. 优先下载 SVG，不使用在线脚本、在线 CSS 或远程字体链接。
3. 优先选线性图标；当卡片使用彩色方块底时，图标应能稳定转为白色。
4. 业务语义必须直观，不用抽象图形替代明确动作：
   - 备份：文件、硬盘、云盘、归档。
   - 导入：向下箭头、进入托盘、文件导入。
   - 导出：向上箭头、离开托盘、文件导出。
   - JSON：文件、代码括号、数据结构。
   - 同步：服务器、连接、循环箭头。
5. iconfont 图标只能作为素材来源，不能直接热链，也不能绕过授权检查。

## 三、本地化目录规范

设置页专用图标放在：

```text
src/assets/icons/settings/
```

通用图标放在：

```text
src/assets/icons/
```

命名规则：

- 使用英文小写。
- 多词使用连字符。
- 语义优先于来源。
- 不在文件名里写颜色、尺寸和临时版本号。

示例：

```text
backup.svg
import-data.svg
export-json.svg
import-json.svg
sync-server.svg
theme-color.png
```

## 四、SVG 清理规则

引入 SVG 后必须检查：

1. 保留 `viewBox`。
2. 删除无关 `width`、`height`，让 CSS 控制尺寸。
3. 删除硬编码彩色填充，优先改为 `currentColor`；如果用于 CSS mask，可保持单色填充。
4. 删除无用 `id`、`class`、`style`、编辑器元数据。
5. 确保图形视觉居中，不要只看路径坐标居中。
6. 不把整段外部 SVG 直接粘到页面模板里，统一通过资源文件导入。

## 五、接入方式

### 5.1 推荐：图片直出

适合 PNG、复杂 SVG、WebView 中 mask 兼容性不稳定的图标。

```js
import backupIcon from '../assets/icons/settings/backup.svg';

function iconImg(icon, alt = '') {
    return `<img src="${icon}" alt="${alt}" aria-hidden="${alt ? 'false' : 'true'}">`;
}
```

```html
<span class="pc-settings-action-icon">${iconImg(backupIcon)}</span>
```

```css
.pc-settings-action-icon {
    width: 42px;
    height: 42px;
    display: inline-flex;
    align-items: center;
    justify-content: center;
}

.pc-settings-action-icon img {
    width: 24px;
    height: 24px;
    object-fit: contain;
}
```

### 5.2 可选：CSS mask

适合简单单色 SVG，但需要确认目标环境支持。

```js
function iconMask(icon) {
    return `style="-webkit-mask-image:url(${icon});mask-image:url(${icon});"`;
}
```

```html
<span class="pc-settings-list-icon" ${iconMask(icon)}></span>
```

```css
.pc-settings-list-icon {
    width: 20px;
    height: 20px;
    background: currentColor;
    -webkit-mask-repeat: no-repeat;
    mask-repeat: no-repeat;
    -webkit-mask-position: center;
    mask-position: center;
    -webkit-mask-size: contain;
    mask-size: contain;
}
```

注意：如果 mask 后只显示实心方块，应立即改为图片直出。

## 六、视觉规范

- 操作卡大图标容器：`42px × 42px`。
- 容器内图标：`24px × 24px`。
- 小图标：`16px` 到 `22px`。
- 图标和文字使用 `align-items: center` 对齐。
- hover 不改变图标尺寸，只改变阴影、边框或轻微位移。
- 图标颜色不能成为唯一状态表达，选中态还需要勾选、边框或文字说明。

## 七、授权记录要求

每次引入外部图标时，在相关计划文档或提交说明中记录：

- 图标来源网站。
- 图标库名称。
- 授权协议。
- 是否允许商用。
- 是否做过二次修改。
- 本地保存路径。

记录示例：

```text
来源：Lucide
授权：ISC License
用途：设置页同步服务图标
本地路径：src/assets/icons/settings/sync-server.svg
修改：清理颜色，统一尺寸
```

## 八、项目默认策略

本项目默认采用：

1. 优先本地图标资源。
2. 优先 SVG 文件导入。
3. 优先图片直出，只有简单单色图标才使用 CSS mask。
4. 不使用运行时外部图标 CDN。
5. 不在业务代码中混入多个风格冲突的图标库。
