# 项目开发经验

## 局域网双端同步的内容比较

日期：2026-05-10

PC 与 Android 进行局域网同步时，冲突判断应比较业务内容摘要，而不是直接比较完整 JSON。

需要排除或弱化的字段：

- 端侧生成型 ID，例如提示词 ID、版本 ID、图片 ID。
- 同步过程写入的时间字段，例如 `createdAt`、`updatedAt`、`created_at`、`updated_at`。
- 图片传输载荷字段，例如 Data URL、体积、MIME 类型。
- 仅由平台差异产生的字段命名差异，例如 `camelCase` 与 `snake_case`。

推荐做法：

- 在后端先把 PC 与 Android 数据归一化为最小业务比较结构。
- 比较提示词名称、分类、标签、收藏状态、版本提示词、反向提示词、参数和图片文件语义字段。
- 同内容但生成字段不同的记录应计入 `skipped`，不生成冲突副本。
- 只有业务内容确实不同且同 ID 冲突时，才进入 `keep_pc` 冲突副本流程。

本项目对应实现位置：

- `python/main.py`：`normalize_prompt_for_compare()` 与 `is_same_prompt_set()`。
- `python/tests/test_main.py`：`test_sync_import_keep_pc_skips_same_content_with_generated_ids`。

## Windows 安装包 HTTP 服务端口独占

日期：2026-05-10

PC 独立安装包与开发后端都可能使用 `8888`。Windows 下如果直接开启地址复用，可能出现新安装包进程看似启动成功，但请求实际落到旧的 `python/main.py` 服务，WebView 显示 `Directory listing for /__pycache__/`。

推荐做法：

- 安装包入口不要依赖 `allow_reuse_address = True` 抢占固定端口。
- Windows 下优先启用 `SO_EXCLUSIVEADDRUSE`，端口已被占用时应明确失败并切换到移动端扫描范围内的端口，例如 `8889-8897`。
- WebView、健康检查和局域网能力接口都必须使用实际绑定端口。
- 回归测试需要覆盖 `/`、`/index.html` 和任意前端路由，确保返回内置前端首页而不是目录列表。

本项目对应实现位置：

- `build/app_main.py`：`ExclusiveThreadingTCPServer` 与 `create_http_server()`。
- `python/tests/test_build_app_main.py`：端口占用回退与前端路由目录列表回归测试。

## 局域网同步端口必须作为目标参数传递

日期：2026-05-10

PC 安装包端口占用时会从 `8888` 回退到 `8889-8897`。移动端若仍在搜索、测试连接、拉取、回传、双向同步或图片下载任一环节写死 `8888`，就会出现“PC 设置页可见服务，但手机同步失败”的断链问题。

推荐做法：

- 同步目标统一建模为 `IP:port`，不要在各调用点临时拼接端口。
- 手动输入支持 `IP`、`IP:port` 和 `http://IP:port`；不带端口时才默认 `8888`。
- 局域网搜索优先探测最近设备，再扫描小范围候选端口，例如 `8888-8897`。
- 最近设备按 `ip:port` 去重并保存令牌、能力和最近同步时间。
- `/api/health`、`/api/sync/capabilities`、`/api/network-info` 都必须返回当前实际监听端口。

本项目对应实现位置：

- `src/js/lan-sync.js`：`parseLanTarget()`、`LanScanner.scan()` 和 `LanSync` 动态端口请求。
- `src/js/mobile-settings.js`：手动输入、搜索结果、最近设备和同步调用的 `IP:port` 传递。
- `python/main.py` / `build/app_main.py`：同步能力与网络信息接口返回实际端口。

## Tauri 安装器壳静态页面必须暴露全局 Tauri API

日期：2026-05-12

安装器壳窗口能打开，不代表页面已经具备调用 Rust 安装命令的能力。`installer-shell/` 当前不是 Vite 打包入口，而是由 Tauri 直接加载 `installer-shell/src/index.html` 和 `main.js`。如果 `tauri.conf.json` 未启用 `app.withGlobalTauri`，页面拿不到 `window.__TAURI__.core.invoke`，点击“开始安装”会落入前端兜底提示“当前环境无法执行安装”。

推荐做法：

- `installer-shell/src-tauri/tauri.conf.json` 必须保留 `app.withGlobalTauri: true`。
- 安装器壳 release 验证不能只检查窗口标题和首屏，还要检查安装按钮能进入 Rust `install_with_nsis` 命令。
- 若只改安装器壳配置，可复用现有 NSIS 安装核心执行 `python scripts\build_installer_shell_package.py --skip-pc-build`，并确认新产物复制到 `releases/`。
- 安装核心可用性用同一静默参数分层验证：`PromptImageManager-Setup-2.3.1.exe /S /D=<目标目录>`，确认主程序和卸载器均落地。

本项目对应实现位置：

- `installer-shell/src-tauri/tauri.conf.json`：`app.withGlobalTauri`。
- `installer-shell/src/main.js`：`getTauriInvoke()` 与 `install_with_nsis` 调用。
- `installer-shell/src-tauri/src/main.rs`：`install_with_nsis()` 与嵌入安装核心释放逻辑。

## 安装器壳收尾任务不能阻断主安装成功

日期：2026-05-12

安装器壳在 `install_with_nsis` 成功后还会创建桌面快捷方式和开始菜单快捷方式。快捷方式属于收尾体验，不属于主安装结果。如果主程序和卸载器已经落地，桌面快捷方式或开始菜单处理失败不应该让用户停在“正在安装”页，也不应该显示为主安装失败。

推荐做法：

- 先以 `install_with_nsis` 返回的 `exe_exists` 和 `uninstaller_exists` 判定主安装是否成功。
- 桌面快捷方式与开始菜单快捷方式逐项执行，逐项捕获错误。
- 快捷方式失败时进入完成页并显示短警告，例如“桌面快捷方式处理失败，可安装后手动创建。”
- Rust 侧调用 PowerShell 创建快捷方式时使用 `-Sta`，并捕获 stdout/stderr，避免只返回笼统失败。

本项目对应实现位置：

- `installer-shell/src/main.js`：`applyShortcutOptions()`。
- `installer-shell/src-tauri/src/main.rs`：`apply_desktop_shortcut()`、`apply_start_menu_shortcut()` 和 PowerShell 执行封装。

## Tauri 安装器壳窗口能力需要显式授权

日期：2026-05-12

安装器壳使用无边框自定义标题栏时，前端调用 `getCurrentWindow().close()`、`minimize()` 或通过 `data-tauri-drag-region` 拖动窗口，都依赖 Tauri capability 权限。只看到按钮渲染出来不代表窗口命令可用；缺少权限时完成页或关闭按钮会显示类似 `Command plugin:window|close not allowed by ACL` 的错误。

推荐做法：

- `installer-shell/src-tauri/capabilities/default.json` 至少保留 `core:window:allow-close`、`core:window:allow-minimize`、`core:window:allow-start-dragging`。
- 完成页点击“完成”应最终关闭安装器壳窗口；启动主程序或快捷方式收尾失败不应把用户困在完成页。
- 发布包验证不能只看窗口打开，还要验证无边框窗口的拖动、最小化和关闭按钮。

本项目对应实现位置：

- `installer-shell/src/index.html`：自定义标题栏和 `data-tauri-drag-region`。
- `installer-shell/src/main.js`：`getCurrentWindow().close()`、`minimize()` 和完成按钮收尾。
- `installer-shell/src-tauri/capabilities/default.json`：窗口命令 ACL。
