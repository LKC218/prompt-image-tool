# 界面6：设置与本地存储

> 版本：1.0 | 最后更新：2026-05-05

## 260510 布局排版规范

PC 端设置页采用参考图式桌面布局，页面信息顺序固定为：

1. 顶部欢迎横幅：使用 `renderPcWelcomeBanner()`，标题为 `设置`，右侧吉祥物插画尺寸和位置与首页欢迎横幅一致，核心传达“数据仅在本地”。
2. 双列核心区：左侧为外观设置，右侧为本地存储。
3. 数据备份与恢复：全宽操作区，包含本地备份、导入数据、导出 JSON、从 JSON 导入四个入口。
4. 局域网互通：展示 PC 服务端地址、互通能力、复制地址和刷新入口。
5. 底部安全提示条：浅绿色提示，强调本地运行和隐私安全。

样式约束：

- 设置页专用样式统一使用 `pc-settings-*` 类名前缀。
- 顶部共享欢迎横幅使用 `pc-welcome-banner-settings` 扩展类，并继承首页 `224px × 178px` 插画基准。
- 主题色入口必须使用按钮语义，并通过 `aria-checked` 表达选中状态。
- 数据环形图使用 CSS `conic-gradient`，不新增图表依赖。
- 数据备份与恢复、局域网互通图标使用 `src/assets/icons/settings/` 下的本地 SVG 资源，运行时不依赖外部图标网站。
- 操作卡图标采用“彩色容器 + 白色语义图标”结构，图标必须在容器内水平、垂直居中。
- 宽度低于 1120px 时核心区改为单列，备份操作区改为两列；更窄视口再改为单列。
- 动效仅使用 150ms 到 300ms 的轻量过渡，并尊重 `prefers-reduced-motion`。

## 1. 页面定位与目标

展示应用设置和本地存储信息，强调本地安全：
- 外观设置（深色模式/浅色模式）
- 本地存储数据查看
- 数据备份与恢复
- 局域网同步
- 本地安全提示

**核心目标**：让用户安心使用，明确数据安全，提供数据管理入口。

## 8. 16:9 与响应式补充

- PC 默认窗口采用 `1366 x 768`，设置页卡片组根据主内容区宽度从四列逐步退化为两列和单列。
- 外观、本地存储、备份恢复、局域网同步模块保持独立卡片，不互相嵌套，避免窄窗口下卡片内部再次挤压。
- IP 地址、同步状态、导入导出按钮组允许整组换行，按钮文字保持单行。
- 存储容量、版本信息等短文本优先保持完整显示，长路径或地址使用换行兜底。
