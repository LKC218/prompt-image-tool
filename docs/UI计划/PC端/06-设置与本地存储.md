# 界面6：设置与本地存储

> 版本：1.4 | 最后更新：2026-07-17

## 260717 正文布局规范

- 顶部欢迎横幅、标题、副标题和 settings 变体走路动画保持不变，不纳入设置页正文重构范围。
- 正文采用单列分区节奏，区块间距为 16px；首行保留外观设置与本地存储约 `5 / 7` 的非对称双列，按内容自然高度排列并顶部对齐，避免通过强制等高制造无效留白。
- 外观设置以两条标准设置行承载主题与外观模式；本地存储保留数据规模、存储空间和元信息，但降低内部块的边框与阴影，避免多层卡片竞争层级。
- 数据备份与恢复将导出位置收敛至标题辅助区，三个主操作必须由 `.pc-settings-action-grid-three` 网格承载，桌面端等宽等高三列、中等宽度两列、小屏单列；兼容 JSON 与说明组成底部辅助区。
- 下载记录和局域网互通继续保持独立区块，首屏优先呈现记录数量、清理操作和同步状态；下载记录按业务标题、实际文件名、来源/保存位置与时间、平台、保存方式分层展示，空记录时清空操作禁用；不改变数据加载或交互逻辑。

## 260716 拟态视觉规范

- 设置页沿用全局 `--pc-neu-*` 令牌，一级模块使用暖白外凸面，取消常规描边和单向投影。
- 外观选择器、导出位置分段、下载记录容器、存储信息块与元信息标签使用内凹层级，表达可选、可查看或嵌入式数据区域。
- 导入、导出、复制、刷新与清空操作统一为外凸按钮，悬停仅增强阴影并上移 1px，按下切换为内凹状态；语义色仅保留在图标和文字。
- 存储容量环、下载图标和局域网图标使用内凹图标槽，状态色保持在进度填充、徽标与文字中，不再使用整块彩色渐变。
- 所有动态效果维持在 150ms 到 300ms，并在 `prefers-reduced-motion` 下停用位移与阴影状态切换。

## 260510 布局排版规范

PC 端设置页采用参考图式桌面布局，页面信息顺序固定为：

1. 顶部欢迎横幅：使用 `renderPcWelcomeBanner()`，标题为 `设置`，右侧吉祥物插画尺寸和位置与首页欢迎横幅一致，核心传达“数据仅在本地”；同时接入 `renderPcWelcomeWalkAnimation({ variant: 'settings' })` 作为轻量氛围动效，本轮正文改造不得修改其结构与视觉。
2. 双列核心区：左侧为外观设置，右侧为本地存储，采用约 `40% / 60%` 的非对称比例，不强制两侧等高。
3. 本地存储区：展示提示词数量、数据大小、存储估算、存储类型、版本和 PC 后端真实数据目录；数据目录长路径使用单行省略并保留悬浮标题。
4. 数据备份与恢复：全宽操作区常驻“导入 JSON”“导入 ChatGPT 对话”和“导出完整备份”三个卡片入口；完整备份默认导出 ZIP v2 并包含原图，卡片下提供“导出兼容 JSON”次级入口。导出位置使用分段选择，默认保存到下载目录，可切换为自定义位置；当前 ZIP 导入恢复在事务化恢复完成前不开放。
5. 局域网互通：展示 PC 服务端地址、互通能力、复制地址和刷新入口。
6. 底部安全提示条：浅绿色提示，强调本地运行和隐私安全。

样式约束：

- 设置页专用样式统一使用 `pc-settings-*` 类名前缀。
- 顶部共享欢迎横幅使用 `pc-welcome-banner-settings` 扩展类，并继承首页 `224px × 178px` 插画基准，像素走路动画采用 settings 变体轨道。
- 主题色入口必须使用按钮语义，并通过 `aria-checked` 表达选中状态。
- 数据环形图使用 CSS `conic-gradient`，不新增图表依赖。
- 数据备份与恢复、局域网互通图标使用 `src/assets/icons/settings/` 下的本地 SVG 资源，运行时不依赖外部图标网站。
- 操作卡保持白底与统一边框，语义色仅用于图标槽、图标和边框强调；图标必须在容器内水平、垂直居中。
- 核心双列分别保留 360px 与 520px 的最小可读宽度；主内容区低于 920px 时核心区改为单列。备份操作区从三列退化为两列、再到单列，卡片文案允许换行；导出位置和底部辅助说明可独立换行，不与操作按钮竞争宽度。
- 动效仅使用 150ms 到 300ms 的轻量过渡，并尊重 `prefers-reduced-motion`。

## 1. 页面定位与目标

展示应用设置和本地存储信息，强调本地安全：
- 外观设置（深色模式/浅色模式）
- 本地存储数据查看
- 数据备份与恢复
- 局域网同步
- 本地安全提示

**核心目标**：让用户安心使用，明确数据安全，提供数据管理入口。

- 存储容量、版本信息等短文本优先保持完整显示，长路径或地址使用换行兜底。

## 主题色实装说明（已落地）

设置-外观页取色器切换后，全站品牌主色实时跟随，链路为：
取色器点击 → `setAccent()` → 根节点 `data-accent` + `localStorage` → CSS 覆盖 `--pc-accent`/`--m-accent` →
派生令牌（`color-mix()`）驱动主按钮渐变与投影、激活/选中高亮、聚焦环、水波纹、进度环、欢迎页主装饰。

- 预设：粉 / 蓝 / 绿 / 紫 / 黄（保持 5 个，不做自定义取色、不新增预设）。
- 语义色（成功绿、警告黄、危险红、信息蓝、分类标签、功能快捷卡）保持独立，不随主题色变化。
- PC 与移动端采用同一套 `color-mix()` 派生策略，切换观感一致。
