# 界面2：提示词库

## 2026-05-10 参考图布局落地说明

本页 PC 端布局调整为「顶部横幅 + 大搜索框 + 筛选工具条 + 表格列表 + 右侧预览面板」结构，严格对齐参考图的信息密度和操作顺序。

### 页面结构

- 顶部继续使用 `renderPcWelcomeBanner()`，标题为“提示词库”，横幅内不放主操作按钮，右侧吉祥物插画尺寸和位置与首页欢迎横幅保持一致。
- 搜索框放在横幅下方左侧，支持按提示词名称、标签和分类过滤；蓝色主按钮“新建提示词”移动到搜索栏同一行右侧。
- 筛选工具条包含“全部”“收藏”“最近使用”，移除场景、人物和高级筛选入口，避免顶部工具条拥挤。
- 主体左侧为表格列表，列为：提示词、标签、创建时间、最近使用、收藏、操作。
- 主体右侧为选中提示词预览面板，展示封面图、标题、收藏、标签、描述、创建时间、最近使用、版本记录和底部操作按钮，不提供关闭预览按钮。
- 底部保留分页控件和每页数量选择，移除独立本地安全提示条。

### 交互规则

- 单击表格行只切换右侧预览，不离开当前页面。
- 双击表格行进入提示词详情页。
- 收藏按钮、更多菜单、右键菜单沿用现有提示词操作能力。
- 重命名弹窗输入框必须受弹窗内容区约束，不得超出弹窗边界。
- 右侧“编辑”进入编辑页，“复制”优先复制当前提示词正文，正文为空时复制标题。
- 右侧预览图片按原图比例完整显示，不裁剪；点击预览图打开完整大图查看层。
- 默认每页 20 条，支持 10 / 20 / 50 条切换，页码和总页数按筛选后的真实总数计算。

### 样式落点

- 页面专用样式统一使用 `pc-library-*` 类名，集中维护在 `src/css/pc.css`。
- 顶部欢迎横幅插画继承共享横幅规范，保持 `224px × 178px` 的首页基准尺寸和右侧顶部位置。
- 列表缩略图使用 `cover` 保持行高整齐；右侧封面使用 `contain` 保证完整图片可见；无图片时显示本地 SVG 占位。
- 表格表头列名保持单行可读，窄宽度下通过表格最小宽度和横向滚动兜底，避免中文列名被压成逐字竖排。
- 列表区作为统一工作台处理：表格卡片、选中行、横向滚动条和分页区共享柔和边界；选中态使用浅蓝底色加左侧强调条，避免强描边与滚动条冲突。
- 单行列表缩略图采用接近 `16:9` 的稳定尺寸，表头、单元格和分页区使用一致内边距，单条数据时也保持紧凑、连续的垂直节奏。
- 动效只用于 hover、选中态和按钮反馈，并通过 `prefers-reduced-motion` 降低动效。

## 1. 页面定位与目标

提示词库是提示词列表页，用于：
- 搜索、筛选提示词
- 查看所有提示词卡片
- 快速操作（重命名/复制/删除/移动）

**核心目标**：让用户高效找到目标提示词并快速操作。

## 8. 16:9 与响应式补充

- PC 默认窗口采用 `1366 x 768`，提示词库在该宽度下优先保护表格阅读宽度。
- 表格与预览面板的大屏双栏只在主内容区足够宽时保留；默认 16:9 窗口和更窄窗口中，右侧预览面板提前下沉到表格下方。
- 表格保留稳定最小宽度和横向滚动兜底，表头使用单行省略，不出现中文逐字竖排。
- 分页区在中窄主内容区允许分行排列，避免页码、跳转和每页数量控件反向挤压表格列宽。
