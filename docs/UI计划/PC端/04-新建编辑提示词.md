# 界面4：新建/编辑提示词

> 版本：1.0 | 最后更新：2026-05-10

---

## 1. 页面定位与目标

用于新建或编辑提示词，包括：
- 输入提示词集合名称
- 输入正向/负向提示词
- 选择分类
- 管理版本

**核心目标**：提供清晰高效的编辑体验。

---

## 2. 页面结构

页面顶部使用共享欢迎横幅组件，新建和编辑状态展示不同标题与副标题；布局参考顶部横幅设计稿，左侧依次为返回按钮、标题和副标题，右侧为柯基插画与保存按钮。新建模式在保存按钮左侧提供低权重“一键清空”按钮，用于清除当前未保存输入、图片预览和新建草稿；编辑模式不显示该按钮，避免误清已保存内容。横幅外层与提示词库一致撑满主内容区，表单主体保持居中双栏编辑布局，底部不追加独立的本地保存提示条。

## 5. 数据绑定

| 数据 | 来源 | 说明 |
|------|------|------|
| 标题 | `promptSet.name` | 新建时为空 |
| 分类 | `promptSet.folder_id` | 新建时为 null |
| 正向提示词 | `version.positive_prompt` | 新建时为空 |
| 负向提示词 | `version.negative_prompt` | 新建时为空 |
| 版本列表 | `promptSet.versions` | 新建时只有 v1 |

### 创建逻辑

```
点击「创建」
├── 校验：标题不为空
├── 创建提示词集合 → createPromptSet()
├── 创建版本 → createVersion()
├── Toast: 「已创建」
└── 切换到详情页
```

## 8. 16:9 与响应式补充

- PC 默认窗口采用 `1366 x 768`，编辑页在中等宽度下提前从主表单加侧栏退化为单栏，避免分类、比例和插画区域压缩主输入区。
- 主表单和侧栏卡片均设置抗挤压基础规则，文本域、标签、比例按钮使用整组换行而不是按钮内部逐字换行。
- 图片预览网格使用自适应列宽，上传、删除和查看按钮保持固定交互尺寸，避免图片加载前后布局跳动；图片区支持点击上传和 `Ctrl+V` 粘贴外部图片，粘贴图片追加到当前图片末尾。
- 新建和编辑页通过本地草稿记录未保存内容，返回其他页面再回来时恢复标题、提示词、标签、分类、比例和新导入图片；正式保存成功后清理对应草稿。
- 新建模式一键清空会先二次确认，确认后删除 `pc-editor-draft:new`，避免清空后切页回来再次恢复旧输入；清空范围包含点击上传、剪贴板粘贴和 prompt-image-tool JSON 导入预填产生的图片。
- 更窄窗口下隐藏编辑页插画，将空间优先留给标题、提示词和保存操作。
