# 界面5：分类与标签

> 版本：1.1 | 最后更新：2026-05-10

---

## 1. 页面定位与目标

用于管理提示词分类和标签，包括：

- 查看所有分类及各分类提示词数量
- 新建、重命名、改色、删除分类
- 通过分类跳转提示词库筛选结果
- 拖拽调整分类顺序
- 合并分类、清理空分类
- 新建、查看、重命名、清除标签

**核心目标**：以接近设计参考图的轻量表格卡片布局，让分类和标签管理更清晰、可扫描、可操作。

---

## 2. 页面结构

页面顶部使用共享欢迎横幅组件，标题为“分类与标签”，并通过 `renderPcWelcomeWalkAnimation({ variant: 'category' })` 接入轻量像素柯基走路动画。主体采用“页签 + 管理卡片 + 快捷操作”的结构。

```text
顶部欢迎横幅
└─ 胶囊页签：分类管理 / 标签管理
   ├─ 分类管理卡片
   │  ├─ 表头：分类名称 / 包含提示词 / 新建分类
   │  ├─ 搜索与总数
   │  └─ 分类行：图标 + 名称 / 数量 / 更多操作 + 拖拽手柄
   └─ 快速操作：排序 / 批量编辑 / 合并分类 / 删除

标签页复用同一张管理卡片：
└─ 表头：标签名称 / 使用次数 / 新建标签 + 清除全部
   ├─ 搜索与总数
   └─ 标签行：图标 + 名称 / 数量 / 更多操作
```

## 3. 视觉规范

### 3.1 布局

| 区域 | 规范 |
|---|---|
| 顶部横幅 | 与页面同材质的暖白拟态凸起面板，右侧吉祥物插画尺寸和位置与首页欢迎横幅一致，标题使用深棕色强化第一视觉；像素走路动画使用较短 category 轨道，避免抢占下方页签注意力 |
| 页签 | 内凹胶囊槽；激活页签凸起并使用主题色文字，按下时切换内凹反馈 |
| 管理卡片 | 暖白拟态凸起容器、24px 圆角、表格式低对比横向分隔线；搜索框使用内凹槽表达输入区域 |
| 分类行 | 固定 72px 最小行高，左侧 42px 图标方块保留 PNG 原色和语义底色；行悬浮使用轻微内凹，右侧更多和拖拽使用小型凸起圆形控件 |
| 快捷操作 | 四个暖白拟态功能块，图标容器采用内凹语义色点缀，右侧配拟态气泡提示插画 |
| 快捷操作图标 | 使用本地图片直出；图标本体按按钮语义色着色，删除类操作统一使用设计稿 PNG 并搭配浅红色底，避免纯白图标块 |
| 危险描边按钮图标 | 使用与按钮文字、边框一致的危险红 filter，保证“清除全部”等操作语义统一 |

分类、标签、快捷操作及创建/重命名弹窗均复用 `--pc-neu-*` 基础令牌，并在 `.pc-category-page` 中定义局部凸起、悬浮和内凹层级。拟态不替代语义色：分类颜色、标签颜色和危险操作仍分别通过图标、文字与滤镜表达。

### 3.2 图标尺寸

| 场景 | 尺寸 |

## 5. 数据绑定

| 数据 | 来源 | API/方法 |
|---|---|---|
| 分类列表 | 本地存储或后端存储适配层 | `getFolders()` |
| 提示词集合 | 本地存储或后端存储适配层 | `getPromptSets()` |
| 分类下提示词数量 | 前端聚合 | 按 `folderId` 统计 |
| 标签列表和计数 | 前端聚合 | `aggregateTags(promptSets)` |
| 自定义标签 | 浏览器本地存储 | `saveCustomTag(tagName)` |
| 标签语义样式 | 共享映射 | `getPcTagStyleClass(tagName)` 输出 `pc-tag-*` |

## 6. 交互规范

| 交互 | 行为 |
|---|---|
| 点击分类更多 | 打开查看、重命名、更改颜色、删除菜单 |
| 右键分类行 | 打开同一组分类操作菜单 |
| 拖拽分类手柄 | 调整排序并写回 `sortOrder` |
| 点击新建标签 | 打开标签创建弹窗，保存为自定义标签 |
| 点击标签行 | 打开查看相关提示词、重命名标签、清除该标签菜单；自定义标签支持重命名和清除 |
| 点击快速排序 | 高亮首个拖拽手柄并提示拖拽方式 |
| 点击合并分类 | 打开源分类和目标分类选择弹窗 |
| 点击删除快捷操作 | 清理空分类确认弹窗 |

动效控制在 150ms 到 300ms，遵守 `prefers-reduced-motion` 降级规则。

### 6.1 内容标签胶囊

- PC 标签仅以 `scene`、`japanese`、`scifi`、`illustration`、`chinese` 五类语义令牌映射颜色，不按渲染顺序轮换。
- 未预设的扩展或自定义标签使用 `default` 样式，不额外分配颜色。
- 默认态使用同色系底色、描边、顶部内高光和轻柔投影；悬浮时上浮 `1px`，按下时切换为内凹投影。
- 编辑器选择器与提示词库标签筛选继承语义色；筛选选中态使用高对比实色，编辑器标签保留 `32px` 高度与删除操作，删除控件提供键盘焦点环。

### 6.2 新建分类弹窗

| 区域 | 规范 |
|---|---|
| 弹窗尺寸 | 内容宽度为 `min(496px, calc(100vw - 112px))`，复用全局居中弹窗 |
| 标题区 | 标题 + 一行辅助说明，增强创建语境 |
| 分类名称 | 固定 56px 高度输入框，使用 `border-box` 防止输入框超过弹窗内容区 |
| 分类颜色 | 8 个色点等距网格排列，选中态使用深色外圈和白色隔离圈 |
| 操作按钮 | 右对齐，取消为次按钮，创建为主按钮；名称为空时创建按钮禁用 |

### 6.3 新建标签弹窗

| 区域 | 规范 |
|---|---|
| 弹窗尺寸 | 复用 `.pc-category-create-dialog` 的内容宽度 |
| 标签名称 | 固定 56px 高度输入框，名称为空时禁用创建按钮 |
| 重复校验 | 与现有聚合标签精确匹配，重复时 Toast 提示 |
| 创建结果 | 调用 `saveCustomTag()` 保存，自定义标签以 `0 个提示词` 显示在标签列表 |
| 管理闭环 | 自定义标签可通过重命名更新本地标签列表，也可通过清除操作从本地标签列表移除 |

## 8. 16:9 与响应式补充

- PC 默认窗口采用 `1366 x 768`，分类与标签页优先保持列表行、名称和操作按钮可读。
- 主内容区变窄时，分类表格和标签行隐藏使用次数等次要列，保留名称与操作列，避免三列布局互相挤压。
- 快捷操作区按主内容宽度从四列退化为两列，按钮文字保持单行，整组允许换行。
- 新建分类和新建标签弹窗保留最大宽度约束，颜色选择器在窄窗口下不溢出弹窗内容区。
