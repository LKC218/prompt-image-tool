# 界面1：首页仪表盘

> 版本：1.1 | 最后更新：2026-05-10

---

## 1. 页面定位与目标

首页是 PC 端进入应用后的仪表盘，用于：

- 快速了解提示词、分类、标签和收藏数量。
- 查看最近使用的提示词，并直接进入详情、收藏或打开更多操作。
- 查看常用分类，并跳转到提示词库分类筛选结果。
- 快速新建提示词或从 JSON 文件导入提示词。
- 强化“本地运行、数据安全”的产品信任感。

**核心目标**：严格贴近参考图的桌面首屏排版，让创作者一眼掌握全局，并能快速进入下一步操作。

## 5. 数据绑定

| 数据 | 来源 | 方法 |
|------|------|------|
| 提示词总数 | 存储适配层 | `getPromptSets().length` |
| 分类数 | 存储适配层 | `getFolders().length` |
| 标签数 | 前端聚合 | `aggregateTags(promptSets).length` |
| 收藏数 | 前端聚合 | `promptSets.filter(p => p.isFavorite).length` |
| 最近使用 | 提示词列表 | 按 `updatedAt` 倒序取前 6 条 |
| 最近使用缩略图 | 提示词摘要 | `firstImage` + `storage.getImageUrl()` |
| 收藏分类 | 分类列表 | `getFolders()` + 提示词数量聚合 |

## 6. 交互规范

| 元素 | 行为 |
|------|------|
| 搜索输入 | 输入时同步筛选最近使用和收藏分类 |
| 搜索回车 | 带搜索词跳转 `/library` |
| 搜索右侧按钮 | 有搜索词时进入提示词库搜索；无搜索词时聚焦输入框 |
| 查看全部 | 最近使用跳转 `/library`，收藏分类跳转 `/category` |
| 最近使用项 | 点击进入详情页 |
| 星标按钮 | 调用 `toggleFavorite()`，保留当前列表位置 |
| 更多按钮 | 打开现有提示词更多菜单 |
| 分类卡片 | 跳转提示词库并带 `folder` 参数 |
| 新建提示词 | 跳转 `/editor/` |
| 导入提示词 | 打开 JSON 文件选择并调用现有导入逻辑 |

## 7. 动效与响应式

- 统计卡和最近使用项使用 0.2s 左右的轻量淡入上移。
- 卡片悬停只做轻微上移和阴影增强，不改变布局尺寸。
- 星标、更多、搜索按钮都有 `focus-visible` 状态。
- `prefers-reduced-motion: reduce` 下关闭首页动效和位移动效。
- 约 1180px 以下最近使用退化为单列，底部双栏退化为上下堆叠。
- 约 980px 以下统计卡改为两列，收藏分类改为两列。

## 8. 验收标准

- [ ] 首页首屏结构符合参考图：欢迎横幅、悬浮搜索、四张统计卡、最近使用、底部双栏和安全提示条。
- [ ] 统计卡显示正确数量，并包含图标、标题和数字。
- [ ] 最近使用最多展示 6 条，按更新时间倒序，缩略图优先显示真实图片。
- [ ] 收藏分类最多展示 4 个，并显示分类名称和数量。
- [ ] 搜索、查看全部、详情跳转、收藏切换、更多菜单、分类跳转、新建和导入均可用。
- [ ] 1024px、1280px、1440px 桌面宽度下无文字重叠、无横向滚动、无布局抖动。
- [ ] 动效尊重 `prefers-reduced-motion`。
