# 移动端全局设计规范

> 版本：1.0 | 最后更新：2026-05-05
> 适用范围：提示词管家 移动端 UI（Android / iOS WebView）

---

## 一、设计理念

### 1.1 核心原则

| 原则 | 描述 |
|------|------|
| **温暖治愈** | 奶油白背景 + 马卡龙配色 + 吉祥物陪伴，营造轻松愉悦的使用氛围 |
| **内容聚焦** | 界面元素服务于提示词内容，减少装饰干扰，让提示词成为视觉中心 |
| **渐进展示** | 首页概览 → 列表浏览 → 详情查看 → 编辑修改，层级递进 |
| **即时反馈** | 所有操作均有 Toast 通知、动画过渡或状态变化，用户始终感知系统状态 |
| **本地优先** | 全程强调数据安全，不联网、不收集数据，UI 中多处体现本地安全理念 |

### 1.2 视觉风格关键词

`奶油白背景` `马卡龙配色` `圆角卡片` `轻柔阴影` `可爱治愈` `柯基吉祥物` `iOS风格` `简体中文` `创作者工具`

### 1.3 用户气质

创作者、AI 生图用户、提示词管理用户、视觉设计师、内容创作者

### 1.4 整体情绪

温暖、可爱、治愈、轻量、干净、专业但不冰冷

| 项目 | 规范 |

| 画面比例 | 9:19 或接近 iPhone 竖屏比例 |

## 三、色彩系统

整体以 **奶油白 + 马卡龙粉 + 浅蓝 + 浅绿 + 浅黄 + 淡紫** 为主。

### 3.1 CSS 变量定义

```css
.mobile-app {
    /* 背景色 */
    --m-bg: #FFFDF8;
    --m-surface: #FFFFFF;
    --m-surface2: #FFF8EC;
    --m-surface3: #FFF0E0;

    /* 品牌色 */
    --m-pink: #FF6F9F;
    --m-pink-light: #FFE3EB;
    --m-pink-bg: #FFF0F5;

    /* 功能色 */
    --m-blue: #2D9CFF;
    --m-blue-light: #E8F4FF;
    --m-green: #72D879;
    --m-green-light: #CFF7D7;
    --m-yellow: #FFD15C;
    --m-yellow-light: #FFE8A3;
    --m-purple: #B99CFF;
    --m-purple-light: #EFE5FF;

    /* 文字色 */
    --m-text: #342721;
    --m-text2: #695B55;
    --m-text3: #958983;

    /* 边框色 */
    --m-border: #EFE8DE;
    --m-border-light: #F5F0E8;

    /* 状态色 */
    --m-danger: #FF5A5A;
    --m-success: #72D879;
    --m-warning: #FFD15C;

    /* 阴影 */
    --m-shadow-sm: 0 2px 8px rgba(120, 90, 50, 0.06);
    --m-shadow-md: 0 4px 16px rgba(120, 90, 50, 0.08);
    --m-shadow-lg: 0 8px 24px rgba(120, 90, 50, 0.10);
    --m-shadow-pink: 0 4px 16px rgba(255, 111, 159, 0.25);

    /* 圆角 */

    --m-radius-md: 16px;
    --m-radius-lg: 24px;
    --m-radius-xl: 28px;
    --m-radius-full: 9999px;

    /* 过渡 */
    --m-transition-fast: 0.15s ease;
    --m-transition-normal: 0.25s ease;
    --m-transition-slow: 0.35s ease;
}
```

### 3.2 色彩角色对照表

| 用途 | CSS 变量 | 色值 | 说明 |
|------|---------|------|------|
| 主背景 | `--m-bg` | `#FFFDF8` | 温暖奶油白背景 |
| 卡片背景 | `--m-surface` | `#FFFFFF` | 白色卡片 |
| 次级背景 | `--m-surface2` | `#FFF8EC` | 暖白输入框背景 |
| 主色粉 | `--m-pink` | `#FF6F9F` | 保存按钮、悬浮加号、主题色 |
| 主蓝色 | `--m-blue` | `#2D9CFF` | 首页选中态、复制按钮、链接文字 |
| 柔和绿 | `--m-green` | `#72D879` | 正向提示词、标签、备份 |
| 柔和黄 | `--m-yellow` | `#FFD15C` | 收藏星标、编辑按钮 |
| 柔和紫 | `--m-purple` | `#B99CFF` | 标签、导出 JSON、分类按钮 |
| 浅粉卡片 | `--m-pink-bg` | `#FFF0F5` | 分类卡片、负向提示词 |
| 主文字 | `--m-text` | `#342721` | 标题、重要文字 |
| 次级文字 | `--m-text2` | `#695B55` | 说明、时间、辅助信息 |
| 弱化文字 | `--m-text3` | `#958983` | 占位符、禁用态、补充信息 |
| 描边色 | `--m-border` | `#EFE8DE` | 输入框、卡片边框 |

### 3.3 色彩使用规则

```
背景层级：--m-bg (页面) → --m-surface (卡片) → --m-surface2 (输入框) → --m-surface3 (强调区)
文字层级：--m-text (主要) → --m-text2 (次要) → --m-text3 (最弱)
品牌强调：--m-pink (主CTA) + --m-pink-light (浅底) + --m-pink-bg (卡片底)
功能区分：--m-green (正向) / --m-pink (负向) / --m-blue (信息) / --m-purple (标签) / --m-yellow (收藏)
```

### 6.2 阴影体系

| 层级 | CSS 变量 | 值 | 使用场景 |
|------|---------|-----|---------|
| 微阴影 | `--m-shadow-sm` | `0 2px 8px rgba(120,90,50,0.06)` | 列表项、小卡片 |

## 九、动画与过渡

### 9.1 过渡时间

| 变量 | 值 | 使用场景 |
|------|-----|---------|
| `--m-transition-fast` | 0.15s ease | 按钮 hover、标签切换、小元素状态变化 |
| `--m-transition-normal` | 0.25s ease | 页面切换、弹窗显隐、主题切换 |
| `--m-transition-slow` | 0.35s ease | 复杂动画、吉祥物动画 |

### 9.2 页面切换动画

| 动画 | 方向 | CSS |
|------|------|-----|
| 前进（push） | 新页面从右侧滑入 | `translateX(100%) → translateX(0)` |
| 后退（pop） | 当前页面向右侧滑出 | `translateX(0) → translateX(100%)` |
| 模态弹出 | 从底部滑入 | `translateY(100%) → translateY(0)` |

### 9.3 组件动画

| 组件 | 动画 | 时长 |
|------|------|------|
| Toast | 从顶部滑入 | 0.3s |
| 卡片 | 淡入 + 轻微上移 | 0.25s |
| 悬浮按钮 | 缩放弹入 | 0.3s |
| 标签切换 | 背景色渐变 | 0.15s |

### 9.4 全局触控反馈

移动端触屏环境不能依赖 hover，所有关键操作必须以 tap / active 反馈为主：

- 主按钮、保存、确认、同步、备份、下载等入口点击时使用 `scale(0.96)` 到 `scale(0.98)` 的即时下压反馈。
- 顶部返回、更多、收藏、复制、图片查看器关闭和下载等图标按钮需要同时提供背景、透明度或缩放反馈。
- 卡片入口、列表行、分类项、设置动作卡点击时只做轻微下压，不改变卡片尺寸和文本流。
- `@media (hover: hover) and (pointer: fine)` 仅用于桌面浏览器调试、平板外接鼠标或触控板增强，真实手机交互不以 hover 为前提。
- 所有可聚焦按钮、输入和弹层操作必须提供 `focus-visible` 轮廓。
- `prefers-reduced-motion: reduce` 下关闭缩放、位移、投影动画和关键帧，仅保留颜色、边框、透明度状态。

## 十、负面设计约束

以下风格**严禁**出现在移动端 UI 中：

- ❌ 暗黑赛博朋克风
- ❌ 大面积深色背景
- ❌ 霓虹过度
- ❌ 网页端 UI / 后台管理系统风格
- ❌ 信息过密、杂乱排版
- ❌ 过多渐变
- ❌ 强烈玻璃拟态
- ❌ 厚重阴影
- ❌ 真实照片风格
- ❌ 3D 厚重卡通
- ❌ 非移动端比例
- ❌ 英文为主的界面

## 十一、设计规范检查清单

新增组件时必须遵循：

- [ ] 使用移动端 CSS 变量（`var(--m-xxx)`），不硬编码颜色值
- [ ] 奶油白背景上的文字对比度 ≥ 4.5:1（WCAG AA）
- [ ] 所有交互元素有触摸态（:active）反馈
- [ ] 所有关键按钮、图标按钮和卡片入口有统一的 tap / focus-visible / disabled 状态
- [ ] `prefers-reduced-motion: reduce` 下不播放缩放、位移、投影或关键帧动效
- [ ] 可点击区域 ≥ 44×44px
- [ ] 圆角遵循 `--m-radius-sm/md/lg/xl/full` 体系
- [ ] 阴影遵循 `--m-shadow-sm/md/lg` 体系
- [ ] 过渡动画使用 `--m-transition-fast/normal/slow`
- [ ] 所有选择器以 `.mobile-app` 为根前缀，避免与 PC 端冲突
- [ ] 输入框字号 ≥ 16px（防止 iOS 自动缩放）
- [ ] 文字使用简体中文
