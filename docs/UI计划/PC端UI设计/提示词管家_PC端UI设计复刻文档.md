# 《提示词管家》PC端卡通风格 UI 设计复刻文档

> 用途：将移动端 6 个界面重设计为 PC 端桌面应用界面，并整理为可供 AI 生图 / UI 复刻 / UI 设计参考的 Markdown 文档。  
> 目标：保持原移动端“提示词管家”的可爱卡通品牌风格，同时强化 PC 端信息密度、横向布局、鼠标操作效率与真实产品感。

---

## 1. 项目定位

### 1.1 产品名称

**提示词管家**

### 1.2 产品类型

本地提示词管理器 / AI 创作提示词资料库 / 桌面端提示词资产管理工具。

### 1.3 核心卖点

- 本地优先
- 隐私安全
- 提示词分类管理
- 标签管理
- 收藏与版本记录
- 快速新建 / 编辑 / 导入提示词
- 支持本地备份、导入、导出 JSON

### 1.4 目标平台

**PC 端 / 桌面端 / 16:9 横版应用界面**

推荐生成比例：

```text
16:9
1920×1080
或 1672×941
或 1440×810
```

## 2. 整体视觉风格

### 2.1 风格关键词

```text
PC端桌面应用界面、浅色主题、卡通风格、可爱柯基吉祥物、提示词管理工具、本地数据安全、圆角卡片、柔和阴影、浅奶油色背景、低饱和马卡龙配色、干净留白、现代产品级UI、轻量化工具软件、中文界面、真实可用的桌面软件界面
```

### 2.2 视觉气质

整体应呈现：

- 友好、可爱、轻松
- 干净、舒适、低压力
- 桌面端产品级 UI，而不是移动端放大版
- 卡通但不过度幼稚
- 信息清晰，层级明确
- 适合长期使用的创作工具界面

- 圆润可爱

- 白色兔子

## 3. PC 端整体布局规范

### 3.1 总体结构

PC 端采用 **左侧固定导航栏 + 右侧主工作区** 的结构。

```text
┌───────────────────────────────────────────────┐
│ 左侧侧边栏 │              主内容区             │
│            │                                   │
│ Logo       │   页面标题 / 顶部横幅 / 搜索       │
│ 导航菜单   │   数据卡片 / 表格 / 表单 / 详情     │
│ 装饰卡片   │   底部安全提示                    │
└───────────────────────────────────────────────┘
```

### 3.2 左侧侧边栏

宽度建议：

```text
260px - 300px
```

内容结构：

1. 顶部 Logo 与产品名称
2. 隐私安全徽章
3. 主导航菜单
4. 吉祥物提示卡片
5. 本地数据大小卡片

侧边栏视觉要求：

- 背景为浅奶油色 / 浅米白色
- 与主内容区之间有极淡分割线
- 导航选中态为浅蓝色圆角条
- 图标为线性图标，选中时变为蓝色
- 导航图标悬浮时轻微上浮、放大并变为蓝色，按钮内部出现柔和蓝色光感；悬浮动效不得改变按钮尺寸、文本位置或侧边栏宽度
- 导航图标点击后播放一次短促回弹，按钮内部出现柔和蓝色高光；动效不得改变按钮尺寸、文本位置或侧边栏宽度，并需要尊重 `prefers-reduced-motion`
- 本地数据大小卡片标题左侧使用 `UI设计稿/图标/插画设计/保存.png` 作为保存插画图标
- 本地数据大小卡片右侧展示圆形统计环；数据加载时从 0% 递增到真实占比，鼠标悬浮时圆环轻微放大、回弹、倾斜并出现淡色光晕，离开后柔和复位
- 文案保持简体中文

### 3.3 主内容区

主内容区应具备：

- 充足横向空间
- 卡片式分区
- 大圆角面板
- 清晰的标题、搜索、筛选、内容列表
- 底部安全提示条

主内容区背景建议：

```text
#FFFDF8
#FFFCF5
#FDFBF7
```

顶部可加入淡蓝天空、云朵、星星、心形等轻装饰，但不能影响信息阅读。

## 5. 全局固定组件

### 5.1 左侧导航栏

必须贯穿 6 个 PC 页面。

导航项：

```text
首页
提示词库
提示词详情
新建/编辑
分类与标签
设置
```

每个页面对应高亮当前导航项。

导航交互反馈：

- 鼠标悬浮导航项时，图标执行轻微上浮、缩放和蓝色染色，作为点击前的预热反馈。
- 点击导航项时，图标执行 300ms 左右的一次性轻微上跳、缩放回弹和小角度复位。
- 点击当前已激活页面时仍保留图标反馈，帮助用户确认操作已被接收。
- 侧边栏收起态下仅播放图标层动效，按钮宽高保持 48px，不产生布局跳动。
- 开启系统减弱动态后，关闭悬浮位移、投影、回弹和柔光动画，仅保留颜色状态变化。

### 5.2 隐私安全徽章

位于侧边栏顶部 Logo 下方。

文案：

```text
本地优先 · 隐私安全
```

样式：

- 粉色浅底
- 圆角胶囊
- 左侧盾牌或爱心盾牌图标
- 文字为粉红色

### 5.2.1 全局按钮与卡片入口微交互

PC 端所有主按钮、次按钮、图标按钮、快捷卡片和列表入口需要遵循同一套轻量反馈：

- 主按钮、导入导出、保存、同步和详情操作按钮悬浮时轻微上浮，按钮阴影变柔和，点击时短促下压。
- 图标按钮悬浮时容器和图标同时反馈，避免只有图标漂移造成错位。
- 卡片入口和列表行悬浮时只做 1px 级上浮、边框或阴影增强，不改变宽高、间距和文本排版。
- 所有可键盘聚焦的按钮必须提供 `focus-visible` 轮廓。
- 禁用态不播放 hover、active 或点击确认动效。
- 开启系统减弱动态后，关闭位移、缩放、投影动画和关键帧，仅保留颜色、边框或透明度状态。

### 5.3 底部安全提示条

位于主内容区底部。

文案示例：

```text
本地应用完全本地运行，不联网，不收集任何数据，您的创作数据安全有保障。
```

样式：

- 浅绿色背景
- 左侧锁头图标
- 右侧箭头
- 低饱和、轻提示感

### 5.4 顶部窗口控制

PC 端需要体现桌面应用感，可在右上角加入：

```text
最小化
最大化
关闭
```

但不要做成浏览器网页标签页。

## 14. AI 复刻时的注意事项

### 14.1 关于文字

AI 生图容易产生错字。如果需要高质量中文 UI，可以采用两种方式：

1. 先生成 UI 结构与视觉风格，再在设计软件中替换文字。
2. 在提示词中明确要求“中文文字清晰、简体中文、不要乱码”。

### 14.2 关于 PC 化改造

不要简单把移动端界面横向拉宽，需要重新组织为 PC 端结构：

- 列表改为表格
- 表单改为双栏
- 详情页增加右侧信息栏
- 设置页使用分组卡片
- 分类管理使用桌面管理表格
- 首页变成仪表盘布局

### 14.3 关于卡通装饰

卡通元素只作为品牌点缀，不能压过功能内容。

推荐放置位置：

- 侧边栏底部
- 顶部横幅右侧
- 页面角落
- 安全提示卡片
- 空状态 / 提示气泡

### 14.4 关于一致性

6 个页面必须保持：

- 同一套侧边栏
- 同一套按钮样式
- 同一套按钮、图标按钮、卡片入口和列表行微交互反馈
- 同一套卡片圆角
- 同一套色彩系统
- 同一套柯基 IP
- 同一套底部安全提示条
- 同一套图标风格
