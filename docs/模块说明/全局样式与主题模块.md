# 全局样式与主题模块

## 职责

提供跨端主题语义 Token、外观与工作台主题状态、PC/移动端样式入口及启动屏首屏样式，并定义全局 CSS 的验收边界。

## 源码位置

- `src/css/theme-tokens.css`
- `src/css/pc.css`
- `src/css/mobile.css`
- `src/index.html`
- `src/js/main.js`
- `src/js/theme-service.js`

## 加载链

`main.js` 先加载 `theme-tokens.css`，再根据端类型加载 PC 或移动端应用模块。PC 应用通过 `pc.css` 聚合有序样式区段；移动端应用通过 `mobile.css` 加载移动端样式。启动屏样式位于 `index.html` 内联 CSS，是主 Token 链的例外，必须纳入首屏验收。

## 状态模型

- `data-appearance`：浅色或深色外观映射。
- `data-workbench-theme`：sky、rose、caramel、forest、night、mint 工作台主题。
- 固定状态色与内容索引色不受工作台主题覆盖。
- 主题服务负责持久化、根节点属性应用和外观偏好恢复。

## 验收入口

全局 CSS 验收遵循 `docs/计划文档/07-测试验证/全局CSS验收测试计划-260716.md`。

- 自动化最低门槛：`npm test` 与 `npm run build`。
- 浏览器验收覆盖启动屏、双端页面、主题矩阵、响应式、焦点、低动效和动态弹层。
- CSS 入口、Token、最终覆盖层或动态弹层变更必须执行全量验收；局部页面变更按计划的变更触发规则执行关联验收。

## 维护说明

- 主题 Token 变更不得直接覆盖固定状态色和内容索引色。
- 不得绕过 PC/移动端稳定样式入口导入局部样式文件。
- 启动屏内联样式改动必须验证浅色、系统深色和应用挂载后的移除过程。
- 设置页的 PC 面板、列表、存储环、开关及移动端设置反馈必须通过语义表面、边框、状态和阴影 Token 取值；深色模式不得保留暖棕投影或亮色硬编码。
