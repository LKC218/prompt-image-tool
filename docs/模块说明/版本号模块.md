# 版本号模块

## 职责

统一读取应用版本号，并为 PC 与移动端设置页和 PC 更新记录模块提供可复用的版本来源。

## 源码位置

- `src/js/version-info.js`
- `src/js/release-notes.js`
- `src/js/release-notes-data.js`

## 设计要点

- **单一读取入口**：`getVersion()` 从 `index.html` 的 `<meta name="version">` 读取版本号，缺省回退到 `2.4.0`。
- **渲染与挂载分离**：`renderVersionInfo(options)` 生成 HTML 字符串；`mountVersionInfo(containerEl)` 在 DOM 已渲染后动态回填版本号文本。
- **跨端复用**：支持通过 `tag` 参数选择根元素标签名，PC 端使用 `span` 融入 `.pc-settings-meta-row`，移动端使用默认 `div` 作为独立信息行。
- **标签可配置**：`label` 参数可选，PC 端当前不显示标签，移动端可扩展显示“版本”等文案。
- **更新记录已读状态**：PC 更新记录以 `pc-release-notes-last-seen-version` 保存已确认阅读的版本；当前版本与已读版本不一致时，侧栏入口显示提示点并在启动后自动展示。
- **运行时数据**：`release-notes-data.js` 保留适合前端渲染的精简结构化内容，完整发布档案继续维护在 `docs/版本记录/changelog.md`，不在运行时解析 Markdown。

## 使用方式

```javascript
import { renderVersionInfo } from './version-info.js';

// PC 端：融入 meta 行
${renderVersionInfo({ tag: 'span', className: 'pc-version-info', label: '' })}

// 移动端：独立信息行
${renderVersionInfo({ className: 'm-version-info' })}
```

## 关联文件

- `src/index.html`：版本号来源 `<meta name="version" content="2.4.0">`。
- `src/js/pc-settings.js`：PC 设置页本地存储面板中渲染版本号。
- `src/js/mobile-settings.js`：移动端设置页本地存储区块下方渲染版本号。
- `src/css/pc.css`：`.pc-version-info` 样式。
- `src/css/mobile.css`：`.m-version-info` 样式。
- `src/css/pc.css`：`.pc-release-notes-*` 轻拟态弹窗及侧栏未读提示样式。
- `src/js/pc-app.js`：在设置入口并列位置提供更新记录按钮，并于桌面端启动后检查当前版本是否未读。

## 维护说明

- 发布新版本时，同步更新根目录与安装器壳的 `package.json`、Tauri 配置、Cargo 清单与锁文件，`android/app/build.gradle`、`build/installer.nsi`、`build.bat`、`README.md` 徽标以及 `src/index.html` 的 `<meta name="version">`。
- `docs/版本记录/changelog.md` 应按发布节奏补录对应版本条目。
- 发布新版本时，同步为 `src/js/release-notes-data.js` 增加当前版本的精简更新说明；发布内容以 `changelog.md` 为完整权威档案。
