# 新拟态按钮设计规范

> 参考案例钩子：Uiverse Neumorphism 标签页  
> `https://uiverse.io/elements?tags=neumorphism`

## 一、设计原则

新拟态（Neumorphism）通过双向柔和阴影模拟物体从背景中凸起或按压凹陷的质感。项目内已有同风格实践：

- `src/css/pc.css` 中的 `.pc-home-search-bar`（首页搜索栏）
- `src/css/pc.css` 中的 `.pc-library-search`（提示词库搜索栏）

二者均采用「外凸容器 + 内凹输入区」的双层结构。本规范将其抽象为一套可复用的**按钮**体系，遵循以下原则：

1. **光影即轮廓**：不使用深色边框，靠 `box-shadow` 定义立体层次。
2. **光源统一**：默认光源从左上 45° 照射，左上为高光，右下为投影。
3. **低对比背景**：按钮背景与页面背景/卡片面色接近，避免色差过大。
4. **状态仅改阴影**：默认凸起、悬停增强、按下凹陷、聚焦显环。
5. **暗部随主题微调**：基础暗部使用项目惯用的暖棕透明色；需要强调主题色时，可通过 `color-mix()` 从 `--pc-accent` / `--m-accent` 派生微色调。

## 二、Token 命名

### 2.1 PC 端

在 `.pc-app` 基础块中定义：

| Token | 含义 | 推荐值 |
|---|---|---|
| `--pc-neu-bg` | 按钮背景 | `var(--pc-surface)` |
| `--pc-neu-light` | 高光阴影（左上） | `rgba(255, 255, 255, 0.92)` |
| `--pc-neu-dark` | 投影暗部（右下） | `rgba(120, 90, 50, 0.14)` |
| `--pc-neu-dark-accent` | 随主题微调的暗部 | `color-mix(in srgb, var(--pc-accent) 22%, rgba(120, 90, 50, 0.12))` |
| `--pc-neu-offset` | 阴影偏移量 | `8px` |
| `--pc-neu-blur` | 阴影模糊半径 | `16px` |
| `--pc-neu-radius` | 按钮圆角 | `var(--pc-radius-md)`（16px） |
| `--pc-neu-radius-full` | 圆形按钮圆角 | `9999px` |
| `--pc-neu-transition` | 过渡曲线 | `0.2s ease` |

## 三、调用方式

### 3.1 PC 端

```html
<button class="pc-neu-btn">主要操作</button>
<button class="pc-neu-btn pc-neu-btn--icon" aria-label="搜索">
    <svg>...</svg>
</button>
<button class="pc-neu-btn pc-neu-btn--pressed">已激活</button>
<button class="pc-neu-btn pc-neu-btn--accent">主题强调</button>
```

```css
/* 基础凸起 */
.pc-neu-btn {
    display: inline-flex;
    align-items: center;
    justify-content: center;
    gap: var(--pc-space-sm);
    padding: 14px 28px;
    border: none;
    border-radius: var(--pc-neu-radius);
    background: var(--pc-neu-bg);
    color: var(--pc-text);
    font: 600 15px/1.4 var(--pc-font-family);
    cursor: pointer;
    box-shadow:
        var(--pc-neu-offset) var(--pc-neu-offset) var(--pc-neu-blur) var(--pc-neu-dark),
        calc(-1 * var(--pc-neu-offset)) calc(-1 * var(--pc-neu-offset)) var(--pc-neu-blur) var(--pc-neu-light);
    transition: box-shadow var(--pc-neu-transition), transform var(--pc-neu-transition);
}

/* 悬停：增强光影 */
.pc-neu-btn:hover {
    box-shadow:
        calc(var(--pc-neu-offset) + 2px) calc(var(--pc-neu-offset) + 2px) calc(var(--pc-neu-blur) + 4px) var(--pc-neu-dark),
        calc(-1 * var(--pc-neu-offset) - 2px) calc(-1 * var(--pc-neu-offset) - 2px) calc(var(--pc-neu-blur) + 4px) var(--pc-neu-light);
}

/* 按下：内凹 */
.pc-neu-btn:active,
.pc-neu-btn--pressed {
    box-shadow:
        inset calc(var(--pc-neu-offset) - 2px) calc(var(--pc-neu-offset) - 2px) calc(var(--pc-neu-blur) - 4px) var(--pc-neu-dark),
        inset calc(-1 * var(--pc-neu-offset) + 2px) calc(-1 * var(--pc-neu-offset) + 2px) calc(var(--pc-neu-blur) - 4px) var(--pc-neu-light);
    transform: translateY(1px);
}

/* 聚焦：品牌环 */
.pc-neu-btn:focus-visible {
    outline: var(--td-focus-outline);
    outline-offset: var(--td-focus-outline-offset);
}

/* 图标按钮 */
.pc-neu-btn--icon {
    width: 48px;
    height: 48px;
    padding: 0;
    border-radius: var(--pc-neu-radius-full);
}

/* 主题强调：暗部混入主题色 */
.pc-neu-btn--accent {
    box-shadow:
        var(--pc-neu-offset) var(--pc-neu-offset) var(--pc-neu-blur) var(--pc-neu-dark-accent),
        calc(-1 * var(--pc-neu-offset)) calc(-1 * var(--pc-neu-offset)) var(--pc-neu-blur) var(--pc-neu-light);
}
```

## 五、与现有搜索栏的关系

- `.pc-home-search-bar` 与 `.pc-library-search` 是**复合输入组件**，不是纯按钮，保持现有实现不变。
- 但新增搜索相关的图标按钮、筛选按钮、操作按钮时，优先使用 `.pc-neu-btn` / `.m-neu-btn` 系列，保证视觉语言统一。
- 如需将搜索栏外层也纳入 Token 体系，可后续用 `--pc-neu-*` 替换其硬编码阴影值，属于可选重构。

## 六、何时使用 / 何时避免

**适合使用**：
- 设置页、操作面板、工具栏等低频高密度按钮场景。
- 需要与现有搜索栏、卡片保持同一视觉语言的页面。

**建议避免**：
- 主流程中的强转化 CTA（拟态低对比会削弱号召力）。
- 深色背景或高饱和度背景（新拟态对浅色背景依赖强）。
- 文本密集列表中的行内操作（容易与背景糊成一片）。

## 八、标签/胶囊变体

### 8.1 使用场景
提示词库页的分类/标签筛选按钮：低频高密度、需要与搜索栏保持同一视觉语言。

### 8.2 参考来源
CodePen **CSS Skeuomorphic Button**（Matteo Piazza / @arcadejhs）  
`https://codepen.io/arcadejhs/pen/jOEBMyB`

核心技法：多层 `box-shadow` 叠加（外高光 + 外投影 + 细微 inset）模拟真实材质；`text-shadow` 增加文字厚度；`hover` 用 `scale(.98)` 制造按压预感；`active` 用 `translateY` + 深色 inset 呈现凹陷。

### 8.3 颜色适配
本项目背景为暖白（`#FFFDF8`），按钮面为纯白（`#FFFFFF`），因此把 CodePen 的冷灰暗部替换为项目惯用的暖棕暗部 `rgba(120, 90, 50, .1~.14)`，高光保持暖白 `rgba(255, 255, 255, .85~.9)`。

### 8.4 PC 端
- 类名：`.pc-library-filter-btn` / `.pc-library-filter-active`。
- 默认态：6 层阴影（左上外高光双层、右下外投影双层、顶部/底部 inset 边缘光），并加极轻文字阴影。
- 悬停态：`scale(0.98)`，阴影收紧加深，文字切换为主题色。
- 按下/激活态：`translateY(3px)`，inset 内凹阴影主导，保留 `var(--pc-accent-light)` 背景与 `var(--pc-accent)` 文字，文字阴影反转。
- 激活 + 悬停：保持内凹阴影与下移，避免悬停把激活态顶回凸起。
- 过渡：`0.3s ease-out`，比基础按钮更柔和。

### 8.5 移动端
- 类名：`.m-filter-tag` / `.m-filter-active`。
- 胶囊形（`border-radius: var(--m-radius-full)`）。
- 默认态：4-5 层阴影，尺寸比 PC 略小。
- 按下/激活态：`translateY(2px)` + inset 内凹阴影 + 主题色文字。
- 无独立 hover，按下与激活合并。
- 聚焦：2px 主题色聚焦环。

### 8.6 与基础按钮的差异
- 标签以「选中/未选中」为主要状态，因此用内凹阴影表达激活，而不是 `.pc-neu-btn--pressed` 的纯视觉按下。
- 保留主题色背景与文字，确保激活项在浅色页面上足够醒目。
- 使用多层阴影替代基础按钮的双层阴影，立体感更强。
- 移动端无独立 hover，按下与激活合并。

## 十、侧边栏折叠按钮变体

> 参考来源：Uiverse `00Kubi/red-crab-76`  
> `https://uiverse.io/00Kubi/red-crab-76`

### 10.1 复刻范围

将 PC 端侧边栏的「收起/展开侧栏」按钮（`.pc-sidebar-toggle`）升级为同款 3D 新拟态风格，并完整复刻以下动效：

- 双层渐变外壳（`::before` 内层高光 + `::after` 边框渐变）。
- 多层柔和阴影（默认凸起 / hover 增强 / active 按下）。
- hover 旋转 conic-gradient 主题色描边。
- 文字拆分后的 hover 波动与点击消失动画。
- 图标点击飞走（take-off）与重新着陆（land）动画。
- 点击尾迹（contrail）效果。

### 10.2 旋转描边优化

为了让 hover 时的 conic-gradient 描边更丝滑地围绕按钮外壳旋转，对 `.pc-sidebar-toggle-outline` 做了以下调整：

- **描边容器对称**：将 `inset` 从非对称的 `-2px -3.5px` 改为对称的 `-3px`，使描边在按钮四周等宽，避免旋转时出现粗细跳动。
- **渐变断点羽化**：将单一的 `transparent 60% → var(--pc-accent) 85% → transparent 100%` 改为多层过渡：
  - `transparent 74%`
  - `color-mix(in srgb, var(--pc-accent) 35%, transparent) 82%`
  - `var(--pc-accent) 88%`
  - `color-mix(in srgb, var(--pc-accent) 35%, transparent) 94%`
  - `transparent 100%`
  
  这样色带从约 25% 收窄到约 12%，边缘经过两层透明过渡，旋转时更像一条光带在流动。
- **外发光增强**：hover 时通过 `filter: drop-shadow(...)` 给描边增加主题色柔光，强化丝滑拖尾感。
- **动画提速**：旋转周期从 2.5s 缩短到 2s，使 hover 反馈更跟手。

### 10.3 与源案例的差异

为适配「切换按钮」语义，做了以下调整：

- 不使用 `:focus` 触发飞走，而是点击时由 JS 添加 `.is-flying` 临时类，动画结束后再切换状态。
- 不保留「发送成功」状态，而是飞走后文字切换为对向状态（收起侧栏 ↔ 展开侧栏）。
- 文字拆分为单字 span，每个字按 `--i` 延迟。
- 折叠态（`.pc-sidebar-collapsed`）下按钮缩小为 44px 图标按钮，文字隐藏。

### 10.4 源码落点

- HTML 结构：`src/js/pc-app.js` 的 `renderShell()` 与 `applySidebarState()`。
- 样式与动画：`src/css/pc.css` 的 `.pc-sidebar-toggle` 系列规则。
- 交互时序：`src/js/pc-app.js` 的 `setupSidebarToggle()`。

### 10.5 无障碍

- 按钮仍通过 `aria-label` / `title` / `aria-expanded` 描述状态。
- 文字 span 使用 `aria-hidden="true"`，避免屏幕阅读器重复朗读拆分后的单字。
- 支持 `prefers-reduced-motion`，在该媒体查询下禁用所有动画与过渡。

## 十一、首页统计卡片变体

### 11.1 使用场景

首页顶部的四格统计卡片（提示词总数 / 分类 / 标签 / 收藏）需要与搜索栏、侧边栏切换按钮保持同一套新拟态视觉语言，同时保留蓝/黄/紫/粉的语义分类色。

### 11.2 设计要点

- **卡片主体**：从「语义色填充 + 实线边框」改为「凸起面 + 双向柔和阴影」，背景统一为 `--pc-neu-bg` / `--m-neu-bg`。
- **语义色表达**：不再用整块卡片背景表达分类，而是保留图标（PC）与文字颜色，并在卡片顶部使用**居中渐变描边**作为分类识别锚点，两侧透明淡出，叠加极淡同色外发光。
- **图标槽（PC）**：将原来的圆角方形图标底改为**圆形内凹槽**，用 `inset` 双向阴影模拟按压凹陷，图标颜色继承语义色。
- **交互状态**：
  - PC 端：`hover` 外凸阴影增强并微上浮；`:active` 切换为内凹阴影 + `translateY(1px)`。
  - 移动端：无独立 hover，`:active` 直接内凹 + `scale(0.97)`。
- **数字稳定性**：统计数字使用 `font-variant-numeric: tabular-nums`，避免数值变化时卡片宽度抖动。

### 11.3 PC 端

- 类名：`.pc-stat-card` / `.pc-stat-card-blue|yellow|purple|pink` / `.pc-stat-icon`。
- 卡片默认态：凸起双向阴影，无 border，背景 `--pc-neu-bg`。
- 顶部渐变描边：通过 `::before` 实现，高 3px，位于卡片顶部，`border-radius: 18px 18px 0 0`；背景为 `linear-gradient(90deg, transparent, var(--语义色), transparent)`，并叠加 `box-shadow: 0 2px 6px color-mix(...)` 的同色微光。
- 卡片 hover：阴影偏移与模糊各增 2px/4px，同时 `translateY(-2px)`。
- 卡片 active：inset 内凹阴影，`translateY(1px)`。
- 图标槽：圆形（`border-radius: 50%`），背景 `rgba(255,255,255,0.42)`，inset 双向阴影；语义色类仅控制图标与标签文字颜色。

### 11.4 移动端

- 类名：`.m-stat-card` / `.m-stat-blue|yellow|purple|pink`。
- 卡片默认态：凸起双向阴影，无 border，背景 `--m-neu-bg`；数字与标签居中对齐。
- 顶部渐变描边：通过 `::before` 实现，高 2px，位于卡片顶部，`border-radius: var(--m-radius-md) var(--m-radius-md) 0 0`；背景为 `linear-gradient(90deg, transparent, var(--语义色), transparent)`，并叠加同色微光。
- 卡片 active：inset 内凹阴影 + `scale(0.97)`。
- 语义色类：仅保留文字颜色，不再设置背景色。

### 11.5 与按钮变体的差异

- 统计卡片不是操作按钮，因此不设置聚焦环（由容器焦点策略决定）；PC 端保留 hover 视觉反馈，移动端仅保留按下反馈。
- 图标区使用内凹阴影表达「槽」而不是凸起按钮。
- 语义色从背景色退化为点缀色，避免新拟态低对比背景下信息丢失。

### 11.6 源码落点

- HTML 结构：`src/js/pc-home.js` 的 `renderStatCard()` 与 `src/js/mobile-home.js` 的静态模板。
- 样式：`src/css/pc.css` 的 `.pc-stat-card` / `.pc-home-page .pc-stat-card` / `.pc-home-page .pc-stat-icon` 系列；`src/css/mobile.css` 的 `.mobile-app .m-stat-card` 系列。
