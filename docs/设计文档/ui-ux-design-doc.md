# 生图提示词管理器 — UI/UX 设计文档

> 版本：2.4.0 | 最后更新：2026-07-16

---

## 一、设计理念

### 1.1 核心原则

| 原则 | 描述 |
|------|------|
| **外观与主题分离** | 浅色、深色、跟随系统与定时切换控制阅读环境；工作台主题仅控制品牌互动与柯基叙事氛围 |
| **内容聚焦** | 界面元素服务于内容，减少装饰性干扰，让提示词和图片成为视觉中心 |
| **渐进展示** | 从集合列表 → 版本详情 → 图片管理，层级递进，避免信息过载 |
| **即时反馈** | 所有操作均有 Toast 通知、hover 状态或动画过渡，用户始终感知系统状态 |
| **平台自适应** | 同一套代码自动适配桌面端并排布局和移动端切换布局，CSS 变量驱动响应式尺寸 |

### 1.2 视觉风格关键词

`可靠清爽` `柯基灵感管家` `深蓝墨夜` `圆角卡片` `微交互` `CSS变量体系`

#### 色彩使用规则

```
背景层级：--bg (最深) → --surface → --surface2 → --surface3 (最浅)
文本层级：--text (主要) → --text2 (次要) → --text3 (最弱)
交互强调：--accent (默认) → --accent2 (hover/渐变) + --accent-bg (背景)
状态反馈：--success (正面) / --warning (中性) / --danger (负面)
场景专用：--viewer-bg / --overlay-bg / --drag-highlight / --card-action-bg
```

### 2.6 层级体系

通过背景色深浅、边框颜色和阴影区分层级：

```
层级 0（页面）：--bg (#0f0f0f / #f5f5f5)
层级 1（容器）：--surface + 1px solid --border + --shadow-sm
层级 2（控件）：--surface2 + 1px solid --border
层级 3（浮层）：--surface + --overlay-bg 遮罩 + --shadow-lg
层级 4（顶层）：--surface + --viewer-bg 遮罩 + --shadow-lg
层级 5（菜单）：--surface + --shadow-lg + scale 动画
```

### 2.7 过渡体系

| 变量 | 值 | 使用场景 |
|------|-----|---------|
| `--transition-fast` | `0.15s ease` | 按钮 hover/focus、列表项 hover、版本标签 hover、图片卡片 hover、输入框焦点、右键菜单显隐 |
| `--transition-normal` | `0.25s ease` | 主题切换、弹窗动画、版本内容淡入、空状态淡入 |

### 3.2 Header（顶部导航栏）

| 属性 | 值 |
|------|-----|
| 定位 | `sticky top: 0`，始终可见 |
| 层级 | `z-index: 100` |
| 背景 | `var(--surface)` + 底部 1px 边框 + `--shadow-sm` |
| 内边距 | `var(--header-padding)`（桌面：12px 24px / 移动端：12px 16px） |
| 布局 | `flex`，两端对齐 |

**左侧**：应用图标（`img.header-icon` 24×24）+ 标题文字（渐变紫色）
**右侧**：导出按钮 + 导入按钮 + 同步按钮（仅 Capacitor 环境显示）+ 主题切换按钮 + IP 显示（仅 PC 端，动态插入）

### 3.3 Sidebar（侧边栏）

| 属性 | 值 |
|------|-----|
| 宽度 | 300px（固定，`min-width: 300px`） |
| 背景 | `var(--surface)` + 右侧 1px 边框 |
| 布局 | `flex-direction: column`，纵向五段式 |

**五段结构**：
1. **Sidebar Header**：标题"提示词集合"（大写 + 字间距）+ 视图切换按钮（☰）+ 「+ 新建」按钮
2. **Search Box**：搜索输入框，`placeholder: "搜索提示词..."`，带🔍图标
3. **Folder Filter**（文件夹视图时显示）：筛选按钮组（"全部" / "未分类"）
4. **Folder List + Prompt List**：可滚动列表区域（`flex: 1; overflow-y: auto`）
5. **Sidebar Footer**（文件夹视图时显示）：「+ 新建文件夹」按钮（全宽）

### 3.4 Main Content（主内容区）

| 属性 | 值 |
|------|-----|
| 布局 | `flex: 1`，占满剩余宽度 |
| 内边距 | `var(--content-padding)`（桌面：24px / 移动端：16px） |
| 滚动 | `overflow-y: auto`，内容超出时纵向滚动 |

**两种状态**：
- **空状态**（`empty-state`）：居中显示 SVG 图标 + 引导文案
- **详情状态**（`detailView`）：集合名称 + 版本标签 + 版本详情 + 版本对比

## 五、响应式设计

### 5.1 断点定义

| 断点 | 宽度范围 | 目标设备 |
|------|---------|---------|
| 桌面端 | > 768px | PC、笔记本 |
| 平板/手机 | ≤ 768px | 平板、大屏手机 |
| 小屏手机 | ≤ 480px | 小屏手机 |

### 5.2 布局策略

#### 桌面端（> 768px）：并排布局

```
┌──────────┬───────────────────────┐
│ Sidebar  │   Main Content        │
│ (300px)  │   (flex: 1)           │
│          │                       │
│ 列表常驻  │   详情常驻             │
└──────────┴───────────────────────┘
```

- 侧边栏和主内容区并排显示
- 侧边栏固定 300px 宽度
- 点击列表项即时更新右侧详情

#### 移动端（≤ 768px）：切换布局

```
列表视图：                    详情视图：
┌──────────────────────┐     ┌──────────────────────┐
│ [← 返回]             │     │                      │
│                      │     │                      │
│   集合列表            │     │   版本详情            │
│                      │     │                      │
│                      │     │                      │
└──────────────────────┘     └──────────────────────┘
```

- 侧边栏和主内容区互斥显示
- 点击集合 → 隐藏列表，显示详情
- 点击「← 返回」→ 隐藏详情，显示列表
- 通过 `body.mobile` 类名控制显示切换

### 5.3 CSS 变量驱动的响应式

通过 CSS 自定义属性在断点处覆盖值，避免大量媒体查询重复：

```css
:root {
    --content-padding: 24px;
    --image-grid-min: 200px;
    /* ... */
}
@media (max-width: 768px) {
    :root {
        --content-padding: 16px;
        --image-grid-min: 140px;
        /* ... */
    }
}
```

| 列表项 | 透明背景 | `--surface2` + `--shadow-sm` | - | `--accent-bg` + `--accent` 边框 | - |
| 文件夹项 | 透明背景 | `--surface2` | - | `--accent-bg` + `--accent` 边框 | - |
| 版本标签 | `--surface2` | `--accent-bg` + `--accent` 边框 | - | `--accent` 填充 | - |
| 输入框 | `--border` 边框 | - | 边框→`--accent` + `3px --accent-bg` 外阴影 | - | - |
| 名称输入 | 透明，无边框 | - | 底部 2px `--accent` | - | - |
| 图片卡片 | `--border` 边框 | 边框→`--accent` + `--shadow-md` + 上移 2px | - | - | - |
| 添加图片区 | 虚线 `--border` | 虚线→`--accent` + `--accent-bg` | - | - | - |
| 右键菜单 | `scale(0.95)` + 透明 | - | - | `scale(1)` + 不透明 | - |
| 颜色选择项 | 2px 透明边框 | `scale(1.15)` | - | 2px `--text` 边框 + 外阴影 | - |

#### 动画关键帧

```css
@keyframes slideIn {
    from { transform: translateX(100%); opacity: 0; }
    to   { transform: translateX(0);    opacity: 1; }
}

@keyframes fadeIn {
    from { opacity: 0; }
    to   { opacity: 1; }
}

@keyframes modalIn {
    from { opacity: 0; transform: scale(0.95) translateY(8px); }
    to   { opacity: 1; transform: scale(1) translateY(0); }
}

@keyframes zoomIn {
    from { opacity: 0; transform: scale(0.9); }
    to   { opacity: 1; transform: scale(1); }
}
```

### 6.3 操作流程

#### 创建提示词集合

```
点击「+ 新建」 → 列表新增一项 → 自动选中 → 右侧显示详情
                                      ↓
                              Toast: "已创建新提示词集合"
```

#### 删除提示词集合

```
右键集合项 → 点击「删除」 → 弹出确认弹窗 → 点击「确定」
                                            ↓
                                  列表移除该项 → 显示空状态
                                            ↓
                                  Toast: "已删除"
```

#### 添加图片

```
方式一：点击添加区域 → 系统文件选择器 → 选择图片
方式二：拖拽图片到添加区域 → 区域高亮（.drag-over） → 释放

                ↓
        图片上传处理 → 刷新版本详情 → Toast: "已添加 N 张图片"
```

#### 版本对比

```
点击「版本对比」 → 展开对比面板（替换版本详情区）
                        ↓
              选择左右版本（下拉框） → 实时更新对比内容
                        ↓
              点击图片 → 全屏查看
                        ↓
              再次点击「版本对比」 → 收起对比面板
```

#### 版本预览

```
点击「预览」 → 打开提示词预览弹窗
                    ↓
          查看正向/反向提示词 + 版本备注
                    ↓
          点击「复制」→ 复制到剪贴板 → Toast: "已复制"
                    ↓
          点击关闭按钮 / ESC → 关闭预览
```

#### 文件夹管理

```
切换到文件夹视图（☰ 按钮） → 显示文件夹列表 + 筛选按钮
                                    ↓
              点击「+ 新建文件夹」→ 弹窗输入名称 + 选择颜色 → 创建
                                    ↓
              点击文件夹 → 展开/折叠 → 显示该文件夹下的集合
                                    ↓
              右键文件夹 → 重命名 / 更改颜色 / 删除
                                    ↓
              右键集合项 → 移动到文件夹 → 选择目标文件夹
```

#### 局域网同步（仅 Capacitor 环境）

```
点击「同步」 → 打开同步弹窗 → 输入 PC 端 IP
                                    ↓
              点击「开始同步」 → 显示进度条 + 状态文字
                                    ↓
              同步完成 → 显示报告（新增/更新/跳过/冲突/图片下载统计）
                                    ↓
              点击「关闭」或「重新同步」
```

#### 主题切换

```
点击主题切换按钮 → 切换暗色/亮色主题
                        ↓
              更新 data-theme 属性 → CSS 变量自动切换
                        ↓
              更新 theme-color meta 标签
                        ↓
              图标切换：暗色模式显示太阳☀，亮色模式显示月亮🌙
                        ↓
              偏好存入 localStorage，下次打开自动应用
```

### 6.6 自动保存机制

| 输入项 | 防抖时间 | 保存方式 |
|--------|---------|---------|
| 集合名称 | 即时（`input` 事件） | `updatePromptSetName()` |
| 正向提示词 | 500ms | `setTimeout` 防抖 → `updatePromptSet()` |
| 反向提示词 | 500ms | `setTimeout` 防抖 → `updatePromptSet()`（复用同一计时器） |
| 版本备注 | 500ms | `setTimeout` 防抖 → `updatePromptSet()` |
| 图片备注 | 500ms | `setTimeout` 防抖 → `updatePromptSet()`（每张图片独立计时器） |

> 所有自动保存均无视觉反馈（无"已保存"提示），依赖用户对系统稳定性的信任。

| 成功 | Toast 提示"已复制" |
| 失败 | Toast 提示"复制失败" |

## 八、主题系统

### 8.1 主题切换机制

| 属性 | 值 |
|------|-----|
| 切换方式 | `document.documentElement.dataset.theme` 设置为 `"dark"` 或 `"light"` |
| 默认主题 | 跟随系统偏好（`prefers-color-scheme`） |
| 手动切换 | 存入 `localStorage`，优先级高于系统偏好 |
| CSS 实现 | `:root` 定义暗色变量，`[data-theme="light"]` 覆盖亮色变量 |
| 过渡动画 | `transition: background-color var(--transition-normal), color var(--transition-normal)` |

### 8.2 主题切换联动

| 联动项 | 行为 |
|--------|------|
| CSS 变量 | 自动切换所有颜色、阴影变量 |
| theme-color meta | 暗色：`#0f0f0f`，亮色：`#f5f5f5` |
| 切换按钮图标 | 暗色模式显示太阳☀图标，亮色模式显示月亮🌙图标 |
| 图片查看器 | 亮色主题下图片添加 1px 边框 |

## 九、设计规范检查清单

### 新增组件时需遵循

- [ ] 使用 CSS 变量（`var(--xxx)`），不硬编码颜色值
- [ ] 暗色/亮色背景上的文字对比度 ≥ 4.5:1（WCAG AA）
- [ ] 所有交互元素有 Hover/Focus 状态
- [ ] 过渡动画统一使用 `var(--transition-fast)` 或 `var(--transition-normal)`
- [ ] 圆角遵循 `--radius-sm` / `--radius-md` / `--radius-lg` / `--radius-full` 体系
- [ ] 阴影遵循 `--shadow-sm` / `--shadow-md` / `--shadow-lg` 体系
- [ ] 移动端输入框字号 ≥ 16px
- [ ] 移动端可点击区域 ≥ 44×44px
- [ ] 弹窗和浮层有遮罩和关闭方式
- [ ] 新增颜色值需同时在 `:root` 和 `[data-theme="light"]` 中定义

### 响应式适配时需检查

- [ ] 768px 断点：布局从并排切换为堆叠
- [ ] 480px 断点：图片网格固定 2 列
- [ ] 移动端删除按钮始终可见
- [ ] 版本标签横向滚动不换行
- [ ] Toast 位置从右上角移至底部
- [ ] 版本对比从双列变为单列
- [ ] 预览弹窗全屏显示
- [ ] 同步 IP 输入框小屏竖向排列
