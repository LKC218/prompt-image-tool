# Tauri-安装器壳-UI-设计规则

## 目标

本文档用于指导 `installer-shell/` 的 Tauri 安装器壳 UI 设计与实现。目标是以参考图片为视觉方向，搭建一个精致、可信、易理解的 Windows 安装向导界面，同时保持工程实现简单、稳定、可维护。

安装器壳只负责用户看到的界面、步骤控制和状态反馈；真正复制文件、创建快捷方式、写注册表和卸载清理仍交给 NSIS 静默安装核心完成。

## 设计定位

### 设计关键词

- 温暖
- 清晰
- 安全
- 创作感
- 轻量故事书
- 可爱但不幼稚

### 推荐视觉方向

采用“打开一本创作手册”的安装体验：

- 背景是浅蓝天空或柔和桌面场景。
- 主内容区像一本展开的书、手册或纸张画布。
- 角色插画位于书本主视觉区域，表达欢迎、陪伴和创作灵感。
- 左侧固定步骤栏，提供安装流程感和安全感。
- 底部固定操作区，保证用户永远知道下一步在哪里。

欢迎页复刻参考图时允许采用“全屏书本欢迎页”模式：首屏不展示左侧步骤栏，将角色插画、欢迎文案、卖点和按钮落在展开书本上；品牌 Logo、标题和口号统一放在窗口左上天空留白区，作为全局窗口层展示，避免遮挡书本插画和书页内容。从许可协议页开始再恢复清晰的步骤结构。

### 不推荐方向

- 不做黑色科技感安装器。
- 不做纯系统原生灰色向导。
- 不做复杂动态宣传页。
- 不做大面积紫蓝渐变背景。
- 不堆叠过多卡片和说明文字。
- 不使用在线图片、在线字体或远程图标资源。

## 界面结构

安装器窗口推荐固定尺寸：

```text
1280 x 720
```

最小可接受尺寸：

```text
1180 x 680
```

窗口结构分为四块：

```text
┌────────────────────────────────────────────┐
│ 左侧步骤栏 │ 顶部自定义标题栏               │
│            ├───────────────────────────────│
│            │ 当前步骤主内容区               │
│            │ 书本/纸张/插画/表单             │
│            ├───────────────────────────────│
│            │ 底部操作区                     │
└────────────────────────────────────────────┘
```

欢迎页主视觉必须固定在 `1280 x 720` 设计画布内实现。书本图片、欢迎标题、卖点列表和按钮需要放在同一个固定比例书本画布中定位；品牌 Logo、标题和口号固定在窗口左上天空留白区，不能遮挡书本页边、角色脸部或花枝插画。欢迎页与后续引导页的书本运行时资源必须保持同一画布比例和同一渲染基准，当前统一为 `1279 x 720` 素材画布、`1210 x 681` 页面书本画布，避免步骤切换时书本大小跳变，并让书本主体充分包住页内 UI。

欢迎页书本内的覆盖内容必须落在书页安全区内：右页标题、卖点列表、按钮不得贴近书页右侧页边、页角装饰或书本外沿。卖点图标需要使用对应语义色，不能只显示纯白图形；推荐使用本地 PNG 作为遮罩，并用 CSS 分别填充蓝色、黄色和粉色。

欢迎页底部操作按钮必须完全位于右页纸张区域内，不能压到书本蓝色外壳、书页下沿或窗口外部背景；按钮区应作为右页安全区的绝对定位元素处理，避免被卖点列表自然下推到书本外。

欢迎页漂浮装饰优先使用本地图片资源，例如 `sparkle-star.png`，不再用 CSS 临时绘制十字星。分割线中间的语义图标可使用 `pet-manage.png`，但必须使用 CSS 遮罩或着色方案，避免浅色纸张上出现纯白低对比图形。

### 书本页统一 UI 基准

后续新建安装位置、附加任务、准备安装、正在安装、安装完成等安装器壳 UI 页面时，必须先参考本节规则，再新增页面模块。

统一基准：

- 窗口画布固定为 `1280 x 720`。
- 书本素材运行时画布统一为 `1279 x 720`，不得为某个步骤单独裁成不同宽高比。
- 页面书本画布统一为 `1210 x 681`，在 `1280 x 720` 视口中的实际位置为 `x=35, y=29.5`。
- 书本图片、右页标题、正文区、表单控件和底部按钮必须都落在同一书本画布里。
- 全局品牌区固定在窗口左上天空留白区，不参与书本页内容排版。
- 新页面只在 `installer-shell/src/pages/` 增加页面模块，并在 `main.js` 注册步骤；不要新增多 HTML、前端路由或页面级样式入口。

右页标题统一基准：

- 标题面板在书本画布内统一使用 `top: 70px; right: 136px; width: 418px`。
- 第一行标题使用深棕色、`31px`、居中、加粗，用于“欢迎安装”“安装协议”等页面主语义。
- 第二行标题使用主蓝色、`29px`、居中、加粗，用于“提示词管家”“阅读许可协议”等页面强调语义。
- 首页欢迎页不得单独放大标题；标题位置和大小必须与后续引导页保持一致，避免步骤切换时视觉跳变。
- 标题周围装饰只能作为辅助点缀，不得遮挡两行标题和副标题。

右页内容安全区：

- 正文、卖点、表单、协议框和按钮不得贴近右侧页边、页角装饰、书脊或蓝色书壳。
- 底部操作区必须位于右页纸张区域内，不得压到书本蓝色外壳或窗口背景。
- 页面内容较多时，优先压缩正文框高度或让正文框内部滚动，不允许把按钮自然下推到书本外。

### 左侧步骤栏

用途：

- 告诉用户当前在第几步。
- 减少安装流程的不确定感。
- 放置品牌图标和一句轻量口号。

内容建议：

```text
提示词管家
安装向导

1 欢迎
2 许可协议
3 选择安装位置
4 选择附加任务
5 准备安装
6 正在安装
7 安装完成
```

规则：

- 当前步骤使用蓝色圆形编号。
- 已完成步骤可以使用对勾图标。
- 未到达步骤使用浅灰编号。
- 每一步只写短标题和一句辅助说明。
- 左侧步骤栏不可滚动，所有内容必须在窗口高度内可见。

### 顶部标题栏

用途：

- 承担拖拽窗口区域。
- 放置最小化和关闭按钮。

规则：

- 不使用系统默认标题栏。
- 右上角只保留最小化与关闭。
- 关闭安装时需要二次确认，避免误触。
- 标题栏高度建议 `44px` 到 `56px`。

### 主内容区

用途：

- 展示当前步骤的核心信息。
- 承载参考图中的书本、纸张、插画和表单。

规则：

- 每一页只解决一个问题。
- 首屏欢迎页可以更重视觉，后续步骤应更偏清晰表单。
- 内容区必须有固定高度，避免底部按钮被挤出窗口。
- 视觉插画优先使用本地图片资源，不建议长期使用 CSS 临时绘制角色。

### 底部操作区

用途：

- 提供取消、上一步、下一步、安装、完成等主操作。

规则：

- 底部操作区固定贴底。
- 主按钮放右侧，使用蓝色。
- 次按钮放左侧或主按钮左侧，使用白底描边。
- 每一页只允许一个主按钮。
- “正在安装”阶段禁用返回和关闭，或关闭时显示确认提示。

## 页面步骤规则

### 1. 欢迎页

目的：让用户确认“这是什么软件，安装它有什么价值”。

推荐内容：

- 大标题：欢迎安装 提示词管家
- 辅助文案：安装向导将引导您完成提示词管家的安装
- 三个核心卖点：
  - 智能提示词管理
  - 激发创作灵感
  - 本地安全可靠

视觉规则：

- 可以使用最大插画和最强氛围。
- 不出现复杂表单。
- 不展示过多技术信息。

### 2. 许可协议页

目的：让用户完成继续安装前的确认。

推荐内容：

- 协议正文区域。
- “我已阅读并同意许可协议”复选框。

规则：

- 未勾选时禁用“下一步”。
- 协议正文可滚动，但页面整体不滚动。
- 协议区使用纸张白底，避免像系统文本框一样生硬。
- 许可协议页应严格沿用参考图的双页书本结构：左页放协议阅读插画和陪伴场景，右页放协议标题、页码分割线、正文框、复选框和底部操作按钮。
- 右页标题区采用两行居中结构：主标题为深棕色，副标题使用主蓝色；标题周围只保留少量星星和爱心点缀，不遮挡文字。
- 协议正文框必须落在右页纸张安全区内，高度固定并独立滚动；滚动条使用柔和纸张色系，避免系统默认文本框的生硬感。
- “上一步 / 下一步 / 取消”按钮必须位于右页底部安全区内，不压到书本蓝色外壳、页角装饰或窗口背景。
- 复选框状态不能只靠颜色表达，选中后必须显示勾选图形；未勾选时禁用的“下一步”需要保留可理解的禁用状态。

### 3. 选择安装位置页

目的：让用户确认安装目录。

推荐内容：

```text
正式默认目录：%USERPROFILE%\Downloads\PIM-Test
原型测试目录：%USERPROFILE%\Downloads\PIM-Test
```

控件：

- 安装路径输入框。
- 浏览按钮。
- 剩余空间提示。

规则：

- 默认目录优先使用当前用户下载目录下的独立子文件夹，避免管理员权限，也避免落到系统级安装目录。
- Tauri 调度层必须在调用 NSIS 前展开 `%USERPROFILE%` 与 `%LOCALAPPDATA%` 前缀，不把带环境变量占位符的路径原样传给安装核心。
- 路径不可为空。
- 路径无权限时给出明确提示。
- 浏览目录是辅助能力，不要让用户必须选择。
- 浏览目录必须优先使用 Tauri dialog 插件的前端异步打开能力，或等价的应用内原生对话框，不通过隐藏 PowerShell/WinForms 进程中转，避免 release 窗口中崩溃、丢焦点、同步阻塞或 stdout 返回路径失败。
- 点击浏览期间需要进入明确的工作态，防止重复点击；取消选择时保持当前路径不变，选择成功后输入框、实际路径和校验提示必须同步更新为所选真实路径。
- 安装位置页必须严格沿用欢迎页和许可协议页的书本画布尺寸，不允许为了容纳表单单独缩小或放大书本图片。
- 标题位置必须沿用右页标题统一基准：标题面板使用 `top: 70px; right: 136px; width: 418px`，主标题、副标题和页码分割线只替换文案。
- 路径输入、浏览按钮、空间信息和提示条必须使用真实 DOM 控件，不使用整图切片模拟交互。
- 底部 `上一步 / 下一步 / 取消` 必须固定在右页纸张安全区内，不压到书本蓝色外壳。

### 4. 选择附加任务页

目的：让用户选择安装后的便利行为。

推荐选项：

- 创建桌面快捷方式。
- 创建开始菜单快捷方式。
- 开机后自动检查更新。

规则：

- 默认勾选桌面快捷方式。
- 开始菜单快捷方式建议默认勾选。
- 开机后自动检查更新默认不勾选，避免未确认的后台行为。
- 每个选项必须用复选框，不用按钮模拟选择。
- 附加任务页位于准备安装页之前，进入本页不得调用 `install_with_nsis`，点击“下一步”只进入准备安装页。
- 右页标题必须沿用引导页排版基准：`top: 70px; right: 136px; width: 418px`，标题和说明在上，`第 4 页 · 安装选项` 分割线在下。
- 附加任务选项、应用摘要和底部按钮必须使用真实 DOM 实现，不能切在整张设计稿里。
- 附加任务状态必须贯穿准备安装页、正在安装页和完成页；桌面快捷方式与开始菜单快捷方式可由 Tauri 收尾命令按最终选择保留或移除。
- 自动检查更新不写入系统开机任务；若主应用尚未接入独立更新设置，只能表述为“应用内更新提醒”，不得误导为系统后台自启动。

### 5. 准备安装页

目的：安装前最后确认。

推荐内容：

- 安装位置。
- 附加任务。
- 即将执行的动作。

规则：

- 主按钮文案从“下一步”改为“开始安装”。
- 信息以摘要卡片展示。
- 不再出现新的复杂设置项。
- 本页必须作为真正安装前的最后确认页；只有点击本页“开始安装”后才进入正在安装页并触发 `install_with_nsis`。
- 摘要必须展示安装目录、附加任务、安装核心校验状态和即将执行动作，避免用户从附加任务页直接进入不可逆安装过程。

### 6. 正在安装页

目的：让用户知道安装正在进行，避免误以为卡死。

推荐内容：

- 当前状态：正在准备、正在安装、正在校验、正在收尾、正在完成。
- 阶段式进度条。
- 简短说明：正在安装提示词管家，请稍候。

规则：

- 由于 NSIS 静默安装不天然提供精确文件进度，优先使用阶段式进度，不假装精确百分比。
- 右页标题必须与许可协议页、安装位置页的引导页排版保持一致，沿用 `top: 70px; right: 136px; width: 418px` 的标题面板基准、居中方式、字号层级和“标题/说明在上、页码分割线在下”的顺序。
- 进度条、当前动作、阶段列表和底部按钮必须使用真实 DOM 实现，不能切在书本图片里，方便后续由安装阶段状态驱动。
- 进度条可以显示阶段百分比，但必须由准备安装、复制文件、校验结果、收尾任务等阶段状态驱动，不声明为真实逐文件复制比例。
- 安装过程中禁用“上一步”。
- 取消安装需要谨慎；第一版可以不支持中断安装，只提供安装完成或失败结果。
- 失败状态必须停留在正在安装页内，展示失败阶段、错误原因，并提供“重试安装”和“修改设置”入口，不得跳转到完成页。
- 安装结束后必须校验 `PromptImageManager.exe` 和 `uninstall.exe` 是否存在。

### 7. 安装完成页

目的：明确告诉用户安装成功，并给出下一步。

推荐内容：

- 标题：安装完成
- 文案：提示词管家已成功安装到您的电脑
- 选项：立即启动提示词管家
- 选项：创建桌面快捷方式
- 选项：保留开始菜单入口

规则：

- 主按钮文案为“完成”。
- 如果勾选立即启动，点击完成后尝试启动主程序；启动失败不能阻塞安装窗口退出。
- 完成页必须沿用书本页统一 UI 基准，书本素材保持 `1279 x 720` 运行时画布和 `1210 x 681` 页面书本画布。
- 右页标题必须与引导页排版保持一致，标题面板宽度和右侧锚点继续沿用 `right: 136px; width: 418px`；完成页当前为收束态视觉节奏，整体面板上移到 `top: 58px`。完成页不显示页码提示，避免弱信息干扰最终成功状态。
- 成功勾选、结果文案、完成后选项和“完成”按钮必须使用真实 DOM/CSS 或本地图标实现，不能切在整张图片里。
- 完成后桌面快捷方式和开始菜单入口选项需要对齐当前安装脚本能力；若 NSIS 已创建快捷方式，完成页可以执行保留或移除收尾处理，不得误导用户以为安装前脚本已被动态改写。
- 完成页必须显示最终安装目录和收尾状态；点击“完成”后必须退出安装器壳窗口，不得停留在完成页或显示 Tauri ACL 权限错误。
- 安装失败时不进入完成页，而进入错误状态页。

## 视觉规则

### 颜色

推荐色板：

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主背景 | `#eaf6ff` | 浅蓝天空感 |
| 纸张背景 | `#fff8ea` | 温暖纸张 |
| 主文字 | `#3d2417` | 深棕，贴合手册感 |
| 辅助文字 | `#81766f` | 柔和灰棕 |
| 主按钮 | `#317cf0` | 清晰蓝色行动点 |
| 高亮蓝 | `#2f7df5` | 当前步骤和重点标题 |
| 温暖点缀 | `#ffc143` | 星星、闪光、小提示 |
| 成功 | `#31b66b` | 仅用于成功状态 |
| 警告 | `#f0a326` | 仅用于警告状态 |
| 错误 | `#df4c4c` | 仅用于错误状态 |

规则：

- 主按钮只能使用蓝色。
- 成功、警告、错误颜色只用于语义状态。
- 背景避免单一蓝色铺满，应混合纸张米白和少量温暖点缀。

### 字体

默认字体：

```css
"Microsoft YaHei", "Segoe UI", sans-serif
```

字号建议：

| 类型 | 尺寸 |
|------|------|
| 书本页第一行标题 | 31px |

## 交互规则

### 按钮

规则：

- 主按钮：下一步、开始安装、完成。
- 次按钮：上一步、取消。
- 取消按钮始终在左侧或次级位置。
- 安装中主按钮不可重复点击。
- 关闭窗口时，如果还未开始安装，可直接退出；如果正在安装，必须确认。

### 步骤切换

规则：

- 步骤切换可以有轻微淡入或位移动画。
- 动画时间控制在 `160ms` 到 `240ms`。
- 尊重系统低动效偏好。
- 不做翻书大动画作为核心交互，避免影响安装效率。

### 进度反馈

规则：

- 使用阶段式进度，不使用虚假的精确百分比。
- 安装状态文案必须真实：
  - 正在准备安装文件
  - 正在执行安装程序
  - 正在校验安装结果
  - 正在完成安装
- 失败时展示错误原因和可操作建议。

## 状态规则

必须设计以下状态：

- 默认状态。
- 当前步骤激活状态。
- 已完成步骤状态。
- 禁用按钮状态。
- 安装中状态。
- 安装成功状态。
- 安装失败状态。
- 路径无效状态。
- 协议未勾选状态。

错误提示规则：

- 不只显示“失败”。
- 要说明失败发生在哪里。
- 要给用户下一步建议。

示例：

```text
安装失败：未找到 NSIS 安装核心。
请确认安装包资源存在后重新运行安装向导。
```

## 参考图转 UI 的落地方法

看参考图时不要直接照抄所有细节，而是拆成五层：

### 1. 氛围层

参考图的天空、云朵、桌面、柔光是氛围层。

实现方式：

- 优先用背景图或少量 CSS 渐变。
- 氛围层不能影响文字可读性。

### 2. 结构层

参考图的左侧步骤栏、主书本区域、底部按钮是结构层。

实现方式：

- 使用 CSS Grid 固定大结构。
- 底部操作区固定高度和位置。

### 3. 内容层

标题、步骤文案、卖点、协议、路径选择是内容层。

实现方式：

- 每页独立组织内容。
- 不让装饰元素进入内容主流程。

### 4. 插画层

角色、书本、星星、贴纸是插画层。

实现方式：

- 第一版可以用占位插画。
- 正式版应替换成本地高质量图片。

### 5. 状态层

当前步骤、按钮禁用、安装中、失败提示是状态层。

实现方式：

- 状态必须由真实安装流程驱动。
- 不用纯视觉假状态误导用户。

## 工程实现边界

### 前端页面拆分规则

安装器壳采用单 HTML 壳、多页面模块的组织方式。`index.html` 保留固定窗口骨架、天空背景、自定义标题栏、全局品牌区和页面挂载容器，不直接承载具体步骤内容。

页面实现规则：

- 每个安装步骤拆成独立页面模块，统一放在 `installer-shell/src/pages/`。
- 页面模块只负责返回当前步骤的语义化 HTML 字符串，不直接绑定全局流程事件。
- `main.js` 负责步骤状态、页面切换、按钮事件、协议勾选状态和后续 Tauri 命令调用。
- `styles.css` 继续承载公共视觉样式和各页面专用类，页面模块优先复用既有类名，避免重复写内联样式。
- 不拆成多个 HTML 文件，不引入前端路由框架；安装向导始终由一个 Tauri 窗口内的单页状态机驱动。

当前已拆分页面：

| 页面 | 模块 | 职责 |
|------|------|------|
| 欢迎页 | `installer-shell/src/pages/welcome.js` | 渲染欢迎书本首屏、核心卖点和开始安装操作 |
| 许可协议页 | `installer-shell/src/pages/license.js` | 渲染协议书本页、协议正文、同意复选框和上一步/下一步/取消操作 |
| 安装位置页 | `installer-shell/src/pages/install-location.js` | 渲染安装位置书本页、路径选择卡片、空间信息和上一步/下一步/取消操作 |
| 附加任务页 | `installer-shell/src/pages/install-tasks.js` | 渲染安装前附加任务书本页、任务复选框、应用摘要和上一步/下一步/取消操作 |
| 准备安装页 | `installer-shell/src/pages/prepare-install.js` | 渲染安装前最终确认页、安装目录、附加任务摘要、安装核心状态和开始安装操作 |
| 正在安装页 | `installer-shell/src/pages/installing.js` | 渲染安装中书本页、阶段式进度、当前动作、阶段列表、失败重试和修改设置操作 |
| 安装完成页 | `installer-shell/src/pages/complete.js` | 渲染安装完成书本页、成功状态、完成后选项、快捷方式收尾和完成按钮闭环 |

后续新增错误状态页面时，必须先按本文档的书本页统一 UI 基准确定书本画布、标题区、右页安全区和底部操作区，再新增页面模块，并只在 `main.js` 注册步骤和状态处理。

### Tauri 负责

- 自定义窗口。
- UI 页面。
- 用户输入。
- 步骤切换。
- 校验安装路径，必要时通过 `tauri-plugin-dialog` 的 `window.__TAURI__.dialog.open` 打开原生目录选择窗口。
- 调用 NSIS 静默安装。
- 展开 `%USERPROFILE%`、`%LOCALAPPDATA%` 测试路径并把真实目标目录传给 NSIS。
- 校验安装结果。
- 根据用户选择保留或移除桌面快捷方式、开始菜单快捷方式。
- 展示成功或失败。
- 由于安装器壳前端是无打包器的静态页面，`installer-shell/src-tauri/tauri.conf.json` 必须启用 `app.withGlobalTauri`，保证 `main.js` 能通过 `window.__TAURI__.core.invoke` 调用 Rust 安装命令。
- 自定义标题栏只展示最小化和关闭按钮，不展示最大化按钮；`installer-shell/src-tauri/capabilities/default.json` 必须显式允许 `core:window:allow-close`、`core:window:allow-minimize` 和 `core:window:allow-start-dragging`。
- 桌面快捷方式和开始菜单快捷方式属于收尾任务；当主程序和卸载器已经安装成功时，快捷方式处理失败只能作为完成页警告，不得把安装流程卡在正在安装页或伪装成主安装失败。
- 正式打包时通过 `embedded-installer` feature 嵌入现有 NSIS 安装核心，保证发布产物双击后第一屏是自定义安装器壳 UI。

### NSIS 负责

- 复制文件。
- 创建快捷方式。
- 写注册表。
- 创建卸载器。
- 静默卸载清理。

### 不要让 Tauri 第一版负责

- 自己复制所有安装文件。
- 自己写完整卸载器。
- 自己处理升级覆盖逻辑。
- 自己模拟精确文件复制进度。
- 使用 Tauri 自带 NSIS/MSI bundle 作为第一屏安装体验；首版发布产物应直接启动自定义安装器壳窗口。

## 首版验收标准

首版 UI 达标条件：

- 1280 x 720 下所有内容完整可见。
- 底部按钮永远可见。
- 左侧步骤不会溢出。
- 欢迎页视觉接近参考图的“创作手册”氛围。
- 每个步骤都有清晰标题和主操作。
- 安装中不会重复触发安装。
- 安装成功和失败都有明确反馈。
- 所有资源来自本地。
