# 跨端配色与主题令牌规范

> 状态：实施中
>
> 本文档是 PC、移动端与安装器壳配色、外观与工作台主题的唯一权威来源。

## 一、主题模型

主题分为三个相互独立的维度：

| 维度 | 根节点属性或数据键 | 作用范围 |
|---|---|---|
| 外观模式 | `data-appearance` / `appearance-preference` | 表面、文字、边框、阴影、状态色的浅深映射 |
| 工作台主题 | `data-workbench-theme` / `workbench-theme` | 主操作、选中态、焦点环、欢迎区与柯基配件环境 |
| 内容索引色 | `colorKey` | 用户分类、文件夹与标签的稳定色相 |

外观模式支持浅色、深色、跟随系统与定时切换。工作台主题支持晴空巡逻、焦糖午后、松林远足、星夜侦察、海盐薄荷。状态色和用户内容索引色不得被工作台主题覆盖。

## 二、令牌层级

| 层级 | 示例 | 规则 |
|---|---|---|
| 原始色板 | `--palette-*` | 仅供 Token 定义使用 |
| 语义令牌 | `--color-surface-canvas`、`--color-surface-control`、`--color-brand-primary` | 明暗和主题覆写的唯一位置 |
| 组件别名 | `--pc-accent`、`--m-accent` | 只能引用语义令牌，用于兼容端侧组件 |

基础 Token 定义在 `src/css/theme-tokens.css`。`pc.css` 与 `mobile.css` 仅将既有端侧变量映射到语义 Token，不应再扩散新的主题十六进制色值。组件的正文、辅助文字、图标和 `currentColor` SVG 必须使用文字或状态语义 Token；不得在组件内写死暗色前景，避免绕过深色模式映射。

## 三、基础语义色

| 语义 | 浅色 | 深色 |
|---|---:|---:|
| `surface-canvas` | `#F7F9FC` | `#101822` |
| `surface-subtle` | `#EEF3F8` | `#152131` |
| `surface-raised` | `#FFFFFF` | `#1B2A3A` |
| `surface-input` | `#FFFFFF` | `#1B2A3A` |
| `surface-control` | `#FFFFFF` | `#1B2A3A` |
| `surface-control-pressed` | `#EEF3F8` | `#152131` |
| `surface-sidebar` | `#EEF3F8` | `#152131` |
| `text-primary` | `#1E2B3A` | `#E8F1FA` |
| `text-secondary` | `#596B7D` | `#AABBCD` |
| `border-subtle` | `#DCE5EF` | `#2C4055` |

深色模式采用深蓝墨夜层级，不使用纯黑画布或 `filter: invert()`。

## 四、夜航材质规则

- 深色普通卡片使用深色表面、细边框和深阴影区分层级；不得使用白色外发光或暖棕投影。
- 控件凸起、压入和轻量悬浮层分别使用 `--color-shadow-control-raised`、`--color-shadow-control-pressed`、`--color-shadow-control-soft`；端侧仅通过 `--pc-control-*`、`--m-control-*` 别名引用，禁止复写白色或暖棕新拟态阴影。
- `--color-shadow-highlight` 只用于亮色柔和高光；深色映射为低透明度冷灰蓝，不能作为容器描边光源。
- `--color-shadow-neu-dark` 统一承接端侧轻拟态暗影；深色下使用深蓝黑阴影，禁止在组件中保留暖棕色阴影。
- 浮动操作菜单使用 `surface-context-*`、`shadow-context-*` 与 `context-*-fg/bg` 令牌。菜单令牌必须在通用 `data-appearance="dark"` 层映射，不得只在某个工作台主题覆写，确保所有暗色工作台主题均使用低透明蓝灰浮层、深色控件表面和冷色轻高光；重命名、移动、复制、粘贴、删除仅使用低亮度状态底色，按压态通过下沉、内凹阴影和与当前动作色混合的压深表面确认操作。菜单操作行保持透明，禁止添加跨越文字、图标和间距的整行矩形底色；菜单项禁用全局波纹，仅保留短促的实体下沉反馈。
- 搜索框与输入框使用 `surface-input`，侧栏使用 `surface-sidebar`；焦点仅使用 `brand-focus` 环，不叠加大面积泛光。
- 统计卡在深色模式使用统一深色表面；分类色仅用于图标、数字、顶部细线或低亮度标识，不为卡片添加彩色外发光。
- 强品牌背景中的文字与图标使用 `text-on-brand`，由外观模式调整对比度。
- PC 侧栏折叠按钮与模拟时钟在深色模式使用深色表面、冷灰蓝边缘高光和深阴影；秒针可使用低饱和暖色，其他读数与指针遵从文字层级。
- 欢迎横幅必须通过独立夜蓝遮罩形成夜航环境，不直接降低整张亮色插画亮度；柯基和像素犬保留原始色彩，仅添加低强度深色投影。
- 全屏遮罩、半透明容器、媒体浮层和实色背景前景必须分别使用 `overlay-*`、`surface-translucent`、`surface-media-overlay`、`text-on-solid` 等语义 Token；禁止在端侧组件中新增黑色遮罩、固定白字或暖棕色弹层阴影。
- 欢迎区、统计卡、列表、预览、分页、空状态、设置图标和教程引导线的前景色必须复用文字、品牌或状态语义 Token；页面后置样式不得重新写入未受外观模式控制的深色文字或图标颜色。
- PC 与移动端的分类色卡必须由 `colorKey` 解析浅色或低亮度深色表面；运行时可通过 CSS 变量或当前外观参数传递颜色，但不得在暗色模式输出固定日间浅色背景。
- 移动欢迎区在深色模式通过独立遮罩保证文字可读，吉祥物取消 `multiply` 混合模式；标签与筛选项的高光、阴影均复用暗色拟态 Token。
- 详情页媒体区在深色模式使用夜蓝渐变与低亮星点建立“夜航媒体画廊”背景；封面、缩略图、计数器、翻页控件和全屏查看器统一使用媒体表面、边框、阴影与前景 Token，不复用亮色云朵、白色玻璃按钮或纯黑查看器背景。

## 五、状态与品牌规则

- 成功、信息、警告、危险分别使用 `--color-state-success-*`、`--color-state-info-*`、`--color-state-warning-*`、`--color-state-danger-*`。
- 品牌交互只使用 `--color-brand-*`，由 `data-workbench-theme` 控制。
- 危险、错误、删除和异常不得使用品牌主题色表达。
- 状态提示必须附带文字、图标或动作描述，不能只依赖颜色。

## 六、运行时与迁移

- `src/js/theme-config.js` 定义主题合法值、旧色主题键映射和默认值。
- `src/js/theme-service.js` 负责迁移旧 `pc-accent` / `accent`、应用根节点属性、监听系统外观与按绝对时间安排定时切换。
- 主题偏好保存于当前设备的 `localStorage`，不等同于跨设备同步数据。
- `src/js/folder-color.js` 已为首页与分类页提供 `colorKey` 优先、历史十六进制兼容、稳定哈希回退与深色低亮度表面映射；PC 与移动端不再按列表位置分配分类色。
- 分类管理、SQLite、PC API、备份与局域网同步的 `colorKey` 双写迁移仍未完成；在完成前不得删除既有 `color` 字段。

## 七、可访问性

- 正文、按钮、链接、焦点环和状态文字必须在浅色与深色模式下满足 WCAG AA 对比度要求。
- 启用减少动态效果时，主题切换不应保留颜色过渡。
- 每增加一个工作台主题，必须同时提供浅色和深色下的 `primary`、`hover`、`subtle`、`emphasis`、`focus`、`decoration` 映射。
