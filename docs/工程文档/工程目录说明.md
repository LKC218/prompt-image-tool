# 工程目录说明

本文说明项目主要目录职责，用于新增文件时快速判断放置位置。

## 根目录

根目录只放项目级配置、依赖清单和必要的一键入口脚本。

应保留：

- `package.json`
- `package-lock.json`
- `vite.config.js`
- `capacitor.config.ts`
- `requirements.txt`
- `.gitignore`
- 必要的启动和构建脚本，例如 `build.bat`、`dev.bat`

不建议长期放置：

- 历史计划文档
- 安装包产物
- 一次性工具脚本
- 临时调试文件

## `src`

前端源码目录，也是 Vite 的 `root`。

主要内容：

- `src/index.html`
- `src/css`
- `src/js`
- `src/assets`

新增前端页面、样式、交互模块和前端资源时，优先放在这里。

## `python`

Python 后端与本地数据目录。

主要内容：

- `python/main.py`
- `python/tests`
- `python/data`

新增后端接口、数据处理逻辑和 Python 测试时，优先放在这里。

## `src-tauri`

Tauri 桌面端工程目录。

新增桌面端权限、Rust 逻辑、Tauri 配置和桌面图标资源时，优先放在这里。

## `android`

Capacitor Android 工程目录。

仅在需要修改 Android 原生配置、权限、图标、启动屏或 Gradle 配置时编辑该目录。

## `docs`

项目文档总目录。

推荐分类：

- `docs/apps-code-map.md`：文件导航和文档更新落点。
- `docs/技术文档`：API、PC、移动端、Web 和局域网同步技术说明。
- `docs/设计文档`：UI/UX 设计系统和组件规范。
- `docs/工程文档`：工程交接、迁移和目录职责说明。
- `docs/UI计划`：PC 与移动端 UI 设计和页面计划。
- `docs/构建方案`：PC、Android、Tauri 等构建方案。
- `docs/计划文档`：版本计划、实施计划和功能规划。
- `docs/版本记录`：版本迭代历史与变更记录。
- `docs/已修复问题`：已定位并修复的问题复盘。
- `docs/对话历史`：重要对话记录。

## `UI设计稿`

原始设计稿、视觉参考、图标草稿和复刻资料。

该目录不承载可运行源码。进入实现阶段后的正式资源，应整理到 `src/assets`。

## `releases`

发布产物归档目录。

建议只保留需要人工分发或追溯的安装包。普通构建产物应由 `.gitignore` 忽略。

## `scripts`

专项维护脚本目录。

当前脚本：

- `scripts/compress_icon.py`
- `scripts/patch-java-version.ps1`

这些脚本通常有副作用，运行前应先检查工作区状态。

## 后续建议目录

以下目录建议后续整理时再创建：

- `packaging`：打包源码和安装器配置。

创建前应同步修改引用脚本和构建文档。
