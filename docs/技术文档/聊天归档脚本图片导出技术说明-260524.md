# 聊天归档脚本图片导出技术说明

## 适用范围

本文说明用户脚本 `gpt本地档案馆工具/ChatGPT Conversation Vault - CheeseTa Integrated-2026.05.24.10.user.js` 中“导出当前对话附带图片”的实现。该能力只挂载在 `导出当前` 链路，不改变保存当前、备份官方历史、导出本地库、导入文件、搜索和复制续聊等既有行为。

当前实现目标是：当当前 ChatGPT 对话页面已经渲染生成图片时，导出 ZIP 同步包含可下载图片文件、图片清单，并让导出的 Markdown / HTML 使用 ZIP 内相对路径引用图片。

## 入口链路

用户点击面板中的 `导出当前` 按钮后，事件从 `handlePanelClick()` 分发到 `exportCurrentZip()`。

核心调用顺序：

```text
handlePanelClick()
  -> exportCurrentZip()
    -> saveCurrentConversation('manual-export')
    -> collectCurrentConversationImageAssets(stored, 'current')
      -> discoverCurrentConversationImages()
      -> downloadImageCandidate()
      -> buildImageAssetFileName(stored, imageNo, extension)
    -> addConversationToZip(zip, stored, 'current', { imageAssets })
      -> conversationToMarkdown(stored, { imageAssets })
      -> conversationToHtml(stored, { imageAssets })
    -> writeImageAssetsToZip(zip, imageAssets)
    -> buildImageManifest(stored, imageAssets)
    -> downloadFile(zip.generateBlob(), filename)
```

`saveCurrentConversation()` 仍然负责读取当前对话、规范化文本内容并写入本地库。图片导出在保存完成后执行，图片下载失败不会回滚本地保存。

## 导出产物结构

无图片时，当前对话导出仍保持原有文本文件结构。

发现图片时，ZIP 中会增加 `current/assets/`：

```text
current/
  对话标题_对话ID_时间.json
  对话标题_对话ID_时间.md
  对话标题_对话ID_时间.html
  assets/
    对话标题-图片-001.png
    对话标题-图片-002.webp
    image-manifest.json
manifest.json
```

`manifest.json` 继续描述对话导出主体。`current/assets/image-manifest.json` 专门描述图片导出结果。

## 图片资源数据结构

图片导出使用统一的 `imageAssets` 数组在各函数之间传递。单个资源字段如下：

```js
{
  id: 'image-001',
  source: 'dom-image',
  url: 'https://...',
  alt: '',
  width: 1024,
  height: 1024,
  status: 'downloaded',
  localPath: 'current/assets/对话标题-图片-001.png',
  relativePath: 'assets/对话标题-图片-001.png',
  mimeType: 'image/png',
  extension: 'png',
  size: 123456,
  error: null,
  bytes: Uint8Array
}
```

字段约定：

- `id`：导出内稳定编号，按发现顺序生成。
- `source`：发现来源，当前包括 `dom-image` 和 `dom-background`。
- `url`：原始图片地址，可为 `https:`、`blob:` 或 `data:image/...`。
- `status`：`downloaded` 表示已写入 ZIP；`failed` 表示发现但下载失败。
- `localPath`：写入 ZIP 时使用的完整路径。
- `relativePath`：Markdown / HTML 中引用的相对路径。
- `bytes`：仅在内存中用于写入 ZIP，不写入 `image-manifest.json`。

## 图片文件命名

图片文件名由 `buildImageAssetFileName(stored, imageNo, extension)` 统一生成，格式为：

```text
{安全化对话名称}-图片-{三位序号}.{扩展名}
```

命名约定：

- 对话名称来自 `stored.title`，为空时回退到 `Untitled Conversation`。
- 标题先经过 `safeFileName()` 清理 Windows 非法字符，再裁剪为最多 72 个字符，避免 ZIP 内路径过长。
- 三位序号继续使用图片发现顺序，例如 `001`、`002`、`003`，避免同一对话多图互相覆盖。
- 扩展名仍由 `imageExtension()` 推断，优先 MIME，其次 URL 扩展名，最后回退 `png`。
- `asset.id` 保持 `image-001`、`image-002` 等内部稳定编号，只改变面向用户的 ZIP 文件名和相对引用。

## 图片发现策略

`discoverCurrentConversationImages()` 从当前页面 DOM 扫描图片候选。

当前扫描范围：

- 优先使用 `document.querySelector('main')`。
- 找不到 `main` 时回退到 `document.body`。

候选来源：

- `img.currentSrc`
- `img.src`
- `img.srcset` 中分辨率最高的候选
- `picture source[srcset]`
- 行内或计算样式中的 `background-image`

过滤策略由 `isLikelyExportableImage()` 控制：

- 排除 URL 或元素属性中含 `avatar`、`favicon`、`logo`、`sprite`、`icon`、`profile`、`emoji`、`glyph` 的资源。
- 排除 `.svg`。
- 若能读取尺寸，则排除宽或高小于 `96px`、面积小于 `12000` 的图片。
- 排除渲染尺寸接近 0 的隐藏资源。

图片数量上限由 `APP.currentExportImageLimit` 控制，当前为 `30`。

## 图片下载策略

`downloadImageCandidate()` 使用页面原生 `fetch(candidate.url, { credentials: 'include' })` 下载图片。

处理流程：

1. 请求失败或非 `2xx` 状态，记录失败。
2. 响应转为 `Blob`。
3. `Blob` 转为 `Uint8Array`。
4. 优先使用响应 `blob.type` 判断 MIME。
5. MIME 缺失时，根据 URL 扩展名推断。
6. MIME 不是 `image/*` 时，用 `looksLikeImageBytes()` 检查 PNG、JPEG、GIF、WEBP 文件头。

失败不会抛出到 `exportCurrentZip()` 外层，而是写入单个图片资源的 `error` 字段。这样导出当前对话不会因为某张图无法下载而整体失败。

## Markdown 和 HTML 引用

`addConversationToZip()` 新增可选参数 `options`：

```js
addConversationToZip(zip, stored, folder, { imageAssets })
```

`conversationToMarkdown(item, options = {})` 在文本对话和文件元数据之后追加 `Images` 区块：

```md
## Images

![Generated image 1](assets/对话标题-图片-001.png)
```

`conversationToHtml(item, options = {})` 使用 `buildImageGalleryHtml(options)` 追加图片画廊：

```html
<section class="assets">
  <h2>Images</h2>
  <div class="asset-grid">...</div>
</section>
```

只有 `status === 'downloaded'` 且存在 `relativePath` 的图片会写入 Markdown / HTML。失败图片只出现在 `image-manifest.json`。

## 图片清单

`buildImageManifest()` 生成 `current/assets/image-manifest.json`，用于追踪图片导出结果。

清单包含：

- `schemaVersion`
- `conversationId`
- `title`
- `exportedAt`
- `counts.total`
- `counts.downloaded`
- `counts.failed`
- `images[]`

`images[]` 不包含二进制 `bytes`，只记录元数据、路径、状态和失败原因。

## prompt-image-tool 专用 JSON

除 `导出当前` ZIP 外，用户脚本还新增 `导出提示词JSON` 入口，用于生成 `prompt-image-tool.import.v1` 专用文件。

导出内容要点：

- 文件名采用 `{对话名称}-prompt-image-tool.json`。
- 顶层 `schema` 用于导入端唯一识别。
- `prompt.title` 默认取对话标题。
- `prompt.positivePrompt` 默认取对话中第一条非空消息的自然语言正文；若开头是 fenced code block、类 JSON 块或 `content_type / asset_pointer / metadata` 等结构化元数据块，则跳过噪声段，并保留后续完整多段正文。空行只作为段落分隔，不作为截断点。
- `prompt.negativePrompt` 仅在显式字段存在时填入，否则留空。
- 图片通过 Data URL 内嵌，且保留 `fileName`、`mimeType`、`width`、`height` 和来源信息。
- 当前页面最多先发现 30 张候选图片，`prompt-image-tool` 专用 JSON 默认内嵌前 10 张可下载且未超限的图片。
- 当图片下载失败、单张超过 4MB 或超过内嵌数量上限时，会写入 `skippedImages` 供导入端参考。

## MIME 和扩展名推断

相关函数：

- `imageExtension(mimeType, url)`
- `imageExtensionFromUrl(url)`
- `inferImageMimeFromUrl(url)`
- `looksLikeImageBytes(bytes)`

扩展名优先级：

1. 响应 MIME。
2. URL 路径扩展名。
3. 默认 `png`。

文件头检查用于降低“URL 像图片但响应不是图片”的误判风险。当前识别 PNG、JPEG、GIF 和 WEBP。

## 兼容边界

当前实现没有新增 `GM_xmlhttpRequest` 权限，也没有新增 `@connect` 配置，因此图片下载依赖页面原生 `fetch` 能力。

已支持：

- 当前页面已渲染的普通图片。
- 当前页面生命周期内仍有效的 `blob:` 图片。
- 同源或允许当前页面请求的跨域图片。
- 无图片对话导出。
- 单张或多张图片下载失败时继续导出。

暂不保证：

- 官方历史批量备份中的图片下载。
- 已离开页面后恢复 `blob:` 图片。
- 被 CDN、CORS 或登录权限拒绝的图片。
- 未渲染到当前 DOM 的图片资源。
- 用户上传参考图和模型生成图的精确分类。

## 验证建议

终端侧：

```powershell
node -e "const fs=require('fs'); const vm=require('vm'); const code=fs.readFileSync('ChatGPT Conversation Vault - CheeseTa Integrated-2026.05.24.10.user.js','utf8'); new vm.Script(code,{filename:'ChatGPT Conversation Vault - CheeseTa Integrated-2026.05.24.10.user.js'}); console.log('syntax ok');"
```

浏览器侧：

1. 安装或更新用户脚本。
2. 打开一条无图片对话，执行 `导出当前`，确认 ZIP 可正常生成。
3. 打开一条含已生成图片的对话，执行 `导出当前`。
4. 解压 ZIP，检查 `current/assets/image-manifest.json`。
5. 检查 `current/assets/` 是否有图片文件。
6. 打开导出的 HTML，确认图片画廊能离线显示。
7. 断网或拦截图片请求后再次导出，确认失败记录进入清单且 ZIP 仍生成。

## 后续演进

可选增强方向：

- 增加 `GM_xmlhttpRequest` 和 `@connect`，增强跨域图片下载能力。
- 结合 ChatGPT 原始对话 `raw.mapping` 解析图片线索，补充 DOM 扫描不到的资源。
- 在 `image-manifest.json` 中区分用户上传参考图和模型生成图。
- 将图片导出能力扩展到官方历史备份，但需要控制请求量、限流和失败重试。
