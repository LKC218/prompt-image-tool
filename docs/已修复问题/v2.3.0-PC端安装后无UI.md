## 问题一：端口冲突导致程序崩溃

### 现象

安装后运行 `PromptImageManager.exe`，程序闪退或无任何界面显示。

### 根因

`build/app_main.py` 中 HTTP 服务器硬编码端口 `8888`：

```python
port = 8888
socketserver.ThreadingTCPServer.allow_reuse_address = True
httpd = socketserver.ThreadingTCPServer(("", port), AppHandler)
```

当端口 8888 被其他进程占用时（如之前未正常退出的实例、其他服务等），`ThreadingTCPServer` 构造函数直接抛出 `OSError` 异常，程序崩溃退出，没有任何 UI 提示。

### 修复方式

添加端口冲突处理逻辑：优先使用 8888 端口，若被占用则自动调用 `find_free_port()` 寻找可用端口。若连续 10 次尝试均失败，弹出 tkinter 错误提示框后退出。

```python
port = 8888
for attempt in range(10):
    try:
        socketserver.ThreadingTCPServer.allow_reuse_address = True
        httpd = socketserver.ThreadingTCPServer(("", port), AppHandler)
        break
    except OSError:
        port = find_free_port()
else:
    import tkinter as tk
    from tkinter import messagebox
    root = tk.Tk()
    root.withdraw()
    messagebox.showerror("启动失败", "无法找到可用端口，请检查网络设置后重试")
    return
```

同时将实际使用的端口号存入全局变量 `SERVER_PORT`，供 `handle_network_info` 接口返回正确端口。

### 影响文件

- `build/app_main.py`

## 问题二：pywebview 异常未处理

### 现象

在缺少 .NET Framework 的 Windows 系统上，程序启动后无界面显示。

### 根因

`pywebview` 依赖 .NET Framework（Windows 上使用 WinForms 后端）。当系统缺少 .NET Framework 或 pywebview 初始化失败时，`webview.start()` 抛出异常，程序直接崩溃，没有 fallback 机制。

原代码：

```python
if HAS_WEBVIEW:
    window = webview.create_window(...)
    webview.start()
    httpd.shutdown()
```

### 修复方式

用 `try/except` 包裹 pywebview 调用，失败时 fallback 到系统默认浏览器：

```python
if HAS_WEBVIEW:
    try:
        window = webview.create_window(...)
        webview.start()
        httpd.shutdown()
    except Exception:
        import webbrowser
        webbrowser.open(url)
        try:
            httpd.serve_forever()
        except KeyboardInterrupt:
            pass
```

### 影响文件

- `build/app_main.py`

## 问题五：安装包和软件乱码

### 现象

安装包界面、快捷方式名称、注册表卸载项中的中文全部显示为乱码。

### 根因

`build/installer.nsi` 文件以 **UTF-8 without BOM** 编码保存。虽然脚本中声明了 `Unicode true`，但 NSIS 要求含中文的 `.nsi` 文件必须以 **UTF-8 with BOM**（`\xef\xbb\xbf`）编码保存。缺少 BOM 时，NSIS 将文件内容按 ANSI 编码解析，导致中文字符变成乱码。

验证方式：

```python
with open('installer.nsi', 'rb') as f:
    raw = f.read()
    print(raw[:3] == b'\xef\xbb\xbf')  # False → 缺少 BOM
```

### 修复方式

将 `installer.nsi` 重新保存为 UTF-8 with BOM 编码：

```python
with open('installer.nsi', 'r', encoding='utf-8') as f:
    content = f.read()
with open('installer.nsi', 'w', encoding='utf-8-sig') as f:
    f.write(content)
```

### 影响文件

- `build/installer.nsi`

## 问题六：PC 端软件图标消失

### 现象

安装后 `PromptImageManager.exe` 的图标在资源管理器中不显示，快捷方式图标也消失。

### 根因

`build/app.spec` 中 PyInstaller 的 EXE 和 COLLECT 配置启用了 `upx=True`。UPX 压缩会破坏 Windows PE 可执行文件中的图标资源段（RT_ICON/RT_GROUP_ICON），导致 exe 图标丢失。

```python
# 原配置
upx=True  # UPX 压缩会破坏图标资源
```

### 修复方式

禁用 UPX 压缩：

```python
exe = EXE(
    ...
    upx=False,  # 禁用 UPX，保护图标资源
    ...
)

coll = COLLECT(
    ...
    upx=False,  # 禁用 UPX
    ...
)
```

> 注：禁用 UPX 后 exe 体积会略微增大（约 0.5MB），但图标显示正常。

### 影响文件

- `build/app.spec`

## 问题七：软件内图标和过渡屏图标消失

### 现象

安装后软件内所有 SVG 图标（包括 Header Logo、按钮图标、空状态图标等）均不显示，过渡屏（Splash Screen）的应用图标也不显示。

### 根因

`图标.svg` 文件名包含中文字符，Vite 构建后输出为 `图标-Cj81MuNF.svg`，在 `dist/index.html` 中引用路径被 URL 编码为 `/assets/%E5%9B%BE%E6%A0%87-Cj81MuNF.svg`。

`build/app_main.py` 的 `serve_frontend_file` 方法直接使用 URL 编码后的路径拼接文件系统路径：

```python
def serve_frontend_file(self, path):
    file_path = os.path.join(FRONTEND_DIR, path.lstrip('/'))
    # path = '/assets/%E5%9B%BE%E6%A0%87-Cj81MuNF.svg'
    # file_path = '.../frontend/assets/%E5%9B%BE%E6%A0%87-Cj81MuNF.svg'  ← 不存在！
```

`os.path.exists()` 返回 False（因为实际文件名是 `图标-Cj81MuNF.svg`），走了 fallback 返回 `index.html`，浏览器将 HTML 当作 SVG 解析失败，图标不显示。

验证：请求 `/assets/%E5%9B%BE%E6%A0%87-Cj81MuNF.svg` 返回 `Content-Type: text/html`（错误），修复后返回 `Content-Type: image/svg+xml`（正确）。

### 修复方式

在 `serve_frontend_file`、`serve_image`、`handle_sync_image` 三个方法中添加 URL 解码：

```python
import urllib.parse

def serve_frontend_file(self, path):
    decoded_path = urllib.parse.unquote(path.lstrip('/'))
    file_path = os.path.join(FRONTEND_DIR, decoded_path)
    # decoded_path = 'assets/图标-Cj81MuNF.svg'  ← 正确匹配文件名

def serve_image(self, relative_path):
    decoded_path = urllib.parse.unquote(relative_path)
    filename = decoded_path.replace('images/', '', 1)

def handle_sync_image(self, filename):
    decoded_filename = urllib.parse.unquote(filename)
    filepath = os.path.join(IMAGES_DIR, decoded_filename)
```

### 影响文件

- `build/app_main.py`

## 问题八：缺少 favicon 和 pywebview 窗口图标

### 现象

1. 浏览器标签页显示默认空白图标，无 favicon
2. pywebview 窗口标题栏/任务栏图标依赖 EXE 内嵌图标，未通过 `icon` 参数显式设置

### 修复方式

1. 在 `src/index.html` 的 `<head>` 中添加 favicon 引用：

```html
<link rel="icon" type="image/svg+xml" href="./assets/icons/图标.svg">
```

2. 在 `build/app_main.py` 的 `webview.create_window` 中添加 `icon` 参数：

```python
icon_path = os.path.join(
    os.path.dirname(sys.executable) if getattr(sys, 'frozen', False)
    else os.path.dirname(os.path.abspath(__file__)),
    'icon.ico'
)
window = webview.create_window(
    ...
    icon=icon_path if os.path.exists(icon_path) else None,
)
```

### 影响文件

- `src/index.html`
- `build/app_main.py`
