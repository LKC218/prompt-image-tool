## 二、架构总览

```
              ┌─────────────────────────────────────┐
              │       共享前端 (Vite + 原生 JS)       │
              │    src/index.html + js + css         │
              └──────────────┬──────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              │        storage.js 抽象层      │
              │  isTauri? isCapacitor?       │
              └──────┬───────────────┬───────┘
                     │               │
        ┌────────────▼──────┐ ┌─────▼──────────────┐
        │   PC 桌面端        │ │   Android 移动端     │
        │   Tauri + Rust    │ │   Capacitor         │
        │   → Python API    │ │   → SQLite 本地      │
        │   ApiStorage      │ │   SqliteStorage      │
        └───────────────────┘ └─────────────────────┘
```

**平台检测逻辑**（`src/js/storage.js`）：

- `isCapacitor === true`（Android WebView）→ 动态加载 `SqliteStorage`，数据存本地 SQLite
- 其他情况（PC Tauri / 浏览器）→ 动态加载 `ApiStorage`，通过 HTTP 连 Python 后端

**PC 端数据流**：前端 → ApiStorage (HTTP Fetch) → Python 后端 (端口 8888) → JSON 文件 + 图片文件

**Android 端数据流**：前端 → SqliteStorage → SQLite 数据库 + Filesystem 插件

| **Git** | 任意 | 版本管理（可选） | https://git-scm.com/ | `git --version` |

### 3.2 PC 桌面端开发额外需要

| 工具 | 用途 | 安装方式 | 验证命令 |
|------|------|---------|---------|
| **Rust** | Tauri 编译依赖 | https://rustup.rs/ | `cargo --version` |

### 3.3 Android 移动端开发额外需要

| 工具 | 最低版本 | 用途 | 安装方式 | 验证命令 |

## 八、构建指南

### 8.1 PC 桌面端 — 方式 A：Tauri（推荐）

```bash
npm install
npx tauri build
# 产物：src-tauri/target/release/bundle/nsis/生图提示词管理器_2.0.0_x64-setup.exe
```

**前提**：需要安装 Rust（https://rustup.rs/）

### 8.2 PC 桌面端 — 方式 B：PyInstaller

```bash
pip install pyinstaller pywebview
npm install
npx vite build
python -m PyInstaller build/app.spec --workpath build/build --distpath build/dist --clean -y
# 产物：build/dist/PromptImageManager/PromptImageManager.exe

# 可选：制作 NSIS 安装包
cd build && makensis installer.nsi && cd ..
# 产物：build/PromptImageManager-Setup-2.0.0.exe
```

### 8.3 Android 移动端

```bash
# 1. 构建前端
npx vite build

# 2. 同步到 Capacitor
npx cap sync android

# 3. 修补 Java 版本（⚠️ 不可跳过！）
powershell -ExecutionPolicy Bypass -File patch-java-version.ps1

# 4. 构建 Release APK
cd android
.\gradlew assembleRelease
cd ..
# 产物：android/app/build/outputs/apk/release/app-release.apk

# 5. 复制并重命名
copy android\app\build\outputs\apk\release\app-release.apk PromptImageManager-v2.2.1.apk
```

**Debug 构建**：将第 4 步的 `assembleRelease` 改为 `assembleDebug`，无需签名配置。

### 8.4 仅构建前端

```bash
npx vite build
# 产物：dist/ 目录
```

**folders.json**：

```json
[{
  "id": "folder001",
```

### 10.2 Android 端（SQLite 数据库）

| 表名 | 字段 |
|------|------|
| folders | id, name, color, sort_order, created_at, updated_at |
| prompt_sets | id, name, folder_id, created_at, updated_at |
| versions | id, prompt_set_id(FK), version, prompt, negative_prompt, note, sort_order, created_at |
| images | id, version_id(FK), name, path, file, note, created_at |

外键关系：`folders ← prompt_sets ← versions ← images`，ON DELETE CASCADE

## 十一、API 接口速查

Python 后端运行在 `0.0.0.0:8888`，所有接口支持 CORS。

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | /api/health | 健康检查 |
| GET | /api/folders | 获取所有文件夹 |
| POST | /api/folders | 创建文件夹 |
| POST | /api/folder/{id} | 更新文件夹 |
| DELETE | /api/folder/{id} | 删除文件夹 |
| GET | /api/prompt-sets | 获取集合列表摘要 |
| GET | /api/prompt-set/{id} | 获取集合详情 |
| POST | /api/prompt-sets | 创建集合 |

## 十二、局域网同步

- **架构**：PC 端作为服务端，Android 端作为客户端
- **前提**：PC 端软件已启动，两端在同一局域网，PC 防火墙允许 8888 端口
- **流程**：输入 PC IP → 连接测试 → 拉取数据 → 冲突检测 → 写入 SQLite → 下载图片 → 校验
- **冲突策略**：仅 PC 有→同步；仅 Android 有→保留；两端都改过→保留两个副本

## 十三、已知问题与注意事项

| 编号 | 问题 | 严重程度 | 说明 |
|------|------|---------|------|
| PC-001 | Python 后端 load_data/save_data 非原子操作 | 中 | 高并发下可能数据丢失 |
| PC-002 | Python 后端日志被静默 | 中 | 生产环境无法排查问题 |
| PC-005 | Python 进程随 Tauri 退出后不会自动清理 | 中 | 可能残留进程 |
| AND-001 | deleteImage() 只删文件不清理数据库记录 | 高 | 数据不一致 |
| AND-002 | 导入数据不覆盖已有集合，与 PC 端行为不一致 | 中 | 导入策略差异 |
| AND-005 | SQLite 未启用 WAL 模式 | 低 | 并发写入性能受限 |
| WEB-005 | @capacitor/camera 和 @capacitor/network 已安装未使用 | 低 | 增加包体积 |
