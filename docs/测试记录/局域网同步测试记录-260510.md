## 问题修复

### Mixed Content 拦截

早期 Android 应用内连接测试被 WebView 拦截：

```text
Mixed Content: The page at 'https://localhost/' was loaded over HTTPS, but requested an insecure resource 'http://192.168.6.109:8888/api/health'.
```

已修复：

- `capacitor.config.ts`：启用 `android.allowMixedContent=true`。
- `android/app/src/main/AndroidManifest.xml`：启用 `android:usesCleartextTraffic="true"`。

复测结果：Android 应用内请求可继续执行，连接、拉取、回传和双向同步均通过。

### 同内容回传误判冲突

现象：Android 从 PC 拉取后再回传，PC 会把同内容数据误判为冲突，生成重复的 `Android冲突副本`。

原因：冲突判断比较了端侧生成型字段，例如记录 ID、版本 ID、图片 ID、时间字段、图片 Data URL、图片体积和 MIME 类型；这些字段在两端可能不同，但不代表提示词内容不同。

已修复：

- `python/main.py`：`normalize_prompt_for_compare()` 比较时忽略生成型 ID、时间、图片 `data`、大小、MIME 类型，并兼容 `camelCase` 与 `snake_case` 字段。
- `python/tests/test_main.py`：新增 `test_sync_import_keep_pc_skips_same_content_with_generated_ids`，覆盖同内容不同生成型字段时应跳过而非冲突。

复测结果：修复后回传报告为新增 1 条、冲突副本 0 条、跳过 3 条。
