## 发现与处理

1. Android 原生返回键监听存在异步注册风险。  
   - 现象：`mobile-app.js` 在 `@capacitor/app` 异步导入完成前调用 `setupBackButton()` 时会直接返回，可能导致真机硬件返回键监听未注册。
   - 处理：在 Capacitor `App` 模块导入完成后再次调用 `setupBackButton()`，并增加 `backButtonRegistered` 防重入标记。
   - 复测：移动端关键 JS 语法检查、移动端页面级回归、前端全量测试、Vite 构建、Capacitor 同步和 Android Debug 构建均通过。

2. 移动端缺少页面级全功能回归测试入口。  
   - 处理：新增 `src/js/mobile-regression.test.js`，覆盖首页、提示词库、详情、编辑、分类与标签、设置页和局域网连接入口。
   - 复测：`npm.cmd run test -- mobile-regression.test.js` 通过，随后全量 `npm.cmd run test` 通过。

3. 移动端详情页图片预览返回上一页后遮罩残留。  
   - 现象：详情页打开图片预览后，如果不先关闭预览而直接返回上一页，`.m-image-viewer-overlay` 仍停留在界面上。
   - 原因：图片预览遮罩追加到 `#mobileApp` 或 `body`，不属于详情页 `pageEl`，路由卸载时不会自动删除。
   - 处理：`mobile-detail.js` 新增活动图片预览遮罩引用和统一关闭函数，在打开新预览前、详情页返回、提示词不存在自动返回和 `unmount()` 时清理遮罩。
   - 复测：`node --check src/js/mobile-detail.js` 通过，`npm.cmd run test -- mobile-regression.test.js` 通过，新增用例覆盖图片预览返回与卸载清理；`npm.cmd run test` 和 `npm.cmd run build` 均通过。
