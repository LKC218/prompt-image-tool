# 局域网同步测试验证计划

> 日期：2026-05-10  
> 目标：验证 Android 手机与 PC 端在同一局域网下是否可以完成搜索、连接、数据同步、图片同步和结果校验。

## 思考与分析

### Skill 嗅探结果

- 本任务属于测试验证计划制定，启用 `planning-with-files` 技能。
- 仓库未发现项目级 `SKILL.md`，本计划遵循项目 `AGENTS.md` 中的 Step-by-Step 工作流。
- 当前只制定计划，不执行测试、不修改功能代码，待确认后再进入验证实施。

### 已知工程依据

- PC 端 Python 后端监听 `8888` 端口，接口文档要求监听 `0.0.0.0`，支持局域网访问。
- 移动端通过 `src/js/lan-sync.js` 实现 `LanScanner` 与 `LanSync`：
  - 搜索阶段请求 `http://{ip}:8888/api/health`。
  - 同步阶段依次请求 `/api/health`、`/api/sync`、`/api/sync/images/{filename}`。
  - 同步数据写入 Android SQLite，图片写入 Capacitor Filesystem。
- 移动端设置页提供「搜索 PC」「测试连接」「同步」三个入口。
- 当前手机已通过 USB 调试连接，并已安装调试包：`com.promptimagemanager.app`，版本 `2.3.1`。

## 实施计划

### 阶段一：测试前置检查

1. 确认 PC 与手机连接同一个局域网，优先使用同一个 Wi-Fi。
2. 确认 PC 端应用或 Python 后端已启动。
3. 确认 PC 本机能访问：
   - `http://127.0.0.1:8888/api/health`
   - `http://{PC局域网IP}:8888/api/health`
4. 确认 Windows 防火墙允许 `8888` 端口入站。
5. 记录 PC 局域网 IP、手机 IP、测试时间、应用版本。

### 阶段二：PC 服务端接口验证

1. 验证 `/api/health` 返回 `status: ok` 和 `device_name`。
2. 验证 `/api/network-info` 返回 PC 局域网 IP 和端口。
3. 验证 `/api/sync` 返回完整同步数据：
   - `folders`
   - `prompt_sets`
   - `versions`
   - `images`
   - `sync_meta`
4. 若存在图片，抽样验证 `/api/sync/images/{filename}` 能返回二进制图片。

### 阶段三：移动端搜索与连接验证

1. 打开 Android 应用进入「设置」页。
2. 点击「搜索 PC」，观察是否能自动发现 PC。
3. 若搜索成功：
   - 检查设备名称、IP、端口显示是否正确。
   - 点击发现结果后确认 IP 自动填入输入框。
4. 若搜索失败：
   - 手动输入 PC 设置页显示的 IP。
   - 记录失败表现，用于判断是 WebRTC 获取网段失败、扫描超时，还是防火墙问题。
5. 点击「测试连接」，确认连接成功提示。

### 阶段四：移动端同步验证

1. 在 PC 端准备一组可识别测试数据：
   - 至少 1 个分类。
   - 至少 2 条提示词。
   - 至少 1 条提示词包含多个版本。
   - 至少 1 张图片。
2. 在手机端执行「同步」。
3. 观察同步状态是否按预期变化：
   - 连接中
   - 拉取数据
   - 合并数据
   - 下载图片
   - 校验数据
   - 成功或部分成功
4. 同步完成后，在手机端逐页检查：
   - 首页统计数量。
   - 提示词库列表。
   - 分类与标签。
   - 提示词详情页版本内容。
   - 图片是否正常显示。

### 阶段五：数据一致性验证

1. 对比 PC 端与手机端的数据数量：
   - 分类数量。
   - 提示词数量。
   - 版本数量。
   - 图片数量。
2. 对比关键字段：
   - 标题。
   - 分类归属。
   - 正向提示词。
   - 反向提示词。
   - 备注。
   - 图片文件。
3. 验证覆盖策略：
   - PC 与手机存在同 ID 数据时，以 PC 数据覆盖手机数据。
   - 手机端独有数据应保留。

### 阶段六：异常场景验证

1. PC 端未启动时，手机「测试连接」应失败并给出明确提示。
2. 手机与 PC 不在同一网络时，搜索与连接应失败。
3. 防火墙阻断 `8888` 端口时，搜索与连接应失败。
4. 同步中断网或关闭 PC 服务时，应进入错误或部分成功状态，不应卡死。
5. 图片下载失败时，数据应尽量同步完成，并在报告中体现失败图片。

## 当前执行结论

2026-05-10 首轮验证结论：部分可用。PC 服务端接口和手机系统网络均可用，但 Android 应用 WebView 以 `https://localhost/` 加载页面，请求 `http://192.168.6.109:8888/api/health` 时被 Mixed Content 策略拦截，导致应用内「搜索 PC」和「测试连接」失败。

2026-05-10 修复后复测结论：通过。补充 `android.allowMixedContent=true` 与 `android:usesCleartextTraffic="true"` 后，手动输入 `192.168.6.109` 可连接 PC 并完成同步；同步报告显示新增 5 条、图片 4 张，首页显示提示词总数 5、分类 2。

测试记录见：`docs/测试记录/局域网同步测试记录-260510.md`。

## 验收标准

- 手机可以通过搜索或手动 IP 连接到 PC。
- `/api/health`、`/api/network-info`、`/api/sync` 和图片接口均可在局域网访问。
- 手机同步后能看到 PC 端准备的分类、提示词、版本和图片。
- 同步过程有可理解的状态反馈，不出现无限加载或按钮卡死。
- 数据数量和关键字段与 PC 端一致，符合 PC 优先覆盖策略。
- 异常场景能给出失败提示，且不会破坏手机端已有数据。

## 风险与观察点

- Windows 防火墙可能拦截 `8888` 端口，导致手机搜索不到 PC。

## 动态端口补充验证

PC 安装包端口占用时可能回退到非 `8888` 端口。移动端搜索与手动输入需要按 `IP:port` 连接，验证时增加以下场景：

- 占用 `8888` 后启动 PC 安装包，确认 `/api/health`、`/api/sync/capabilities`、`/api/network-info` 返回实际端口。
- Android 搜索 PC 时确认能发现 `8888-8897` 范围内的回退端口。
- Android 手动输入 `PC_IP:实际端口` 后执行连接测试、拉取、回传和双向同步。
- 图片下载地址必须使用同一个实际端口。
- 手机浏览器/WebView 获取本机局域网 IP 可能失败，此时需要依赖常见网段扫描或手动输入 IP。
- 若 PC 数据中图片文件缺失，数据同步可能成功但图片下载失败。
- 若 Android SQLite 表结构迁移未完成，可能出现写入失败，需要通过 `adb logcat` 定位。
- 当前同步是 PC 优先覆盖，不是双向合并，测试前应避免把手机端重要独有数据误判为同步失败。
