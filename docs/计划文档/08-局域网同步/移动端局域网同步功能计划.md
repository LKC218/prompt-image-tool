## 二、各任务详细分析与实施方案

### R1: 在移动端设置页面添加局域网同步区域

**当前状态**：设置页面目前有外观设置、本地存储、数据备份与恢复等区域，缺少局域网同步相关内容。

**现有功能基础**：
- `src/js/lan-sync.js` 已实现 `LanScanner` 类和 `LanSync` 类 ✅
- 后端 `/api/health`、`/api/sync`、`/api/network-info` 接口已实现 ✅

**实施方案**：

在设置页面的"数据备份与恢复"区域下方新增"局域网同步"区域，包含：
1. 同步按钮（大按钮，视觉突出）
2. 状态显示区域
3. 扫描/手动输入切换选项

**涉及文件**：
- `src/js/mobile-settings.js` — 设置页面渲染和事件
- `src/js/mobile-lan-sync.js` — 新创建，同步相关UI和逻辑

---

### R2: 实现自动扫描局域网内可用PC端功能

**当前状态**：`lan-sync.js` 中的 `LanScanner` 类已实现 `scan()` 方法，能够扫描局域网内的设备。

**实施方案**：

1. 在同步区域添加"自动扫描"按钮
2. 点击扫描时显示扫描进度动画
3. 扫描到设备后以列表形式展示，支持点击选择
4. 显示设备名称（`device_name`）和IP地址
5. 扫描超时或无结果时给出提示

**涉及文件**：
- `src/js/mobile-lan-sync.js` — 扫描UI和逻辑
- `src/js/lan-sync.js` — 复用现有 `LanScanner` 类

---

### R3: 实现手动输入IP地址功能

**实施方案**：

1. 提供"手动输入"切换选项
2. 显示IP地址输入框（格式验证：`xxx.xxx.xxx.xxx`）
3. 添加"连接测试"按钮，测试输入的IP是否可用
4. 测试成功后显示设备信息，失败则提示错误

**涉及文件**：
- `src/js/mobile-lan-sync.js` — 手动输入UI和连接测试逻辑

---

### R4: 实现同步进度显示和结果反馈

**当前状态**：`LanSync` 类已实现完整的同步逻辑和状态管理。

**实施方案**：

1. 点击"开始同步"后禁用按钮，防止重复点击
2. 显示当前同步阶段（连接中 → 拉取数据 → 合并数据 → 下载图片 → 校验中）
3. 下载图片时显示进度条（当前/总数）
4. 同步完成后显示详细报告：
   - 新增/更新/冲突数量
   - 下载图片数量
   - 失败项（如有）
5. 同步失败时显示错误信息和重试选项

**涉及文件**：
- `src/js/mobile-lan-sync.js` — 同步进度UI和报告展示
- `src/js/lan-sync.js` — 复用现有 `LanSync` 类

---

### R5: 保存最近使用的设备列表

**实施方案**：

1. 使用 `localStorage` 保存最近使用的3-5个设备
2. 在同步区域顶部展示"最近同步"列表
3. 支持快速选择并连接
4. 记录最后同步时间

**涉及文件**：
- `src/js/mobile-lan-sync.js` — 设备列表保存和读取

---

## 三、执行顺序

```
R1（添加同步区域）→ R2（自动扫描）→ R3（手动输入）→ R4（同步进度）→ R5（保存设备）
```

理由：
- R1 是基础UI框架，需要先搭建
- R2 和 R3 是两种连接方式，先实现自动扫描（更常用）
- R4 是核心同步流程，依赖连接功能
- R5 是体验优化，最后实现
