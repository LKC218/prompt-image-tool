# 双端设置页图片下载记录与移动端相册保存计划-260524

> 版本：v1.0.0  
> 创建日期：2026-05-24  
> 状态：已实施，待 Android 真机相册验收  
> 范围：PC 端和移动端设置页新增图片下载记录列表与一键清除历史；移动端补齐图片下载后保存到手机相册的原生链路、回退策略和验证闭环。

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 使用 `planning-with-files` 技能：本次先沉淀计划文件，再进入实现。
- 现有仓库已经有稳定的设置页结构：
  - PC：`src/js/pc-settings.js`
  - 移动端：`src/js/mobile-settings.js`
- 现有图片下载共享入口已经收敛：
  - `src/js/image-download-utils.js`
  - `src/js/pc-utils.js`
  - `src/js/mobile-detail.js`
- 当前移动端设置页已经有“最近设备”记录能力，说明本仓库可以接受“设置页内维护一份本地历史列表”的交互模型；本次只需要把这个模式扩展到图片下载历史，不必重新发明一套信息架构。

### 1.2 问题初判

用户提出的两个需求其实是同一条链路的两端：

1. 设置页需要一个“我刚刚下载过什么”的可见历史，方便回看和清理。
2. 移动端下载当前图片后，必须真的落到手机相册，而不是只停留在应用可见目录或浏览器下载行为。

因此本次不应把它拆成两个孤立功能，而要按“下载行为采集 + 历史展示 + 平台保存落点”三层来做。

### 1.3 当前代码定位

- `src/js/image-download-utils.js` 目前只负责取图、命名、下载、保存，不负责持久化记录。
- `src/js/pc-settings.js` 目前只有外观、本地存储、备份恢复和局域网互通，没有下载历史区域。
- `src/js/mobile-settings.js` 已经有局域网最近设备列表，可复用“记录列表 + 清空”的交互方式。
- Android 端当前依赖 `@capacitor/filesystem` 管理文件，但“保存到相册”不是现有默认落点，必须单独补原生保存能力或等价桥接。

## 二、实施计划

### Phase 1：统一下载历史数据模型

1. 定义图片下载记录的最小字段：
   - 唯一标识
   - 图片名称或文件名
   - 来源页面或来源类型
   - 保存时间
   - 保存结果
   - 平台标识（PC / 移动端）
2. 明确历史范围：
   - 只记录“成功保存”的图片下载，失败和取消先不进入历史列表。
   - 历史默认只保留最近一段数量，避免设置页膨胀。
3. 抽出统一的记录读写入口，供 PC 和移动端共用调用点。

### Phase 2：PC 端设置页增加下载历史

1. 在 `src/js/pc-settings.js` 新增“图片下载记录”区域。
2. 列表展示至少包含：
   - 文件名
   - 下载时间
   - 来源入口
   - 保存结果或保存位置摘要
3. 增加一键清除按钮：
   - 清除前做二次确认。
   - 清空后立即刷新页面状态。
4. 保持 PC 现有备份与互通区域布局不被挤乱，历史区域采用独立面板，不塞进现有卡片里硬挤。

### Phase 3：移动端设置页增加下载历史

1. 在 `src/js/mobile-settings.js` 新增“图片下载记录”区域。
2. 列表展示优先做轻量密度：
   - 文件名
   - 时间
   - 保存结果
   - 必要时显示来源页签
3. 增加一键清除历史：
   - 先确认再清空。
   - 清空后保留当前页面其他设置不变。
4. 列表为空时显示简洁空态，不做重装饰。

### Phase 4：移动端保存到手机相册

1. 先确认移动端当前下载链路的真实落点：
   - 只保存到应用内可见目录，还是已经具备外部相册写入能力。
2. 优先目标：
   - 在 Android 上把图片写入系统相册可见位置。
   - 保存成功后返回可复用的结果，供历史记录落库。
3. 实施路径建议：
   - 优先复用现有 Capacitor 原生能力。
   - 若当前插件能力不足，则补一个轻量 Android 原生桥接，专门负责把图片写入相册/媒体库。
4. 失败回退策略：
   - 若相册写入失败，至少保留原有下载能力，不影响图片可获取性。
   - 失败原因要可见，避免用户误以为已进相册。

### Phase 5：下载入口接记录

1. 在图片下载完成后统一写入历史：
   - PC 图片查看器下载
   - 移动端图片查看器下载
2. 统一调用点尽量放在共享下载工具层，避免每个页面重复记历史。
3. 如保存被取消，不写历史；如保存失败，按需要保留错误态日志，但不进入成功列表。

### Phase 6：测试与文档

1. 前端测试：
   - 历史列表渲染
   - 清空历史
   - 空态展示
   - 下载后记录新增
2. Android 验证：
   - 图片下载后可在相册/媒体库中找到。
   - 关闭应用后相册仍可见。
3. 文档同步：
   - `docs/apps-code-map.md`
   - PC / 移动端就近设置页说明文档
   - 如新增原生保存约定，再补技术文档和经验文档
