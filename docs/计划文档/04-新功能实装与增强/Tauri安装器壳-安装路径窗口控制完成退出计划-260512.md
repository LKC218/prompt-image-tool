# Tauri安装器壳-安装路径窗口控制完成退出计划

## 思考与分析

本轮目标来自三张安装器壳截图，属于已实装 Tauri 安装向导上的窄范围交互修正，不改变安装核心、不调整页面步骤结构，也不引入新的前端框架。

技能与上下文嗅探结果：

- 已使用 `planning-with-files` 技能，本计划以持久化 Markdown 文档落地，先计划、后实施。
- 当前项目没有根目录 `SKILL.md`；本轮遵循仓库现有 `AGENTS.md` 工作流，先制定方案并等待确认。
- `installer-shell/` 现有结构是单 HTML 壳体加 `src/pages/*.js` 页面模块，继续保持该结构。
- 安装位置页已有 `浏览` 按钮和 Rust 命令 `choose_install_dir`，但默认状态仍偏向 `%LOCALAPPDATA%\PIM-Test`，需要按需求调整到下载目录语义。
- 窗口已配置 `decorations: false`，页面壳体已有 `data-tauri-drag-region`、最小化和关闭按钮，但仍保留不可用的最大化按钮；完成页关闭报错表现为 `Command plugin:window|close not allowed by ACL`，优先补齐 Tauri 权限而不是改成假关闭。
- 完成页当前会先处理快捷方式和启动软件，再调用窗口关闭；需求是“最后一步完成就退出安装窗口”，因此完成按钮收尾成功后必须可靠关闭安装器壳。

## 实施计划

### 1. 安装位置页增加本地浏览并默认到下载目录

- 将安装位置默认值从 `%LOCALAPPDATA%\PIM-Test` 调整为用户下载目录下的稳定子目录，例如 `%USERPROFILE%\Downloads\PIM-Test` 或 Rust 侧解析后的 `Downloads\PIM-Test`。
- 扩展 `resolve_target_dir()`，支持 `%USERPROFILE%` 或专门的默认下载目录函数，避免在前端写死绝对路径。
- 调整 `default_install_dir()` 与 `choose_install_dir()`，让浏览目录窗口默认打开下载目录目标文件夹；如果当前输入路径有效，则仍优先用当前输入路径作为初始位置。
- 保留用户手动输入路径能力，继续走 `validate_install_path()` 校验可写性和实际路径展示。
- 安装位置页文案同步为“默认安装到下载目录，可点击浏览选择本地位置”，避免用户误解路径展开状态。

### 2. 窗口支持拖动、最小化和关闭，移除最大化按钮

- 从 `installer-shell/src/index.html` 移除 `#maximize` 按钮，不再展示禁用的最大化控件。
- 保留 `#minimize` 和 `#close`，继续通过 `getCurrentWindow().minimize()`、`getCurrentWindow().close()` 调用 Tauri 窗口能力。
- 检查 `.titlebar` 的拖动区域，确保非按钮区域可拖动，按钮本身不误触拖动。
- 更新窗口控制样式，重新对齐右上角两个按钮的间距和命中区域，保持截图中的轻量标题栏视觉。
- 补齐 `installer-shell/src-tauri/capabilities/default.json` 中窗口最小化、关闭、拖动所需权限，重点解决 `window|close not allowed by ACL`。

### 3. 完成页点击完成后退出安装窗口

- 保持完成页的快捷方式收尾和可选启动逻辑，但在收尾完成后必须调用窗口关闭。
- 如果启动软件失败，不阻塞安装器壳退出；仅把失败信息作为短状态或日志处理，避免用户卡在完成页。
- 完成按钮进入工作态后防止重复点击，关闭失败时恢复按钮并显示明确原因。
- 验证 release 窗口中点击“完成”不再出现 ACL 报错，安装器壳进程退出。

## 风险与边界

- 本轮不改 NSIS 安装核心，不改变静默安装参数。
- 下载目录默认安装可能与传统应用安装习惯不同，但按本次需求执行；如后续要发布给普通用户，可再评估默认回到 `%LOCALAPPDATA%`，只让浏览窗口默认打开下载目录。
- Tauri ACL 权限需按最小权限补齐，只允许当前窗口需要的关闭、最小化、拖动能力。
- 完成页的“立即启动软件”如果执行较慢，可能影响退出体感；实施时优先保证安装器壳最终退出。
