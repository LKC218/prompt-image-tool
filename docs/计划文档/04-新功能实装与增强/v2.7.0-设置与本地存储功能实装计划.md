### 1.2 关键决策：取消深色模式

**决策依据**：

1. 设计规范 `00-全局设计规范.md` 的「负面设计约束」明确禁止「大面积深色背景」和「暗黑赛博朋克风」
2. 移动端设计理念为「温暖治愈」——奶油白背景 + 马卡龙配色，深色模式与核心设计理念冲突
3. 当前深色模式 CSS 变量完全缺失，实装成本高且与设计规范矛盾
4. 移动端固定浅色主题，保持视觉一致性

**影响**：

- 移除设置页中「深色模式开关」和「跟随系统开关」两行 UI
- 移除 `isDarkMode`、`isFollowSystem` 状态变量及相关逻辑
- 移动端始终使用浅色主题，不受 `data-theme` 属性影响
- PC 端主题切换功能不受影响

### 🔍 审核点 1：主题色影响范围

**背景**：设计规范定义了 5 种主题色（粉/绿/蓝/紫/橙），当前仅 UI 展示。实装后需要确定哪些 UI 元素跟随主题色变化。

**需要确认**：

> 主题色切换后，以下哪些元素应跟随变色？
>
> | 元素 | 是否跟随 | 说明 |
> |------|---------|------|
> | 底部导航栏选中态颜色 | ？ | 当前首页/分类/设置为蓝色，提示词库为粉色 |
> | 悬浮加号按钮颜色 | ？ | 当前为粉色渐变 |
> | 保存/主操作按钮颜色 | ？ | 当前为粉色 |
> | 进度条/加载指示器 | ？ | 当前为蓝色 |
> | 卡片选中态边框 | ？ | 当前为蓝色 |
> | 开关开启态颜色 | ？ | 当前为绿色 |
>
> **方案 A：最小影响（推荐 Phase 1）**
> - 仅变更：底部导航栏选中态、悬浮加号按钮、保存按钮
> - 保留：开关绿色（语义色）、进度条蓝色（信息色）
>
> **方案 B：全面影响**
> - 所有品牌色元素跟随主题色变化
> - 需要引入 CSS 变量 `--m-accent` 替代硬编码的品牌色引用
>
> **方案 C：仅 UI 展示，不实装功能**
> - 保持现状，主题色选择器仅做视觉展示

**影响范围**：决定是否需要重构 CSS 变量体系，影响所有移动端页面的品牌色引用

### 🔍 审核点 2：本地数据大小计算方式

**背景**：设计规范要求显示「本地数据大小」，当前硬编码为 `128 MB`。移动端使用 CapacitorSQLite 存储，数据包含提示词集合、版本、图片等。

**需要确认**：

> 本地数据大小应如何计算？
>
> **方案 A：仅计算数据库大小（推荐）**
> - 通过 `exportData()` 导出后计算 JSON 字符串大小
> - 图片以 DataURL 存储在数据库中，已包含在内
> - 实现简单，性能好
>
> **方案 B：计算数据库 + 图片文件总大小**
> - 需要遍历文件系统统计图片目录大小
> - 更准确但实现复杂，需 Capacitor Filesystem API
> - 性能开销较大
>
> **方案 C：Phase 1 保持 mock，Phase 2 实装**
> - 当前版本保持硬编码值，后续版本实装真实计算

**影响范围**：决定是否需要引入 Capacitor Filesystem API 依赖

### 🔍 审核点 3：4 个备份恢复按钮的功能区分

**背景**：设计规范定义了 4 个按钮，但当前实现中「本地备份」=「导出 JSON」、「导入数据」=「从 JSON 导入」。

**需要确认**：

> 4 个备份恢复按钮是否需要功能区分？如果区分，如何区分？
>
> **方案 A：合并为 2 个按钮（推荐）**
> - 保留「本地备份」（导出）和「导入数据」（导入）2 个按钮
> - 移除「导出 JSON」和「从 JSON 导入」
> - 理由：当前所有数据本身就是 JSON 格式，4 个按钮功能完全重复
>
> **方案 B：区分格式**
> - 「本地备份」= 完整备份（含图片 DataURL），文件较大
> - 「导出 JSON」= 仅文本数据（不含图片），文件较小
> - 「导入数据」= 自动识别格式（完整/精简）
> - 「从 JSON 导入」= 同上
> - 需要后端/存储层支持两种导出格式
>
> **方案 C：保持 4 个按钮但共享逻辑**
> - UI 保持 4 个按钮（视觉丰富）
> - 功能上本地备份=导出JSON，导入数据=从JSON导入
> - 当前实现即为此方案

**影响范围**：决定是否需要修改存储层 API 和 UI 布局

### 🔍 审核点 4：存储空间环形图实现方式

**背景**：设计规范要求存储空间卡片显示环形图，当前仅有文字。

**需要确认**：

> **方案 A：CSS conic-gradient（推荐）**

## 三、任务执行顺序

按照 **依赖关系** 和 **影响范围** 合理排列，前置依赖未完成时后续任务不可开始：

```
Step 0: 人工审核（4 个审核点）                    ← 阻塞所有开发
    ↓
Step 1: 移除深色模式/跟随系统 UI 和逻辑            ← 清理废弃代码，为后续腾出空间
    ↓
Step 2: 主题色系统实装                             ← 依赖审核点 1 决策
    ↓
Step 3: 本地存储信息实装                           ← 依赖审核点 2、4 决策
    ↓
Step 4: 备份恢复功能区分                           ← 依赖审核点 3 决策
    ↓
Step 5: 安全提示栏增强                             ← 独立，低优先级
    ↓
Step 6: 全页面验证 + 构建测试                      ← 收尾验证
```

### 排序理由

1. **人工审核优先**：4 个审核点的决策直接影响后续所有任务的实施方案，必须先确认
2. **清理废弃代码先行**：移除深色模式/跟随系统相关代码，减少干扰，为后续修改腾出空间
3. **主题色系统第二**：主题色需要确认影响范围（审核点 1），且可能需要重构 CSS 变量引用
4. **本地存储信息第三**：相对独立，但需要确认计算方式（审核点 2）和环形图方案（审核点 4）
5. **备份恢复第四**：需要确认功能区分方案（审核点 3）
6. **安全提示栏第五**：改动最小，优先级最低
7. **全页面验证收尾**：确保所有功能正常工作

#### 3.1 真实本地数据大小计算

```javascript
async function calculateLocalDataSize() {
    try {
        const data = await getStorage().exportData();
        const jsonStr = JSON.stringify(data);
        const bytes = new Blob([jsonStr]).size;
        return formatBytes(bytes);
    } catch (e) {
        return '计算失败';
    }
}

function formatBytes(bytes) {
    if (bytes === 0) return '0 B';
    const k = 1024;
    const sizes = ['B', 'KB', 'MB', 'GB'];
    const i = Math.floor(Math.log(bytes) / Math.log(k));
    return parseFloat((bytes / Math.pow(k, i)).toFixed(1)) + ' ' + sizes[i];
}
```

#### 3.2 存储空间信息获取

```javascript
async function getStorageInfo() {
    if (navigator.storage && navigator.storage.estimate) {
        const estimate = await navigator.storage.estimate();
        return {
            used: formatBytes(estimate.usage || 0),
            quota: formatBytes(estimate.quota || 0),
            usedPercent: estimate.quota ? Math.round((estimate.usage / estimate.quota) * 100) : 0,
        };
    }
    return { used: '未知', quota: '未知', usedPercent: 0 };
}
```

#### 4.2 增加导入确认流程

导入前弹出 ActionSheet 确认，防止误操作覆盖数据：

```javascript
function handleImport() {
    showActionSheet([
        {
            action: 'confirm',
            icon: '⚠️',
            label: '确认导入（将与现有数据合并）',
            handler: async () => {
                const input = document.createElement('input');
                input.type = 'file';
                input.accept = '.json';
                input.onchange = async (e) => {
                    const file = e.target.files?.[0];
                    if (!file) return;
                    try {
                        const text = await file.text();
                        const data = JSON.parse(text);
                        const result = await getStorage().importData(Array.isArray(data) ? data : [data]);
                        showMobileToast(`导入成功，共 ${result.imported} 条`);
                    } catch (err) {
                        showMobileToast('导入失败，文件格式不正确', 'error');
                    }
                };
                input.click();
            }
        },
        {
            action: 'cancel',
            icon: '❌',
            label: '取消',
            handler: () => {}
        }
    ]);
}
```

#### 4.3 导出成功后显示文件信息

```javascript
async function handleExport() {
    try {
        const data = await getStorage().exportData();
        const blob = new Blob([JSON.stringify(data, null, 2)], { type: 'application/json' });
        const url = URL.createObjectURL(blob);
        const a = document.createElement('a');
        a.href = url;
        a.download = `prompt-backup-${new Date().toISOString().slice(0, 10)}.json`;
        a.click();
        URL.revokeObjectURL(url);
        const size = formatBytes(blob.size);
        showMobileToast(`导出成功（${size}）`);
    } catch (e) {
        showMobileToast('导出失败，请重试', 'error');
    }
}
```

## 五、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 主题色影响范围过大导致 UI 混乱 | 部分元素变色后可读性下降 | Phase 1 仅变更少量元素，逐步扩大范围 |
| `navigator.storage.estimate()` 在部分浏览器不可用 | 存储空间信息无法获取 | 提供 fallback 显示"未知" |
| CSS 变量嵌套兼容性 | `var(--m-accent)` 引用 `var(--m-pink)` 在旧 WebView 可能不支持 | 测试 Android 8+ WebView 兼容性 |
| 主题色切换影响底部导航栏颜色 | 导航栏选中态颜色变化可能让用户困惑 | 仅变更当前选中项颜色，非选中项保持灰色 |
| 移除深色模式后 PC 端主题切换可能影响移动端 | `data-theme` 属性在 `documentElement` 上全局生效 | 移动端初始化时强制设置 `data-theme="light"` |
