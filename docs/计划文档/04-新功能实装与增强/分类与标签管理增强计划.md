# 分类与标签管理增强 — 实施计划文档

> 文档创建日期：2026-05-05
> 基线：当前 mobile-category.js（261行）+ mobile-editor.js（AVAILABLE_TAGS 硬编码 15 个标签）
> 目标：完善分类新建体验 + 实现标签管理真实功能（含清除现有标签）
> 技术栈：原生 JS + Vite + Capacitor 8.x (Android) + SQLite + Python 后端

---

## 一、版本目标

将移动端「分类与标签」页面从当前的**半成品**状态提升至**完整可用**状态，核心聚焦于：

1. **新建分类体验升级** — 从原生 `prompt()` 弹窗升级为自定义弹窗（名称输入 + 颜色选择），对齐设计规范
2. **标签管理真实化** — 从 Mock 数据升级为基于实际数据的标签统计与管理，支持清除现有标签
3. **标签数据去硬编码** — 标签列表从 `mobile-editor.js` 硬编码迁移为动态聚合，实现全局一致性

## 五、T2 — 标签数据聚合引擎（P0）

### 5.1 当前状态

- 标签存储在 `prompt_sets.tags` 字段中，格式为 JSON 字符串（如 `'["场景","日系"]'`）
- `mobile-category.js` 中的 `MOCK_TAGS` 是硬编码假数据

- 没有从实际数据中聚合标签的机制

### 5.2 实现设计

#### F2.1 标签聚合函数

- 在 `mobile-category.js` 中新增 `aggregateTags(promptSets)` 函数
- 遍历所有 `prompt_sets`，解析每条的 `tags` 字段，统计每个标签的出现次数

```javascript
function aggregateTags(promptSets) {
    const tagMap = new Map();
    promptSets.forEach(set => {
        let tags = [];
        try {
            tags = JSON.parse(set.tags || '[]');
        } catch (e) {
            tags = [];
        }
        if (!Array.isArray(tags)) tags = [];
        tags.forEach(tag => {
            if (typeof tag !== 'string' || !tag.trim()) return;
            const name = tag.trim();
            tagMap.set(name, (tagMap.get(name) || 0) + 1);
        });
    });
    return Array.from(tagMap.entries())
        .map(([name, count]) => ({ name, count, style: getTagStyleClass(name) }))
        .sort((a, b) => b.count - a.count);
}
```

#### F3.3 清除标签功能

**核心功能：从所有提示词中移除指定标签**

- 实现逻辑：
  1. 遍历所有 `prompt_sets`
  2. 解析每条的 `tags` JSON 字符串
  3. 过滤掉目标标签
  4. 序列化回 JSON 字符串
  5. 调用 `storage.updatePromptSet()` 更新

```javascript
async function clearTagFromAll(tagName) {
    const storage = getStorage();
    const promptSets = await storage.getPromptSets();
    const updates = [];

    for (const set of promptSets) {
        let tags = [];
        try {
            tags = JSON.parse(set.tags || '[]');
        } catch (e) {
            tags = [];
        }
        if (!Array.isArray(tags)) tags = [];

        const newTags = tags.filter(t => t !== tagName);
        if (newTags.length !== tags.length) {
            updates.push(storage.updatePromptSet(set.id, { tags: JSON.stringify(newTags) }));
        }
    }

    await Promise.all(updates);
    return updates.length;
}
```

## 九、T6 — 标签重命名功能（P2）

### 9.1 实现设计

#### F9.1 重命名逻辑

- 遍历所有 `prompt_sets`，将 `tags` 中的旧标签名替换为新标签名
- 需要存储层支持批量更新

```javascript
async function renameTag(oldName, newName) {
    const storage = getStorage();
    const promptSets = await storage.getPromptSets();
    const updates = [];

    for (const set of promptSets) {
        let tags = [];
        try {
            tags = JSON.parse(set.tags || '[]');
        } catch (e) {
            tags = [];
        }
        if (!Array.isArray(tags)) tags = [];

        const newTags = tags.map(t => t === oldName ? newName : t);
        if (JSON.stringify(newTags) !== JSON.stringify(tags)) {
            updates.push(storage.updatePromptSet(set.id, { tags: JSON.stringify(newTags) }));
        }
    }

    await Promise.all(updates);
    return updates.length;
}
```

- 在标签操作菜单中增加「重命名」选项
- 弹出输入弹窗，预填当前标签名
