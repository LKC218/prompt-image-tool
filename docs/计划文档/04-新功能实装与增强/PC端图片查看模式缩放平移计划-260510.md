# PC端图片查看模式缩放平移计划-260510

> 范围：PC 端图片预览点击后的全屏查看模式。目标是在现有查看器中增加鼠标滚轮缩放与拖拽平移能力，不改图片存储、上传、删除和移动端查看逻辑。

### 当前代码定位

- 公共查看器入口在 `src/js/pc-utils.js`：
  - `showImageViewer(src)` 创建 `#pcImageViewer`。
  - 当前结构只有 `<img id="pcImageViewerImg">`。
  - 当前关闭逻辑是 `viewer.addEventListener('click', closeImageViewer)`，点击查看器任意位置都会关闭。
- 样式入口在 `src/css/pc.css`：
  - `.pc-image-viewer` 是全屏固定遮罩，黑色背景，居中图片。
  - `.pc-image-viewer img` 使用 `max-width: 90%`、`max-height: 90%`、`object-fit: contain`。
- 复用入口：
  - `src/js/pc-detail.js` 的封面图片点击调用 `showImageViewer(imageUrls[currentImageIndex].url)`。
  - `src/js/pc-library.js` 的右侧预览封面点击调用 `showImageViewer(previewImg.src)`。
  - `src/js/pc-app.js` 中 `Escape` 会关闭 `#pcImageViewer`。

### 问题本质

当前查看模式只是“全屏居中展示 + 点击关闭”，不具备查看局部细节的能力。对于生图结果，用户经常需要检查局部纹理、手部、文字、构图细节，因此需要在不离开当前页面的情况下完成：

- 鼠标滚轮以鼠标所在位置为视觉锚点缩放。
- 图片放大后可拖拽平移。
- 双击或按钮快速复位到适屏状态。
- 点击遮罩仍可关闭，但点击图片本身不能误关闭。
- `Escape` 关闭时清理缩放和平移状态，下一张图从默认状态打开。

### 设计目标

- 保持 KISS：只改公共查看器，不在详情页和提示词库分别实现。
- 不新增依赖：用原生 `wheel`、`pointerdown`、`pointermove`、`pointerup`、`dblclick` 事件完成。
- 不改变数据：只处理当前图片元素的 DOM 状态和 CSS transform。
- 不破坏原行为：仍支持 `Escape` 关闭、点击遮罩关闭、图片加载失败兜底。
- 对用户可感知：鼠标指针、轻量工具栏、缩放比例反馈要明确。
- 对实现可维护：缩放状态集中在 `pc-utils.js` 内部，暴露的 `showImageViewer(src)` 调用方式保持不变。

## 实施计划

### Phase 1：重构查看器 DOM 结构

1. 在 `showImageViewer(src)` 首次创建查看器时，将结构从单图片升级为：
   - 外层：`#pcImageViewer.pc-image-viewer`。
   - 画布区：`.pc-image-viewer-stage`，承载图片和拖拽事件。
   - 图片：`#pcImageViewerImg.pc-image-viewer-img`。
   - 右上工具区：关闭、复位、缩放比例展示。
2. 保持 `showImageViewer(src)` 的外部调用签名不变。
3. 打开新图片时统一执行 `resetImageViewerTransform()`。

### Phase 2：缩放状态模型

1. 在 `pc-utils.js` 内部维护查看器状态：
   - `scale`：当前缩放倍数，默认 `1`。
   - `translateX` / `translateY`：当前平移量，默认 `0`。
   - `isDragging`：是否正在拖拽。
   - `dragStartX` / `dragStartY`：拖拽起点。
   - `startTranslateX` / `startTranslateY`：拖拽开始时的平移量。
2. 建议参数：
   - 最小缩放：`1`，表示适屏状态。
   - 最大缩放：`5`，足够检查细节，避免 WebView 绘制压力过大。
   - 滚轮缩放步进：按 `deltaY` 使用 `1.12` 左右的倍数递增/递减。
3. 所有状态变化统一走 `applyImageViewerTransform()`，只负责写入 `transform`、比例文本和拖拽光标状态。

### Phase 3：鼠标滚轮缩放

1. 在查看器激活时监听 `wheel`：
   - `preventDefault()` 阻止页面滚动或 WebView 默认滚动。
   - 根据滚轮方向计算新缩放倍数，并限制在 `1` 到 `5`。
2. 缩放锚点采用鼠标位置：
   - 读取图片或画布当前几何信息。
   - 计算鼠标相对图片中心的偏移。
   - 调整 `translateX` / `translateY`，让鼠标指向的图片区域在缩放后尽量保持在原位置。
3. 当缩放回 `1` 时自动归零平移，回到适屏居中。

### Phase 4：拖拽平移

1. 仅当 `scale > 1` 时允许拖拽图片。
2. 使用 Pointer Events：
   - `pointerdown`：记录起点并 `setPointerCapture()`。
   - `pointermove`：计算平移增量。
   - `pointerup` / `pointercancel`：结束拖拽。
3. 点击关闭逻辑调整为：
   - 点击遮罩空白区关闭。
   - 点击关闭按钮关闭。
   - 点击图片不关闭。
   - 拖拽结束后的释放动作不触发误关闭。
4. 平移增加边界约束：
   - 放大后允许移动到边缘，但不让图片完全拖出视口。
   - 图片小于视口时对应方向平移归零。

### Phase 5：复位、双击和键盘辅助

1. 新增复位按钮，点击后回到 `scale = 1`、`translateX = 0`、`translateY = 0`。
2. 图片双击：
   - 当前为 `1` 时放大到 `2`，并以双击位置为锚点。
   - 当前大于 `1` 时复位。
3. 保持 `Escape` 关闭，并在关闭时调用状态清理。
4. 可选增强：
   - `+` / `-` 控制缩放。
   - `0` 或 `Ctrl+0` 复位。
   - 若实现键盘缩放，需避免影响输入框场景。

### Phase 6：样式与交互反馈

1. 更新 `src/css/pc.css` 的 `.pc-image-viewer` 相关样式：
   - 遮罩保持沉浸黑底。
   - 图片添加 `will-change: transform`。
   - 缩放动画控制在 `120ms` 到 `180ms`，拖拽时禁用过渡。
   - `cursor` 状态：默认 `zoom-in`，可拖拽时 `grab`，拖拽中 `grabbing`。
2. 工具栏保持轻量：
   - 半透明深色背景。
   - 小尺寸按钮，不遮挡图片主体。
   - 按钮有明确 `title` / `aria-label`。
3. 遵守 `prefers-reduced-motion`：
   - 减少或关闭 transform 过渡。
