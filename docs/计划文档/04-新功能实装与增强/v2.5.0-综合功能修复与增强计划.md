**实施内容**：

#### 1.1 SQLite 存储层增加 `is_favorite` 字段

- 文件：`src/js/sqlite-storage.js`
- 在 `init()` 中增加 `ALTER TABLE` 迁移：
  ```sql
  ALTER TABLE prompt_sets ADD COLUMN is_favorite INTEGER DEFAULT 0;
  ```
- `getPromptSets()` 返回结果增加 `isFavorite: !!row.is_favorite`
- `getPromptSet()` 返回结果增加 `isFavorite: !!s.is_favorite`
- 新增 `toggleFavorite(id)` 方法：
  ```sql
  UPDATE prompt_sets SET is_favorite = CASE WHEN is_favorite = 1 THEN 0 ELSE 1 END, updated_at = ? WHERE id = ?
  ```

#### 1.2 Python 后端增加 `isFavorite` 字段

- 新增 API `POST /api/prompt-set/{id}/toggle-favorite`：切换收藏状态

#### 1.3 ApiStorage 层增加收藏方法

- 文件：`src/js/api-storage.js`

## 四、风险与注意事项

1. **数据库迁移安全**：SQLite 的 `ALTER TABLE ADD COLUMN` 是幂等操作（列已存在时会抛异常，已被 try-catch 包裹），无需额外处理
2. **PC/移动端数据一致性**：Python 后端和 SQLite 存储层都需要同步增加 `isFavorite` 字段
3. **图片加载性能**：详情页加载多张图片时需注意性能，缩略图应使用小尺寸，全屏查看时才加载原图
4. **标签解析容错**：`set.tags` 为 JSON 字符串，解析时需 try-catch，兼容空值和非法格式
5. **收藏状态实时性**：页面切换时需重新从存储层加载数据，不能依赖内存缓存
