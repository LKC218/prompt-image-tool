## 思考与分析

### 技能嗅探结果

- 根目录未发现 `SKILL.md`。
- 本任务是“制定计划”，适用 `planning-with-files` 技能，因此先把 README 介绍页设计方案沉淀为计划文档，等待确认后再实施。
- 参考图展示的是典型 GitHub README 首页结构：顶部品牌区、徽章、界面预览、推荐区、快速开始、下载说明与平台差异说明。

### 当前问题判断

当前根目录 `README.md` 更像工程入口说明，内容偏短，缺少 GitHub 首页应有的产品展示、截图、下载路径、功能亮点和用户视角说明。终端读取时还出现中文显示异常，因此后续实施时必须把 README 作为 UTF-8 文件整体校验，不能用容易破坏中文的大段 PowerShell 管道拼接。

本次设计目标不是把 README 写成完整技术文档，而是把它变成仓库首页的“第一屏介绍页”：让用户在 30 秒内理解这是什么、能做什么、长什么样、怎么下载、如何开发运行，并能继续跳转到详细文档。

### 设计原则

- README 面向用户和潜在贡献者，先讲产品价值，再讲工程入口。
- 介绍内容必须基于当前已实现能力，避免写未来承诺。
- 截图和图标优先使用仓库内本地资源，避免外链失效。
- Release 下载说明要指向 GitHub Releases，安装包不进入 Git 历史。
- `docs/apps-code-map.md` 只做导航落点，不记录本次执行过程。
- README 保持简洁，长篇技术细节转到 `docs/技术文档/`、`docs/构建方案/`、`docs/工程文档/`。

## 实施计划

### 1. 信息架构设计

README 建议按以下顺序组织：

1. 顶部品牌区
   - 应用图标或 Logo。
   - 项目名称：`PromptImageManager` 或中文主名“生图提示词管理器”。
   - 一句话定位：面向 AI 生图工作流的提示词、图片、分类和跨端同步管理工具。
   - 徽章：版本、平台、构建状态、许可证、Release 下载入口。

2. 界面预览
   - 放置 PC 端首页或提示词库截图。
   - 放置移动端首页或设置页同步截图。
   - 如后续支持明暗主题，再补充浅色和深色主题对比。

3. 核心功能
   - 提示词库管理。
   - 分类与标签。
   - 图片与封面管理。
   - 数据导入导出。
   - PC 与 Android 局域网同步。
   - PC 独立安装包与 Android 安装包。

4. 下载与安装
   - 普通用户优先：跳转 `Releases` 下载。
   - Windows：说明安装包名称和适用系统。
   - Android：说明 APK 安装方式和首次安装未知来源提醒。
   - 开发者：保留源码运行命令。

5. 快速开始
   - 前端开发运行。
   - Python 后端运行。
   - PC 安装包构建。
   - Android 安装包构建。
   - 只保留最短命令，细节跳转到构建文档。

6. 项目结构与文档入口
   - 简短目录说明。
   - 链接到 `docs/README.md`、`docs/apps-code-map.md`、构建方案、技术文档和版本记录。

7. 数据与隐私说明
   - 本地数据优先。
   - 运行时数据、备份、构建产物不提交到 Git。
   - 局域网同步仅在用户指定的网络环境中使用。

8. 推荐或鸣谢区
   - 如果需要放置类似参考图中的“推荐服务”，应单独成段，标题明确。
   - 推荐内容不能打断下载和快速开始主流程。
   - 外部链接需要清楚标注，不与项目官方功能混淆。

### 2. 素材规范

建议新增素材目录：

```text
docs/assets/readme/
```

建议素材清单：

- `logo.png`：README 顶部展示图标。
- `preview-pc.png`：PC 端主界面截图。
- `preview-mobile.png`：移动端主界面截图。
- `preview-sync.png`：局域网同步或设置页截图，可选。

素材要求：

- 截图中不出现真实隐私数据、访问令牌、本地路径、健康数据或个人信息。
- 图片宽度控制在 GitHub README 友好的尺寸，优先压缩后提交。
- README 使用相对路径引用，例如 `docs/assets/readme/preview-pc.png`。
- 不把构建产物、安装包或临时截图缓存提交进仓库。

### 3. 文案结构草案

README 顶部建议文案方向：

```md
# PromptImageManager

面向 AI 生图工作流的本地提示词与图片管理工具，支持 PC 与 Android 局域网同步。
```

功能亮点建议使用表格，避免 README 过长：

```md
| 能力 | 说明 |
| --- | --- |
| 提示词库 | 保存、搜索、复用正向与反向提示词 |
| 分类标签 | 用分类、颜色和标签组织素材 |
| 图片管理 | 绑定封面图、参考图和本地图片资源 |
| 数据备份 | 支持导入、导出和迁移 |
| 局域网同步 | 支持 PC 与 Android 拉取、回传和双向同步 |
```

下载区建议优先面向普通用户：

```md
## 下载

- Windows：前往 Releases 下载 `PromptImageManager-Setup-版本号.exe`
- Android：前往 Releases 下载 `PromptImageManager-v版本号-Android.apk`
```

开发命令保留但后置：

```powershell
npm install
npm run dev
```

### 4. 文档同步落点

实施 README 介绍页时，同步检查：

- `docs/apps-code-map.md`：更新 `README.md` 的职责描述和 README 素材目录导航。
- `docs/工程文档/工程目录说明.md`：如新增 `docs/assets/readme/`，补充目录职责。
- `docs/构建方案/README.md` 或相关构建文档：如果 README 调整构建入口链接，确保目标文档存在且表述一致。
- `docs/版本记录/changelog.md`：如 README 改动属于发布展示优化，可按需记录。

### 5. 验证方案

实施后需要做以下检查：

1. 使用 UTF-8 方式读取 README，确认中文未乱码。
2. 检查 README 中所有相对链接是否存在。
3. 检查图片路径是否能在 GitHub README 中正常渲染。
4. 检查 Release 链接是否指向真实可下载页面。
5. 检查 `.gitignore` 没有误排除 README 素材，也没有放开安装包、缓存、运行时数据。
6. 如修改构建命令说明，至少运行 `npm.cmd run test` 和必要的文档链接检查。
