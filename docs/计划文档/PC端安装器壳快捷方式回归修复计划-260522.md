# PC端安装器壳快捷方式回归修复计划-260522

## 思考与分析

### Skill嗅探结果

- 根目录未发现 `SKILL.md`。
- 本次是计划制定任务，使用 `planning-with-files` 工作流沉淀为计划文档。
- 既有计划 `docs/计划文档/PC端安装后快捷方式缺失修复计划-260515.md` 显示：NSIS 核心安装包已能创建桌面快捷方式、开始菜单启动快捷方式和开始菜单卸载快捷方式；上一轮根因曾收敛到 Tauri 安装器壳收尾层缺少真实存在校验。

### 当前判断

这次问题描述为“PC端带安装壳 UI 安装后没有创建桌面快捷键和开始快捷”，应按回归问题处理，而不是直接重写 NSIS 脚本。优先排查三条链路：

1. 用户安装的是否为最新安装器壳产物，避免旧 `releases/` 或旧 GitHub Release 资产被误用。
2. Tauri 安装器壳是否正确嵌入最新 NSIS 核心安装包，避免 `--skip-pc-build` 复用过期核心。
3. 安装完成后是否真实回读 `.lnk` 落地状态，避免主程序安装成功但快捷方式失败仍显示完整成功。

## 实施计划

### Phase 1：产物身份与现状确认

1. 记录当前待验证安装包路径、大小、修改时间和哈希：
   - `releases/PromptImageManager-Shell-Setup-*.exe`
   - `build/PromptImageManager-Setup-*.exe`
   - `releases/PromptImageManager-Setup-*.exe`
2. 检查 `package.json` 版本、`scripts/build_installer_shell_package.py` 嵌入的核心安装包路径，以及 Tauri `embedded-installer` feature 绑定的实际文件。
3. 在安装前检查当前用户桌面和开始菜单是否已有旧快捷方式，必要时记录后清理，避免残留干扰判断。

### Phase 2：分层复现

1. 单独静默运行 NSIS 核心安装包到临时目录，验证：
   - 主程序存在。
   - 卸载器存在。
   - 桌面 `生图提示词管理器.lnk` 存在。
   - 开始菜单启动快捷方式存在。
   - 开始菜单卸载快捷方式存在。
2. 再运行 Tauri 安装器壳完整流程，验证同一组结果。
3. 对比两次结果：
   - NSIS 成功、安装器壳失败：集中修 Tauri 壳嵌入、附加任务状态、收尾命令或状态回读。
   - NSIS 失败、安装器壳失败：集中修 `build/installer.nsi`、编码、权限或 Windows 特殊目录解析。
   - 二者都成功但用户仍失败：集中排查用户实际安装的发布资产是否过旧。

### Phase 3：最小修复方向

1. 保持 NSIS 创建快捷方式为主路径，不重复造一套安装逻辑。
2. 若安装器壳复用旧核心，修正构建脚本的核心安装包校验，让 shell 构建前明确失败，而不是静默嵌入旧文件。
3. 若安装器壳收尾状态误判，修正 `installer-shell/src-tauri/src/main.rs` 的状态回读和错误返回。
4. 若完成页状态展示不准确，修正 `installer-shell/src/main.js`，确保桌面、开始菜单、卸载入口三项真实状态都参与完成页提示。
5. 若 OneDrive 桌面或开始菜单路径导致检测误判，补齐路径候选和验证逻辑。

### Phase 4：验证闭环

1. 静态检查：
   - `node --check installer-shell/src/main.js`
   - `node --check installer-shell/src/pages/*.js`
   - `cargo check --manifest-path installer-shell/src-tauri/Cargo.toml`
2. 构建验证：
   - `python scripts/build_pc_package.py --skip-env-check`
   - `python scripts/build_installer_shell_package.py --skip-env-check`
3. 真实安装验证：
   - 临时目录安装。
   - 检查桌面快捷方式、开始菜单启动快捷方式、开始菜单卸载快捷方式。
   - 检查快捷方式目标、工作目录和图标。
   - 启动安装后的主程序并探测 `/api/health`。
4. 卸载验证：
   - 执行卸载器。
   - 检查安装目录、桌面快捷方式、开始菜单目录和卸载注册表是否清理。

### Phase 5：文档同步

如进入实施并修改代码或构建流程，需要同步：

1. `docs/apps-code-map.md`
2. `docs/构建方案/PC独立安装包快速打包流程.md`
