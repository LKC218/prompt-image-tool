# 顶部欢迎横幅复用计划

> 状态：编辑页横幅撑满修复已实施并通过自动化验证，待页面预览确认

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 本次触发 `planning-with-files`：需求属于多页面 UI 调整，需要先沉淀计划，再实施。
- 当前红框 UI 位于 `src/js/pc-home.js` 的 `.pc-welcome-banner`。
- 样式位于 `src/css/pc.css` 的 `.pc-welcome-banner`、`.pc-welcome-section`、`.pc-welcome-text`、`.pc-welcome-mascot`。
- 背景图资源为 `src/assets/pc/home-bg.png`，右侧柯基资源为 `src/assets/mobile/mascots/corgi-home.png`。
- 目标复用页面：
  - `src/js/pc-library.js`：提示词库
  - `src/js/pc-editor.js`：新建/编辑
  - `src/js/pc-category.js`：分类与标签

### 1.2 初步判断

首页横幅当前直接写在 `pc-home.js` 中，如果直接复制到其他页面，会带来三类问题：

- 后续改背景图、标题尺寸、横幅高度时需要多处同步。
- `pc-editor.js` 已有独立头部和保存按钮，直接复制容易造成头部职责混乱。
- `.pc-welcome-banner` 当前使用负边距铺满内容区顶部，复用到其他页面前需要确认其对页面标题、搜索栏、编辑表单的间距影响。

因此建议抽出一个 PC 端共享横幅渲染模块，由各页面传入标题、副标题和可选操作区。

## 二、实施计划

### Phase 1：抽取共享横幅

新增 `src/js/pc-welcome-banner.js`：

- 统一导入 `home-bg.png` 和 `corgi-home.png`。
- 导出 `renderPcWelcomeBanner(options)`。
- 支持参数：
  - `title`：主标题
  - `subtitle`：副标题
  - `className`：页面修饰类
  - `actionsHtml`：可选操作区，用于编辑页放置保存按钮等关键操作

首页 `pc-home.js` 改为调用共享函数，确保原首页视觉不变。

## 六、追加计划：新建/编辑页横幅撑满

### 6.1 问题现象

新建/编辑页顶部欢迎横幅当前没有像提示词库页面一样横向撑满内容区域。参考截图中，编辑页横幅左右留白明显，视觉上更像页面内卡片；提示词库横幅则贴近主内容区左右边缘，背景图完整覆盖顶部区域。

### 6.2 根因判断

- 提示词库页面使用统一 `.pc-page` 容器，`.pc-welcome-banner` 通过负外边距抵消页面左右内边距，实现顶部横幅满宽。
- 新建/编辑页使用独立 `.pc-editor-page` 容器，虽然当前也复用了 `.pc-welcome-banner`，但编辑页专用横幅样式和表单主体布局混在同一页面容器内，容易让横幅表现成局部内容块。
- 编辑页横幅不仅要满宽，还要保留参考图的内部排版：左侧返回按钮、标题、副标题，右侧柯基插画和保存按钮。

### 6.3 实施方案

推荐采用“容器对齐提示词库，内部排版保留编辑页专属”的方案：

1. 统一横幅外层撑满逻辑
   - 保留 `.pc-welcome-banner` 的通用负外边距策略。
   - 移除 `.pc-editor-page` 的重复页面内边距，让外层 `.pc-page` 继续负责统一页面留白。
   - 避免给 `.pc-welcome-banner-editor` 添加会破坏满宽效果的额外 `max-width`、外层居中或局部卡片式边距。

2. 拆分编辑页横幅内部布局
   - 外层横幅负责撑满。
   - 内部 `.pc-welcome-section` 负责参考图排版。
   - 返回按钮、标题、副标题固定在左侧内容组。
   - 柯基插画和保存按钮固定在右侧内容组。

3. 表单主体保持居中卡片布局
   - 横幅下方 `#pcEditorContent` 继续控制编辑表单宽度与双栏布局。
   - 不让表单的最大宽度反向影响顶部横幅。

4. 响应式约束
   - 1280px 宽度下：标题不换行，保存按钮不挤压柯基。
   - 1440px 宽度下：横幅视觉应接近提示词库满宽效果。
   - 1600px 宽度下：右侧柯基与保存按钮不漂得过远，保持横幅内容平衡。
