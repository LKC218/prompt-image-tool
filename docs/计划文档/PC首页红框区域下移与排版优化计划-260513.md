# PC首页红框区域下移与排版优化计划

> 版本：v1.0.0  
> 创建日期：2026-05-13  
> 范围：PC 端首页红框内容区位置、模块间距、首屏排版与文档同步

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 根目录未发现 `SKILL.md`。
- 本次使用 `planning-with-files`，因为任务目标是先制定可审核的调整计划。
- 当前 PC 首页由 `src/js/pc-home.js` 渲染结构，由 `src/css/pc.css` 承载样式，并复用 `src/js/pc-welcome-banner.js` 顶部欢迎横幅。
- 就近文档落点为 `docs/UI计划/PC端/01-首页仪表盘.md`；文件导航落点为 `docs/apps-code-map.md`。
- 截图红框覆盖区域包括：首页搜索框、四张统计卡、最近使用面板、收藏分类与快速创建区域。

### 1.2 当前问题判断

- 红框内容区整体起点偏高，搜索框与顶部欢迎横幅的副标题、右侧柯基插画视觉距离不足。
- 当前关键样式为 `.pc-home-search-bar { margin: -54px 0 12px; }`，该负外边距会把搜索框大幅上拉到横幅下沿，适合做悬浮效果，但在当前截图中压缩了横幅内容呼吸感。
- 首页内部模块已经具备清晰顺序：欢迎横幅、搜索、统计、最近使用、底部双栏。本次不需要重构数据渲染和交互逻辑。
- 若只把搜索框下移而不调整后续间距，可能导致首屏底部内容被挤出更多；因此需要同步压缩或统一统计卡、最近使用、底部双栏之间的节奏。

### 1.3 调整目标

- 红框内容区整体下移，让搜索框离欢迎文案和右侧插画更从容。
- 保留搜索框轻微悬浮层级，不完全失去横幅与内容区的连接感。
- 四张统计卡、最近使用面板和底部双栏保持对齐、等距、无重叠。
- 在 1366x768 默认 PC 窗口下，首屏仍能看到搜索、统计和最近使用主体，底部双栏至少露出标题或上沿。
- 不改动首页业务行为：搜索、查看全部、最近使用跳转、收藏切换、更多菜单、分类跳转、新建和导入均保持原逻辑。

## 二、实施计划

### Phase 1：视觉定位与基准确认

- 启动或复用 Vite 预览，打开 PC 首页。
- 截图确认当前红框区域位置，重点记录搜索框顶部、统计卡顶部、最近使用面板顶部与欢迎横幅的距离。
- 对照 1366x768、1440x900、1920x1080 三个桌面视口确认当前压迫点。

### Phase 2：红框区域整体下移

- 优先调整 `src/css/pc.css` 中 `.pc-home-search-bar` 的顶部负外边距。
- 建议从 `margin: -54px 0 12px;` 调整为较轻的悬浮值，例如 `margin: -18px 0 18px;` 或 `margin: -12px 0 18px;`，以实际截图效果为准。
- 如搜索框下移后横幅和内容区断层明显，再微调 `.pc-welcome-banner-home` 的 `padding-bottom` 或背景定位，保持上下视觉连贯。

### Phase 3：内容区排版节奏优化

- 检查 `.pc-home-page` 的 `gap` 与搜索框、统计卡、最近使用面板之间的 margin 是否重复叠加。
- 统一红框区域内的纵向节奏：
  - 搜索框到统计卡：保持 18px 到 22px。
  - 统计卡到最近使用：保持 20px 到 24px。
  - 最近使用到底部双栏：保持 20px 到 24px。
- 保持统计卡四列等宽，避免下移后卡片高度或数字区域发生跳动。
- 最近使用空状态仍需居中，但面板不应因为空状态过高导致底部双栏完全不可见。

### Phase 4：响应式与首屏验收

- 在侧边栏展开态和收起态分别检查首页主内容宽度变化。
- 检查 1024px、1280px、1366px、1440px、1920px 桌面宽度。
- 重点验收：
  - 搜索框不压住欢迎副标题和插画。
  - 统计卡横向排列稳定。
  - 最近使用面板空状态不溢出、不偏移。
  - 底部双栏在默认窗口下仍能露出清晰入口。

### Phase 5：文档同步与测试记录

- 若仅调整 CSS 间距，更新 `docs/UI计划/PC端/01-首页仪表盘.md` 的尺寸和验收描述。
- 如 `src/js/pc-home.js` 结构没有变化，`docs/apps-code-map.md` 可保持不变；若新增首页内容容器或类名，则同步更新该文件。
- 运行前端基础检查，优先执行 `npm.cmd run test`。
- 完成后在本计划文档中勾选任务，并记录最终采用的间距值与视觉验收结果。

## 五、待确认问题

- 红框区域下移幅度建议先按“小幅下移”处理：搜索框仍与横幅轻微重叠，但不压迫副标题和插画。若工程师希望完全脱离横幅，可以把搜索框改为横幅下方普通内容流。
- 最近使用空状态当前高度较大。若红框下移后首屏底部双栏露出不足，建议同步降低空状态面板最小高度，而不是继续压缩顶部欢迎横幅。
