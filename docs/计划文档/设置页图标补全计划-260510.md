# 设置页图标补全计划

> 版本：v1.0.0  
> 创建日期：2026-05-10  
> 范围：PC 端设置页图标资源补全、视觉对齐与本地化接入

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现项目级 `SKILL.md`。
- 本次使用 `frontend-design`，用于图标视觉体系、对齐和动效约束。
- 本次使用 `planning-with-files`，用于沉淀计划文档和执行清单。
- 用户指定 iconfont 图标搜索页作为外部图标来源，但当前环境访问该页面超时；本轮采用本地 SVG 图标补全，目录和命名保持与后续 iconfont SVG 替换兼容。

### 1.2 当前问题

- 设置页“数据备份与恢复”操作卡和“局域网同步”区域已有彩色图标容器，但语义图标不够完整。
- 需要补齐备份、导入、导出 JSON、从 JSON 导入、同步服务等专用图标。
- 运行时不应依赖外部网络，图标必须进入 `src/assets/` 由 Vite 打包。

## 二、实施计划

### Phase 1：资源落点

- 新增 `src/assets/icons/settings/`。
- 放入设置页专用 SVG：
  - `backup.svg`
  - `import-data.svg`
  - `export-json.svg`
  - `import-json.svg`
  - `sync-server.svg`

### Phase 2：接入渲染

- 在 `src/js/pc-settings.js` 中导入新增图标。
- 替换数据备份与恢复区的通用导入/导出图标。
- 替换局域网同步区图标。
- 保留当前 `mask-image` 染色方式，继续由卡片语义色控制图标颜色。

### Phase 3：视觉校准

- 图标容器统一居中，避免图标贴边。
- 图标尺寸统一控制在 34px 到 36px。
- 图标线条使用 `currentColor` 或适配 mask 的单色填充。
- hover 不改变图标尺寸，避免布局跳动。

### Phase 4：文档与验证

- 更新 `docs/apps-code-map.md`。
- 更新 `docs/UI计划/PC端/06-设置与本地存储.md`。
- 运行 `npm run build`。
- 必要时运行 `npm run test`。
