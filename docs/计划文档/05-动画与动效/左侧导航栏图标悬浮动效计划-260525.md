# 左侧导航栏图标悬浮动效计划

> 版本：v1.0.0  
> 创建日期：2026-05-25  
> 范围：PC 端左侧侧边栏导航图标鼠标悬浮动效、点击动效协同、低动效兜底、文档同步与回归验证

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现项目级 `SKILL.md`，本轮遵循用户提供的 `AGENTS.md` 指令和仓库既有文档工作流。
- 本次使用 `planning-with-files`，用于把计划沉淀到 `docs/计划文档/` 并同步 `docs/apps-code-map.md`。
- 本次使用 `ui-ux-pro-max`，用于校准导航 hover 微交互、动效时长、低动效兜底和稳定布局约束。
- 已确认 PC 左侧导航由 `src/js/pc-app.js` 的 `NAV_ITEMS` 与 `renderShell()` 渲染，样式集中在 `src/css/pc.css` 的 `.pc-nav-item`、`.pc-nav-icon`、`.pc-nav-active`、`.pc-nav-clicking` 区域。

### 1.2 当前现状

- 当前 `.pc-nav-item:hover` 只改变按钮背景和文字颜色，`.pc-nav-item:hover .pc-nav-icon` 只改变图标颜色，缺少"鼠标靠近时图标被唤醒"的动效。
- 已完成的点击动效使用 `.pc-nav-clicking` 类名触发一次性图标回弹和按钮柔光，适合作为确认反馈。
- 悬浮动效应作为点击前的轻反馈，不能抢走点击回弹的节奏，也不能让图标持续跳动。
- 左侧导航有展开态和 84px 收起态，hover 动效必须只作用于图标层、按钮内部光感和阴影，不改变按钮尺寸、间距、文本位置。

### 1.3 交互目标

- 鼠标悬浮到导航项时，图标轻微上浮、放大、染成蓝色，并出现柔和光感。
- 悬浮保持时不使用无限跳动，避免工作台导航长期抖动造成干扰。
- 从 hover 到 click 的体验应顺滑：hover 是"预热"，click 是"确认"。
- 键盘用户通过 `focus-visible` 也能获得等价但克制的图标反馈。
- 开启系统减弱动态后，关闭位移、缩放、阴影动画，只保留颜色变化。

## 二、实施计划

### Phase 1：CSS 层悬浮动效

- 优先只改 `src/css/pc.css`，不新增 JS hover 状态。
- 在 `.pc-nav-icon` 增加更完整的 transition：
  - `background-color`
  - `transform`
  - `filter`
- 在 `.pc-nav-item:hover .pc-nav-icon` 增加轻微悬浮：
  - `transform: translateY(-2px) scale(1.08)`
  - `background-color: var(--pc-blue)`
  - 可选 `filter: drop-shadow(0 4px 8px rgba(45, 140, 255, 0.20))`
- 在 `.pc-nav-item:hover` 增加轻微阴影或背景层变化，但不使用 `padding`、`height`、`width` 的变化。
- 在 `.pc-nav-item:hover::before` 或复用伪元素时增加非常短的柔光浮现，和点击态 `.pc-nav-clicking::after` 分开命名，避免动画冲突。

### Phase 2：点击动效协同

- 保留现有 `.pc-nav-clicking` 的一次性点击回弹。
- hover 状态只做静态悬浮姿态，点击态触发关键帧时覆盖 hover transform。
- 若出现 transform 冲突，优先让 `.pc-nav-item.pc-nav-clicking .pc-nav-icon` 的动画权重高于 hover。
- 不在 JS 中新增 `mouseenter` / `mouseleave`，避免悬浮与点击状态都由脚本管理导致复杂度上升。

### Phase 3：状态兼容

- 展开态：图标上浮不挤压文字，文字只保留颜色增强。
- 收起态：图标按钮为 48px，hover 位移幅度不超过 2px，避免贴近按钮边界。
- 激活态：当前页面的蓝色图标 hover 时只增强光感，不改变选中背景结构。
- 非激活态：灰色图标 hover 后过渡到蓝色，帮助用户识别可点击状态。
- `focus-visible`：键盘聚焦时沿用图标轻微上浮或至少提供蓝色图标与外轮廓。
- `prefers-reduced-motion: reduce`：关闭 transform、filter、animation、box-shadow 过渡，只保留颜色变化。

## 五、待工程师审核

- 建议第一版只做"图标轻微上浮 + 蓝色染色 + 柔和投影"，不做循环摆动、粒子、闪光或复杂路径动画。
- hover 动效定位为点击前预热，点击动效仍负责最终确认；两者不合并成同一动画类名。
- 如果工程师希望更活泼，第二阶段可考虑为不同导航图标配置极轻的语义动作，例如首页上浮、设置轻微旋转，但第一阶段先保持统一规则，降低维护成本。
