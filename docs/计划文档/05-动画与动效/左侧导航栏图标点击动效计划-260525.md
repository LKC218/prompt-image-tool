# 左侧导航栏图标点击动效计划

> 版本：v1.0.0  
> 创建日期：2026-05-25  
> 范围：PC 端左侧侧边栏导航图标点击动效、无障碍动效兜底、文档同步与回归验证

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现项目级 `SKILL.md`，本轮遵循用户提供的 `AGENTS.md` 指令和仓库既有文档工作流。
- 本次使用 `planning-with-files`，用于把计划沉淀到 `docs/计划文档/` 并同步 `docs/apps-code-map.md`。
- 本次使用 `ui-ux-pro-max`，用于校准导航点击微交互、动效时长、减弱动态和稳定布局约束。
- 已确认 PC 左侧导航由 `src/js/pc-app.js` 的 `NAV_ITEMS` 和 `renderShell()` 渲染，样式集中在 `src/css/pc.css` 的 `.pc-nav-item`、`.pc-nav-icon`、`.pc-nav-active` 区域。

### 1.2 参考效果拆解

- 参考图 1 的 B 站顶部图标点击反馈重点是“点按后图标有轻快弹跳和强调态”，不是复杂转场。
- 图 2 的左侧导航是工作台侧边栏，视觉语气更安静，适合做短促、克制的图标动效：图标轻微上跳、缩放回弹、按钮底色短暂高亮。
- 左侧导航存在展开态和 84px 收起态两种布局，动效必须不改变按钮宽高、间距和文本布局。

### 1.3 当前现状

- 现有 `.pc-nav-item:hover` 只改变背景与颜色。
- 现有 `.pc-nav-item:active` 只有整体 `scale(0.97)`，按下时有反馈，但点击完成后没有类似 B 站图标的“释放回弹”。
- 图标本身是 `.pc-nav-icon`，通过 `mask-image` 使用本地 PNG 导航图标，适合直接对图标层做 `transform` 和 `filter`，不需要新增图片资源。
- `src/css/pc.css` 已有 `@media (prefers-reduced-motion: reduce)`，应把新增动画纳入该兜底。

## 二、实施计划

### Phase 1：交互触发层

- 在 `src/js/pc-app.js` 新增轻量函数，例如 `playNavIconClickMotion(item)`。
- 在 `setupSidebarNav()` 的点击处理内，拿到 `.pc-nav-item` 后先触发动效，再保持原有路由逻辑。
- 通过移除再添加一次性类名实现重复点击可重播，例如 `pc-nav-clicking`。
- 清理逻辑优先监听 `animationend`，避免依赖固定 `sleep` 或累计计时。
- 即使点击当前已激活导航，也允许播放一次动效，让用户获得明确反馈。

### Phase 2：动效样式层

- 在 `src/css/pc.css` 的侧边栏导航样式附近补充：
  - `.pc-nav-item` 增加 `position: relative` 和 `overflow: hidden`，用于容器内高光但不影响布局。
  - `.pc-nav-icon` 增加 `transform-origin: 50% 60%` 和 `transition: background-color ... , transform ...`。
  - `.pc-nav-item.pc-nav-clicking .pc-nav-icon` 绑定一次性关键帧。
  - `.pc-nav-item.pc-nav-clicking::after` 绑定短暂柔光或涟漪，强调“点击成功”。
- 建议关键帧节奏：
  - 0%：`translateY(0) scale(1)`。
  - 35%：`translateY(-4px) scale(1.16) rotate(-4deg)`。
  - 65%：`translateY(1px) scale(0.94) rotate(3deg)`。
  - 100%：`translateY(0) scale(1) rotate(0)`。
- 总时长控制在 260ms 到 360ms，使用 `cubic-bezier(.2,.8,.2,1)`，避免长时间干扰导航切换。

### Phase 3：状态兼容

- 展开态：图标动效不挤压文字，文字只保留现有颜色变化。
- 收起态：按钮为 48px 图标按钮，动效幅度适当收敛，避免图标贴近边界。
- 激活态：点击当前页面时仍播放图标回弹，但保持蓝色高亮态不闪烁。
- 快速连续点击：每次点击重播动画，不累积多个事件监听。
- 减弱动态：在 `@media (prefers-reduced-motion: reduce)` 中关闭新增 `animation` 和 `transform`，只保留颜色反馈。

### Phase 4：文档与验证

- 同步更新 `docs/apps-code-map.md`，登记本计划入口。
- 实施后更新就近模块说明文档，优先检查 `docs/UI计划/PC端UI设计/提示词管家_PC端UI设计复刻文档.md` 或 `docs/UI计划/PC端/` 下的 PC 端导航说明落点。
