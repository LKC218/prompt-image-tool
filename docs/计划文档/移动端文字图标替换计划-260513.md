# 移动端文字图标替换计划-260513

> 范围：移动端页面、移动端公共操作菜单、移动端样式中仍以文字、字符、Emoji 充当图标的 UI 点位。
> 目标：补齐本地 SVG/PNG 图标资源，彻底去掉移动端操作控件里的文字图标、字符图标和 Emoji 图标。

## 思考与分析

### 技能嗅探

- 已使用 `planning-with-files`：本任务需要沉淀定位结果、实施阶段和验收清单。
- 已使用 `ui-ux-pro-max`：图标治理属于移动端 UI/UX 一致性优化，需统一图标风格、尺寸和可访问语义。
- 已检查项目现有外部图标本地化规范：iconfont 可作为素材来源，但必须下载到本地、核验授权、记录来源，不能热链或接入在线字体。

### 当前判断

移动端目前已经有部分本地资源，例如底部导航、首页搜索、空状态、备份卡片和分类页快捷操作图标。但提示词卡片、详情页、编辑页、设置页确认菜单和公共 ActionSheet 仍散落使用 Emoji、箭头、星号、省略号、叉号、加号、对勾和 CSS 伪元素勾号。若只在单个页面逐点替换，后续新增页面仍容易回退，因此需要先补一个移动端图标统一出口。

### iconfont 补充入口

计划从以下页面补齐缺口图标，并按授权逐个核验：

- https://www.iconfont.cn/search/index?searchType=icon&q=%E4%BF%9D%E5%AD%98&page=1&fromCollection=-1

iconfont 页面为动态渲染页面，实际下载时需在浏览器内选择单个图标，确认作者授权、商用限制和下载协议后保存 SVG 源文件。优先只补现有通用图标库没有覆盖、且业务语义更贴近中文产品语境的图标。

## 实施计划

### 阶段一：建立移动端图标资产出口

1. 新增 `src/js/mobile-icon-assets.js`。
2. 统一导入移动端使用的本地 SVG/PNG。
3. 暴露 `mobileIcon(name, options)` 或语义化辅助函数，输出 `<img>`，默认 `alt=""`、`aria-hidden="true"`。
4. 对纯图标按钮保留 `aria-label` 和 `title`，不把可访问名称藏在图标文件名里。

### 阶段二：补齐缺失图标

1. 优先复用 `src/assets/icons/` 已有资源：`plus.svg`、`copy.svg`、`trash.svg`、`trash-2.svg`、`folder.svg`、`check.svg`、`more-horizontal.svg`、`sync.svg`、`palette.svg`、`pencil-line.svg`。
2. 可复用 PC 端已有资源：`src/assets/icons/pc/chevron-left.svg`、`chevron-right.svg`、`star.svg`、`star-filled.svg`、`x.svg`、`clipboard.svg`、`save.svg`。
3. iconfont 补充候选：保存、全屏预览、加载、文件、警告、关闭、展开收起、添加版本、版本对比。
4. 新增 SVG 必须清理为本地资源，保留 `viewBox`，统一 `currentColor` 或按 CSS 控制尺寸。
5. 在图标授权记录中写明来源、授权、用途、路径和是否二次修改。

### 阶段三：逐页面替换字符图标

1. 先替换全局高频入口：`mobile-app.js`、`mobile-utils.js`、`mobile-menu-actions.js`。
2. 再替换列表和详情：`mobile-home.js`、`mobile-library.js`、`mobile-detail.js`。
3. 再替换编辑和分类：`mobile-editor.js`、`mobile-category.js`。
4. 最后替换设置页和样式伪元素：`mobile-settings.js`、`mobile.css`。

### 阶段四：样式统一

1. 在 `src/css/mobile.css` 增加统一图标尺寸类：
   - 顶部工具按钮：20px 到 22px。
   - 列表操作按钮：18px 到 20px。
   - ActionSheet 图标：22px 到 24px。
   - 卡片大图标：24px 到 32px。
2. hover/active/focus 只改变颜色、背景、阴影或透明度，不改变图标尺寸。
3. 所有图标容器设置稳定宽高，避免替换后移动端布局跳动。

### 阶段五：文档与验证

1. 同步更新 `docs/apps-code-map.md`。
2. 同步更新 `docs/技术文档/mobile-technical-doc.md` 中移动端资源和图标约定。
3. 如新增 iconfont 资源，同步更新或新增移动端图标授权记录。
4. 执行静态扫描，确认移动端源码中不再存在 Emoji、箭头、星号、省略号、叉号、对勾等作为图标的字符。
5. 运行前端测试和构建验证。
6. 打开 `http://127.0.0.1:5173/?ui=mobile` 做首页、库、详情、编辑、分类、设置页视觉回归。

## 验收标准

- 移动端页面源码不再用 Emoji、单字符箭头、星号、省略号、叉号、对勾、加号作为 UI 图标。
- 纯图标按钮都有 `aria-label` 或 `title`。
- 图标资源全部本地化，不依赖 iconfont 在线字体、远程脚本或 CDN。
- 新增外部图标有授权记录。
- 移动端首页、提示词库、详情、编辑、分类、设置页在 320px 到桌面预览宽度下无图标错位、挤压和布局跳动。
