# 聊天归档脚本 prompt-image-tool JSON 首段跳过代码块计划

> 状态：已实施，待回归观察真实 ChatGPT 页面导出结果。

## 背景

当前 `prompt-image-tool` 专用 JSON 导出已能把 ChatGPT 对话导入到新建提示词页，并自动预填标题、正向提示词、负向提示词和图片。

现有取词规则按“对话首段”抽取正向提示词，但在真实对话里，首段可能是代码块、结构化参数块或其它中间产物，如截图所示的 `content_type / asset_pointer / metadata`。这会把不该进入提示词库的调试/结构化文本一起保存进去。

本计划的目标是：导入或导出时继续保留真正的提示词正文，但明确跳过首段代码块和结构化元数据，并保留后续完整自然语言正文。

## 总体目标

1. 正向提示词不再包含首段代码块内容。
2. 若首段是代码块或类 JSON 块，则跳过该段，保留后续完整多段自然语言正文。
3. 保持现有专用 JSON schema、图片导入、标题、负向提示词和普通备份导入逻辑不变。
4. 只调整提取规则，不改文件名、不改导入入口、不改普通 JSON 恢复行为。

## 规则建议

建议把提取规则收敛为以下优先级：

1. 先按段落切分文本。
2. 跳过首段中的 fenced code block、纯 JSON 块、明显的结构化元数据块。
3. 从第一个符合“自然语言正文”特征的段落开始，保留后续所有有效自然语言段落作为 `positivePrompt`。
4. 如果后续段落中有明确 `正向提示词` / `Positive Prompt` 字段，再优先按字段抽取。
5. `negativePrompt` 维持现有显式字段提取规则，没有字段则留空。
