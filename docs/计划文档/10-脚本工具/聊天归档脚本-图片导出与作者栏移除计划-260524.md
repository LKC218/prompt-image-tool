# 聊天归档脚本图片导出与作者栏移除计划

## 背景

用户脚本 `gpt本地档案馆工具/ChatGPT Conversation Vault - CheeseTa Integrated-2026.05.24.10.user.js` 当前支持保存当前对话、备份官方历史、导出本地库、导入归档文件和复制续聊上下文。原导出流程只覆盖对话文本、原始响应、Markdown、HTML 和 ZIP 清单；附件扫描停留在元数据层，不能把当前页面已经生成并显示的图片一并写入导出包。

界面面板标题下方展示了作者、反馈和昵称信息。该区域属于作者展示 UI，不影响核心归档能力，本次计划移除该展示区域，保持标题、按钮、设置、本地历史和导入导出功能不变。

## 目标

1. `导出当前` 功能在保持原 ZIP 结构和文本导出能力不变的前提下，额外附带当前对话中已生成并可访问的图片文件。
2. 导出的 Markdown 和 HTML 尽量引用 ZIP 内的本地图片相对路径，解压后可离线查看文本与图片。
3. 图片下载失败时不阻断当前对话导出，导出包内记录失败原因和原始线索。
4. 移除工具可视化 UI 中红框位置的作者展示区域，保留工具标题和主操作区。
5. 不改变自动保存、官方历史备份、本地库导出、导入、搜索、复制续聊和清空本地库等现有行为。

## 非目标

1. 不重构整个用户脚本结构。
2. 不引入外部 ZIP、Markdown 或 HTML 依赖库。
3. 不在第一阶段强制支持历史批量备份中的所有图片下载。
4. 不破解或绕过 ChatGPT 图片权限；只在当前登录态和浏览器允许的能力范围内下载。
5. 不保存用户密钥或持久化 Access Token。

## 实现思路

### 图片发现

优先从当前页面 DOM 发现已经渲染的图片，因为“导出当前对话”发生在当前 ChatGPT 页面中，可直接扫描当前对话上下文里的图片元素和下载链接。

候选来源：

- 当前对话区域内的 `img[src]`。
- `picture source[srcset]` 或图片容器中的高分辨率候选地址。
- 图片下载按钮附近的链接地址。
- 对话原始响应 `raw.mapping` 中的图片线索，作为 DOM 发现失败后的补充。

第一版需要避免误抓头像、站点图标、按钮图标和页面装饰图。建议通过尺寸、所在消息区域、URL 类型和文件 MIME 进行过滤。

### 图片下载

新增图片资源收集函数，返回统一清单：

```js
{
  id,
  source,
  url,
  localPath,
  mimeType,
  extension,
  status,
  error
}
```

下载策略：

- `blob:` 地址优先用当前页面 `fetch` 读取，因为离开页面后可能失效。
- 同源或允许跨域的 `https:` 图片直接 `fetch` 为 Blob。
- 如果普通 `fetch` 被跨域限制，再评估是否增加 `GM_xmlhttpRequest`、`@grant GM_xmlhttpRequest` 和必要的 `@connect` 配置。
- 每张图片独立失败，不影响 ZIP 主导出。

### ZIP 结构

保持现有导出文件不变，追加图片目录和清单：

```text
current/
  对话标题_对话ID_时间.json
  对话标题_对话ID_时间.md
  对话标题_对话ID_时间.html
  assets/
    image-001.png
    image-002.webp
    image-manifest.json
manifest.json
```

`image-manifest.json` 记录图片下载结果、原始 URL、来源、MIME、文件名和失败原因。

### Markdown 和 HTML 引用

当前 `conversationToMarkdown()` 与 `conversationToHtml()` 只接收对话对象。实施时可增加可选参数，例如 `assets`，在不传参时保持现有行为。

建议签名：

```js
conversationToMarkdown(item, options = {})
conversationToHtml(item, options = {})
```

当前导出 ZIP 时传入图片清单，让导出的 Markdown/HTML 增加图片区块。单条历史记录的 `MD` / `HTML` 按钮可以先保持原行为，避免扩大改动面。

### UI 移除

作者展示区域来自面板头部的副标题/子信息 DOM 和对应 CSS。计划只移除红框位置的作者、反馈、昵称展示，不改：

- 面板标题 `ChatGPT 本地档案馆`
- 左侧下载图标
- 关闭按钮
- 主操作按钮
- 设置、搜索、本地历史列表

如需保留内部元信息，可只在代码常量里保留 `APP.author`，用于导出文件元数据，不在 UI 中展示。

## 实施状态

已在 `gpt本地档案馆工具/ChatGPT Conversation Vault - CheeseTa Integrated-2026.05.24.10.user.js` 中完成当前对话图片导出和作者展示区域移除。当前实现仅扩展 `导出当前` 链路；官方历史备份、本地库导出、历史单条 MD/HTML 下载保持原行为。

终端侧已通过 Node `vm.Script` 语法解析检查。真实 ChatGPT 页面中的图片资源下载受页面 DOM、图片 CDN 跨域策略和用户脚本运行环境影响，仍需在浏览器里安装脚本后做最终交互验证。

## 验收标准

1. 无图片对话执行 `导出当前` 后，ZIP 内容与原能力等价，不出现报错。
2. 含已生成图片的当前对话执行 `导出当前` 后，ZIP 内存在 `current/assets/image-manifest.json`。
3. 可下载图片以稳定文件名写入 `current/assets/`，Markdown/HTML 能引用相对路径。
4. 单张图片失败时，ZIP 仍可生成，清单中记录失败状态和原因。
5. 面板顶部不再展示作者、反馈、昵称信息，其他按钮和设置项位置稳定。
6. `保存当前`、`备份官方历史`、`导出本地库`、`导入文件`、搜索和本地历史操作保持可用。

## 风险与验证重点

- ChatGPT 页面 DOM 和内部接口可能变化，图片发现逻辑需要宽容失败。
- 图片 CDN 可能存在跨域限制，必要时再增加用户脚本跨域请求权限。
- `blob:` 图片必须在当前页面生命周期内读取，不能依赖后续历史恢复。
- 页面中头像、上传参考图、生成图可能混在一起，需要通过上下文和尺寸过滤降低误抓率。
- 导出 HTML 中插入图片时必须继续使用 HTML 转义和安全链接策略。
