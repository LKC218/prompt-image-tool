# 移动端提示词详情页布局优化计划-260513

## 1. 目标与边界

本次仅优化移动端提示词详情页，不改 PC 端、不改后端、不改数据结构。

目标是修复详情页正向/负向提示词卡片头部在窄屏下的挤压问题，重点对应截图 2 中红框区域：标题、字数统计和复制按钮同处一行时，标题文字与按钮相互抢占空间，导致视觉拥挤。

## 2. 当前定位

涉及文件：

- `src/js/mobile-detail.js`
  - 第 151-164 行附近渲染正向/负向提示词卡片。
  - 当前标题、字数统计和复制按钮都在 `.m-prompt-section-header` 内。
  - 字数统计目前以内联样式嵌在 `.m-prompt-section-title` 中。
- `src/css/mobile.css`
  - 第 1002-1058 行附近定义 `.m-prompt-section`、`.m-prompt-section-header`、`.m-prompt-section-title`、`.m-prompt-copy-btn` 和正文样式。
  - `.m-prompt-section-header` 当前为单行 `flex` + `space-between`。
  - `.m-prompt-section-title` 使用 20px 标题字级，在 360-460px 窄屏上与复制按钮容易挤压。

当前已验证：

- `node --check src/js/mobile-detail.js` 通过。
- `npm.cmd run test` 通过，8 个测试文件、86 个用例全通过。
- `npm.cmd run build` 通过。
- 曾尝试 `npm.cmd run test -- --runInBand`，失败原因是 Vitest 不支持该参数，不属于代码失败。

## 3. 问题根因

根因不是数据或渲染异常，而是移动端卡片头部缺少抗挤压布局：

- 标题“正向提示词（Positive）”本身较长。
- 字数统计追加在标题同一行，进一步拉长标题区域。
- 复制按钮使用固定胶囊按钮，且需要保留触控尺寸。
- 父容器单行排布，没有允许标题换行或拆分信息层级。

## 4. 实施方案

### 4.1 结构调整

在 `src/js/mobile-detail.js` 中只调整移动端详情页提示词卡片头部结构：

- 将标题文本和字数统计拆成独立元素。
- 新增语义类名，例如：
  - `.m-prompt-title-wrap`
  - `.m-prompt-count`
- 保持复制按钮仍在头部右侧，交互逻辑和按钮 `id` 不变。

推荐结构：

```html
<div class="m-prompt-section-header">
  <div class="m-prompt-title-wrap">
    <span class="m-prompt-section-title">正向提示词（Positive）</span>
    <span class="m-prompt-count">2000 字</span>
  </div>
  <button class="m-prompt-copy-btn">复制</button>
</div>
```

### 4.2 样式调整

在 `src/css/mobile.css` 中优化移动端提示词卡片：

- `.m-prompt-section-header` 保持横向布局，但允许标题区域压缩。
- `.m-prompt-title-wrap` 设置 `min-width: 0`、纵向或自适应排布，避免撑爆。
- `.m-prompt-section-title` 允许在窄屏换行，必要时降低行高。
- `.m-prompt-count` 改为独立弱化文本，不再挤在标题字面后。
- `.m-prompt-copy-btn` 设置 `flex-shrink: 0` 和稳定最小触控宽高。
- 对极窄屏补充媒体查询：
  - 卡片内边距适度收紧。
  - 标题字级从 20px 降到 18px。
  - 复制按钮保持可点击，不被压缩成畸形。

### 4.3 视觉目标

- 360px 宽度下不出现标题、字数和复制按钮互相挤压。
- 标题可自然换行，但按钮尺寸稳定。
- 正文首行与卡片头部保持清晰间距，不被红框区域压住。
- 正向和负向卡片使用同一套布局规则。
- 底部固定操作栏不遮挡详情页最后内容。
