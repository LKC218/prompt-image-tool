# PC端各页面顶部栏走路动画计划

> 版本：v1.1.0  
> 创建日期：2026-05-25  
> 范围：PC 端顶部栏区域，将首页像素柯基走路序列帧扩展为可复用顶部栏动画，并补充提示词详情页顶部红框区域

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现项目级 `SKILL.md`，本轮遵循用户提供的 `AGENTS.md` 指令、仓库文档同步规范和既有计划文档格式。
- 本次使用 `planning-with-files`，用于把计划沉淀到 `docs/计划文档/` 并同步 `docs/apps-code-map.md`。
- 本次使用 `ui-ux-pro-max`，用于校准跨页面动画、响应式稳定性、连续动效克制原则和 `prefers-reduced-motion` 兜底。
- 已确认 PC 端顶部欢迎横幅由 `src/js/pc-welcome-banner.js` 的 `renderPcWelcomeBanner()` 统一渲染。
- 当前已使用共享横幅的页面包括：首页、提示词库、新建/编辑、分类与标签、设置页。
- 详情页当前不是共享欢迎横幅结构，而是独立详情页顶部区域；v1.1 按用户红框位置补充详情页专属装饰层，不把详情页改成共享欢迎横幅。
- 已有首页动画资源为 `src/assets/pc/home-pixel-dog-sprite.png`，透明背景 8 帧横向精灵图，总尺寸 576px × 108px，每帧 72px × 108px。
- 已有首页实现已经在 `src/js/pc-welcome-banner.js` 增加 `decorationsHtml` 插槽，并在 `src/css/pc.css` 中落地 `.pc-welcome-pixel-*` 动画样式。

### 1.2 当前现状

- 首页顶部横幅已经可以显示像素柯基走路动画，但动画 DOM 目前由 `src/js/pc-home.js` 单独传入。
- 提示词库、分类与标签、设置页同样使用共享欢迎横幅，但未传入动画。
- 新建/编辑页顶部横幅右侧还有返回、保存和清空等操作区域，动画需要避开按钮，不宜照搬首页的水平轨道。
- 共享横幅的背景、标题、副标题、右侧静态柯基插画已经跨页面统一，适合作为动画扩展入口。
- 连续动画如果在所有页面同时过强，会干扰工作台效率；因此动画需要低透明度、慢速、避让核心内容，并在窄宽和低动效模式下降级。

### 1.3 设计目标

- 让各 PC 页面顶部栏都有统一的“像素柯基走路”氛围动效，形成跨页面品牌一致性。
- 不增加新的图片依赖，不复制多套资源，优先复用 `home-pixel-dog-sprite.png`。
- 动画只出现在顶部横幅安全区域，不遮挡标题、副标题、右侧插画、编辑页操作按钮和页面主内容。
- 保持每个页面的工作效率：动画小、慢、轻，不做大面积穿屏、不做粒子、不做复杂视差。
- 低动效模式下停止序列帧和横向移动，窄宽下直接隐藏，避免挤压布局。

## 二、实施计划

### Phase 1：范围归类与页面策略

- 将页面分为三类：
  - 标准共享横幅：`pc-home.js`、`pc-library.js`、`pc-category.js`、`pc-settings.js`。
  - 操作型共享横幅：`pc-editor.js`，右侧有返回、保存、清空等操作，需要更保守的动画轨道。
  - 非共享横幅：`pc-detail.js`，v1.1 通过 `.pc-detail-walk-decoration` 接入顶部红框区域走路动画，不改变详情页顶部结构。
- 首页继续保留现有动画，但将渲染函数从 `pc-home.js` 提升到共享模块或轻量工具函数，避免其他页面重复拼 DOM。
- 为不同页面定义动画轨道规则：
  - 首页：保持当前中部轨道。
  - 提示词库：标题与右侧柯基之间轻量走动，位置略靠中。
  - 分类与标签：轨道略短，避免页签和管理区域视觉抢占。
  - 设置：轨道更靠右侧空白，不干扰“本地数据”语义。
  - 新建/编辑：默认隐藏或缩短轨道，只在按钮区左侧安全距离充足时展示。

### Phase 2：共享动画渲染入口

- 在 `src/js/pc-welcome-banner.js` 中新增可复用渲染辅助函数，例如：
  - `renderPcWelcomeWalkAnimation(options = {})`
- 该函数统一输出：
  - `.pc-welcome-pixel-stage`
  - `.pc-welcome-pixel-runner`
  - `.pc-welcome-pixel-sprite`
- 支持传入轻量配置：
  - `variant`：`home`、`library`、`category`、`settings`、`editor`
  - `hiddenOnCompact`：窄宽时隐藏
  - `aria-hidden="true"`：保持装饰语义
- `renderPcWelcomeBanner()` 保留 `decorationsHtml` 插槽，不把动画强行写死进所有横幅，避免未来页面需要禁用时难以控制。

### Phase 3：页面接入

- 修改标准共享横幅页面：
  - `src/js/pc-home.js`：改为调用共享 `renderPcWelcomeWalkAnimation({ variant: 'home' })`。
  - `src/js/pc-library.js`：传入 `variant: 'library'`。
  - `src/js/pc-category.js`：传入 `variant: 'category'`。
  - `src/js/pc-settings.js`：传入 `variant: 'settings'`。
- 修改操作型共享横幅页面：
  - `src/js/pc-editor.js`：先按安全策略传入 `variant: 'editor'`，默认在宽度不足时隐藏。
- 详情页接入：
  - `src/js/pc-detail.js`：在浅蓝天空顶部区域增加 `pc-detail-walk-decoration`，传入 `variant: 'detail'`。
- 页面接入后补充测试，确认动画只出现在目标共享横幅页面，不影响详情页。

### Phase 4：CSS 轨道与响应式

- 在 `src/css/pc.css` 中保留现有 `.pc-welcome-pixel-*` 基础样式，新增页面变体：
  - `.pc-welcome-pixel-stage-home`
  - `.pc-welcome-pixel-stage-library`
  - `.pc-welcome-pixel-stage-category`
  - `.pc-welcome-pixel-stage-settings`
  - `.pc-welcome-pixel-stage-editor`
  - `.pc-welcome-pixel-stage-detail`
- 不同变体只控制轨道位置、宽度、透明度和是否显示，不复制关键帧。
- 动画基础参数建议：
  - 帧动画：`0.72s steps(8) infinite`
  - 横向移动：首页约 `6.4s`，其他页面可略慢到 `7.2s`
  - 透明度：`0.80` 到 `0.92`
  - 资源尺寸：保持每帧 `72px × 108px`
- 响应式策略：
  - 主内容区宽度充足时展示。
  - 约 `1180px` 以下统一隐藏。
  - 编辑页可在更宽阈值下隐藏，例如 `1280px` 以下隐藏。
  - 不改变 `.pc-welcome-section` 的 Flex 布局，不新增会撑开横幅的真实占位。

### Phase 5：低动效与无障碍

- 动画容器统一设置 `pointer-events: none`，不阻挡横幅按钮。
- 动画容器统一设置 `aria-hidden="true"`，不进入辅助技术阅读顺序。
- `prefers-reduced-motion: reduce` 下：
  - 关闭 `pc-pixel-dog-cross` 横向移动。
  - 关闭 `pc-pixel-dog-frames` 序列帧。
  - 保留静态第一帧或直接隐藏，优先不影响页面阅读。
- 不引入 JS 定时器，不使用累计 `sleep` 计时；全部使用 CSS 动画和媒体查询控制。

### Phase 6：测试、构建与预览

- 新增或扩展 `src/js/pc-home-pixel-animation.test.js`：
  - 检查共享渲染函数存在。
  - 检查首页、提示词库、分类、设置、编辑页均传入对应 `variant`。
  - 检查详情页未误接入。
  - 检查 CSS 存在各页面变体、`steps(8)`、低动效兜底。
- 验证命令：
  - `npm.cmd run test -- pc-home-pixel-animation.test.js`
  - `npm.cmd run test`
  - `npm.cmd run build`
  - `python scripts/start_dev_server.py --no-open`
- 预览验收：
  - PC 首页：`http://127.0.0.1:5173/?ui=pc`
  - 手动切换提示词库、新建/编辑、分类与标签、设置页。
  - 通过 Edge 无头截图或人工预览确认动画不遮挡标题、按钮、右侧插画和主内容。

## 五、实现建议

- 第一版已覆盖共享顶部横幅页面；v1.1 补充详情页红框区域，但仍不把详情页纳入共享横幅结构重构。
- 动画轨道建议短而轻，不让像素柯基横穿整条横幅，以免每个页面都产生过强视觉噪声。
- 各页面只通过 `variant` 调整轨道，不复制 HTML 或关键帧。
- 如果后续要移动端也增加顶部走路动画，应另立“双端顶部动画”计划，因为移动端顶部空间和触控区约束完全不同。
