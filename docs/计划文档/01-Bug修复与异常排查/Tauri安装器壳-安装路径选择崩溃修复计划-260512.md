# Tauri安装器壳-安装路径选择崩溃修复计划

## 思考与分析

### Skill嗅探结果

- 本任务明确要求“制定计划”，已使用 `planning-with-files` 技能，将分析、实施路径和验证清单落到持久化 Markdown 文档。
- 当前任务属于 `installer-shell/` 安装器壳的窄范围稳定性修复，不改变 NSIS 安装核心，不改安装向导步骤结构，不引入前端路由。
- 既有架构约束继续保持：`installer-shell/src/index.html` 是单 HTML 壳体，页面内容由 `installer-shell/src/pages/*.js` 输出，`installer-shell/src/main.js` 管理步骤状态和 Tauri 命令调用。

### 问题判断

本次反馈集中在安装位置页：

1. 点击“浏览”选择安装路径时安装程序崩溃。
2. 即使弹出选择行为，安装路径也无法稳定选择成功。
3. 修复后必须做真实窗口和真实安装链路验证，不能只看编译通过。

当前代码中，`installer-shell/src-tauri/src/main.rs` 的 `choose_install_dir()` 通过隐藏 PowerShell 进程启动 `System.Windows.Forms.FolderBrowserDialog`。这条链路存在几个高风险点：

- Tauri release 窗口是无控制台子系统，外部 PowerShell 对话框没有明确父窗口，焦点、置顶和退出状态都不稳定。
- `creation_flags(0x08000000)` 会隐藏 PowerShell 控制台，但对 WinForms 目录选择对话框的生命周期和前台焦点不够可靠。
- 选择结果通过 stdout 传回 Rust，中文路径、取消选择、异常退出和编码都可能导致前端拿不到稳定结果。
- 前端选择成功后会 `renderCurrentStep()` 再触发路径校验，若返回路径为空、异常或校验覆盖状态，用户会感觉“选择没有成功”。

因此本轮优先从根因修复：不要继续依赖 PowerShell 弹目录窗口，改为 Tauri 内部目录选择能力；同时加固前端状态更新和验证链路。

## 实施计划

### 1. 复现与日志基线

- 启动当前 `releases/PromptImageManager-Shell-Setup-2.3.1.exe`，进入安装位置页。
- 点击“浏览”，记录是否出现崩溃、无响应、窗口丢焦点或错误提示。
- 若进程退出，检查 Windows 事件日志中与 `PromptImageManager-Shell-Setup-2.3.1.exe`、`powershell.exe` 相关的错误摘要。
- 用一个安全测试目录作为验证目标，例如 `%USERPROFILE%\Downloads\PIM-Test-PathSelect-Verify`，避免污染真实安装目录。

### 2. 替换目录选择实现

- 在 `installer-shell/src-tauri/Cargo.toml` 中引入 Tauri 官方目录对话框能力，优先使用 `tauri-plugin-dialog`。
- 在 `installer-shell/src-tauri/src/main.rs` 中注册对话框插件，并把 `choose_install_dir()` 从 PowerShell WinForms 实现改为 Tauri 内部目录选择实现。
- 继续保留前端调用入口 `invoke("choose_install_dir", { currentDir })`，避免扩大前端改动面。
- 目录选择初始位置规则保持不变：
  - 当前输入路径可解析时，优先定位到该路径或最近存在父目录。
  - 当前输入路径不可解析时，回退到下载目录。
  - 用户取消选择时返回 `None`，前端保持当前路径并提示“已取消目录选择”。
- 返回给前端的路径必须是真实本地路径，不返回 `%USERPROFILE%` 这类占位符。

### 3. 加固前端选择成功链路

- 检查 `installer-shell/src/main.js` 的 `chooseInstallDirectory()`：
  - 选中目录后先写入 `state.installPath`。
  - 同步 `state.resolvedInstallPath`。
  - 重新渲染后必须执行 `validateInstallPath({ renderOnSuccess: false })`。
  - 校验成功后页面输入框、实际路径、提示条都必须显示选中的真实路径。
- 防止浏览按钮重复点击：
  - 目录选择进行中禁用“浏览”和“下一步”。
  - 成功、取消或失败后恢复按钮。
- 对异常情况给出明确中文提示，不允许安装器直接崩溃或静默失败。

### 4. 路径校验与安装传参确认

- 保持 `resolve_target_dir()` 对 `%USERPROFILE%`、`%LOCALAPPDATA%` 的解析能力。
- 对真实路径继续执行 `validate_install_path()` 可写性检查。
- 验证“下一步”和“准备安装页”展示的是选择后的目录。
- 验证 `install_with_nsis()` 收到的是选择后的真实路径，并最终通过 `/D=<真实路径>` 传给 NSIS。

## 风险与边界

- 本轮只修复安装路径选择崩溃和选择不成功，不重构整个安装流程。
- 本轮不改变 NSIS 安装核心和静默安装参数。
- 若引入 `tauri-plugin-dialog` 需要新增 Cargo 依赖，必须以最小权限接入，不扩大无关能力。
- 真实安装验证只使用下载目录下的测试子目录，避免覆盖用户真实安装目录。
