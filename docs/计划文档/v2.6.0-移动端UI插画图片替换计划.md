## 一、背景与现状

### 1.1 当前问题

移动端 UI 中所有插画位置均使用 **Emoji 字符**（🐕、🐰）替代实际插画图片，与设计规范中定义的「柯基吉祥物插画」和「兔子辅助角色插画」不符，缺乏品牌辨识度和视觉精致感。

| 界面 | 当前实现 | 设计规范要求 | 代码位置 |

## 二、任务执行顺序

按照 **依赖关系** 和 **影响范围** 合理排列执行顺序：

```
Step 1: ~~人工审核插画图片~~ ✅ 已确认映射
    ↓
Step 2: 创建资源目录 + 复制/重命名插画文件
    ↓
Step 3: 替换首页插画（最高频页面，优先处理）
    ↓

Step 7: 构建验证 + 真机测试
```

### 排序理由

1. ~~**人工审核优先**~~ ✅ 已确认：`首页.png` → 首页，`设置页.png` → 设置页，`分类与标签页.png` → 分类与标签页

4. **CSS 适配后置**：三个界面的样式调整可合并处理，避免反复修改
5. **构建验证收尾**：确保所有替换在构建产物中正确生效

### Step 6: CSS 样式适配

**检查项**：

| 检查项 | 说明 |
|--------|------|
| 插画尺寸 | 确保各界面插画尺寸与周围元素协调 |
| 圆角 | 插画是否需要圆角处理（建议首页/设置页插画加 `border-radius: var(--m-radius-md)`） |
| 阴影 | 插画是否需要轻阴影（建议 `--m-shadow-sm`） |
| 深色模式 | 插画在深色模式下是否可见（如有深色背景需调整） |
| 响应式 | 不同屏幕宽度下插画是否正常显示 |
| 加载态 | 图片加载中/加载失败的 fallback 处理 |

**深色模式适配建议**：

```css
[data-theme="dark"] .mobile-app .m-welcome-mascot,
[data-theme="dark"] .mobile-app .m-mascot-img {
    filter: brightness(0.95);
}
```

## 四、风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 插画图片与设计规范不匹配 | 视觉效果不一致 | Step 1 人工审核严格对照规范 |
| 图片文件过大影响加载速度 | 移动端性能下降 | 控制单张图片 < 100KB，使用 PNG 压缩 |
| 图片路径在构建后不正确 | 插画无法显示 | 验证 Vite 构建配置的静态资源处理 |
| 深色模式下插画不可见 | 用户体验差 | 添加深色模式适配 CSS |
| Emoji 到 IMG 的替换影响布局 | 界面错位 | 仔细调整 CSS 尺寸和对齐方式 |

## 六、底部导航栏图标替换实施详情

### 6.1 图标资源映射

| 源文件 | 目标路径 | 导航项 |
|--------|---------|--------|
| `UI设计稿/图标/首页.png` | `src/assets/mobile/nav-icons/home.png` | 首页 |
| `UI设计稿/图标/提示词库.png` | `src/assets/mobile/nav-icons/library.png` | 提示词库 |
| `UI设计稿/图标/分类.png` | `src/assets/mobile/nav-icons/category.png` | 分类 |
| `UI设计稿/图标/设置.png` | `src/assets/mobile/nav-icons/settings.png` | 设置 |

### 6.2 点击反馈颜色变化机制（已优化为 CSS mask 方案）

| 状态 | 图标效果 | 文字颜色 |
|------|---------|---------|
| 默认（未选中） | `background-color: var(--m-text3)` — 灰色 | `var(--m-text3)` |
| 选中 - 首页/分类/设置 | `background-color: var(--m-blue)` — 品牌蓝 | `var(--m-blue)` |
| 选中 - 提示词库 | `background-color: var(--m-pink)` — 品牌粉 | `var(--m-pink)` |
| 点击（:active） | `background-color: var(--m-blue)` — 即时变色 + `transform: scale(0.92)` | — |

### 6.3 CSS 关键实现（CSS mask 方案）

```css
/* 图标使用 CSS mask 实现精确颜色控制 */
.m-nav-icon {
    width: 24px;
    height: 24px;
    background-color: var(--m-text3);
    -webkit-mask-size: contain;
    mask-size: contain;
    -webkit-mask-repeat: no-repeat;
    mask-repeat: no-repeat;
    -webkit-mask-position: center;
    mask-position: center;
    transition: background-color var(--m-transition-fast);
}

/* 选中：品牌蓝色 */
.m-nav-active .m-nav-icon {
    background-color: var(--m-blue);
}

/* 选中：品牌粉色 */
.m-nav-active-pink .m-nav-icon {
    background-color: var(--m-pink);
}

/* 点击反馈：即时变色 + 缩放 */
.m-nav-item:active .m-nav-icon {
    background-color: var(--m-blue);
}

.m-nav-item:active {
    transform: scale(0.92);
}
```

### 6.4 技术选型说明

**为什么选择 CSS mask 而非 CSS filter？**

| 对比项 | CSS filter（旧方案） | CSS mask（新方案） |
|--------|---------------------|-------------------|
| 颜色精度 | ❌ 无法精确控制颜色，只能灰化/恢复原色 | ✅ 通过 `background-color` 精确控制任意颜色 |
| 选中态 | ❌ 恢复图标原色，与品牌色不一致 | ✅ 精确匹配品牌蓝 `--m-blue` 和品牌粉 `--m-pink` |
| 点击变色 | ❌ 只能做透明度变化 | ✅ 即时变为品牌蓝色 |
| 深色模式 | ❌ 需要额外 filter 适配 | ✅ 自动跟随 CSS 变量 |
| 过渡动画 | ❌ filter 过渡不流畅 | ✅ background-color 过渡平滑 |
