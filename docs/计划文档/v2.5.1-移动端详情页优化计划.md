## 二、各任务详细分析与实施方案

### R1: 图片点击放大二级预览窗口

**当前状态**：上一轮已实现 `showImageViewer()` 全屏图片查看弹窗，但存在以下问题：

1. **CSS 样式可能未生效**：`m-image-viewer-overlay` 的 `background` 初始为 `rgba(0,0,0,0)`，通过 `.m-image-viewer-show` 过渡到 `rgba(0,0,0,0.92)`，但 `requestAnimationFrame` 可能太快导致过渡动画未触发
2. **封面图点击事件绑定位置**：点击事件在 `setupContentEvents()` 中绑定，但该函数在 `renderDetail()` 末尾调用，而 `renderDetail()` 是异步函数，可能存在时序问题
3. **缩略图区域仅在图片>1张时显示**：单张图片时无法点击放大

**修复方案**：

1. 修复 CSS 过渡动画：改用 `setTimeout` 替代 `requestAnimationFrame`，确保浏览器完成首次渲染后再添加 show 类
2. 单张图片也支持点击放大：移除 `imageUrls.length > 1` 的缩略图限制，单张图片时封面图仍可点击查看
3. 优化图片查看器 UI：
   - 增加双指缩放支持（移动端）
   - 优化关闭按钮位置（适配安全区域）
   - 增加图片加载中状态

**涉及文件**：
- `src/js/mobile-detail.js` — `showImageViewer()` 函数、`setupContentEvents()` 函数
- `src/css/mobile.css` — 图片查看器样式优化

### R2: 修复提示词收藏功能

**当前状态分析**：

存储层已完整实现：
- `sqlite-storage.js`：`is_favorite` 列 + `toggleFavorite()` 方法 ✅
- `api-storage.js`：`toggleFavorite()` 方法 ✅
- `python/main.py`：`/api/prompt-set/{id}/toggle-favorite` API ✅

UI层已实现：
- 详情页星标按钮：读取 `set.isFavorite` 初始化 + 调用 `storage.toggleFavorite()` ✅
- 提示词库星标按钮：调用 `storage.toggleFavorite()` ✅
- 首页星标按钮：调用 `storage.toggleFavorite()` ✅

**可能的问题**：

1. **PC端（浏览器预览）走 ApiStorage**：需要 Python 后端运行才能正常工作。如果后端未启动，`toggleFavorite()` 调用会失败
2. **`getPromptSets()` 返回数据缺少 `isFavorite`**：检查 `api-storage.js` 的 `getPromptSets()` 是否正确传递后端返回的 `isFavorite` 字段
3. **提示词库列表渲染未读取 `isFavorite`**：`renderList()` 中 `item.isFavorite` 可能为 `undefined`（如果 `getPromptSets()` 未返回该字段）

**修复方案**：

1. 检查 `api-storage.js` 的 `getPromptSets()` 是否正确透传 `isFavorite` 字段
2. 在 `renderList()` 中增加 `isFavorite` 的 fallback：`item.isFavorite ?? false`
3. 收藏筛选 `currentFilter === 'favorite'` 时增加容错：`p.isFavorite === true`
4. 确保后端未启动时收藏操作给出明确提示

**涉及文件**：
- `src/js/api-storage.js` — 确认 `getPromptSets()` 返回 `isFavorite`
- `src/js/mobile-library.js` — 收藏筛选容错
- `src/js/mobile-home.js` — 收藏状态容错

### R3: 正反向提示词过长截断 + 查看详细按钮

**当前状态**：

[mobile-detail.js:152-160](file:///d:/Work/git/prompt-image-tool/src/js/mobile-detail.js#L152-L160) 中，正反向提示词直接完整显示在 `.m-prompt-text` 中，没有任何截断：

```html
<div class="m-prompt-text">${positivePrompt || '...'}</div>
<div class="m-prompt-text">${negativePrompt || '...'}</div>
```

CSS 中 `.m-prompt-text` 也没有 `max-height` 或 `overflow` 限制，长提示词会占据大量屏幕空间。

**修复方案**：

1. **CSS 截断**：为 `.m-prompt-text` 添加 `max-height` 和 `overflow: hidden`，超过3行时截断
2. **展开/收起按钮**：在提示词区域底部添加"查看完整提示词"按钮，点击后展开显示全部内容
3. **二级弹窗查看**：点击"查看详细"按钮时，弹出底部弹窗（ActionSheet 或自定义弹窗），全屏显示提示词内容，支持复制
4. **字数统计**：在提示词标题旁显示字数，如"正向提示词（128字）"

**涉及文件**：
- `src/js/mobile-detail.js` — 提示词渲染逻辑、展开/收起事件、详细查看弹窗
- `src/css/mobile.css` — 提示词截断样式、展开按钮样式
