## 一、决策摘要

当前不对 `05-pages-overlays.css` 进行第二轮文件拆分。

该文件约 7,000 余行，包含桌面端业务页面、共享欢迎横幅、全局弹层、深色模式、响应式和后置视觉补丁。继续按页面语义立即重排会触及同名选择器的级联顺序，风险高于短期维护收益。

本计划采用“冻结结构、约束增量、记录覆盖、满足条件后无行为拆分”的渐进治理方式：

1. 当前版本保持文件结构和加载顺序不变。
2. 后续功能样式按既有区段落位，禁止无序追加到文件末尾。
3. 对跨页面覆盖与后置补丁建立最小记录。
4. 达到明确触发阈值后，才进行第二轮连续物理拆分。
5. 文件拆分与选择器去重、视觉优化、主题重构必须分为独立批次。

```mermaid
flowchart LR
    A[05 页面与弹层样式] --> B[当前冻结结构]
    B --> C[新增规则按既有区段落位]
    C --> D[记录后置覆盖与回归范围]
    D --> E{满足拆分触发条件}
    E -->|否| C
    E -->|是| F[连续物理拆分]
    F --> G[字节比对与构建测试]
    G --> H[独立进行补丁收敛]

    style B fill:#c8e6c9,color:#1a5e20
    style E fill:#fff3e0,color:#e65100
    style F fill:#bbdefb,color:#0d47a1
    style H fill:#f3e5f5,color:#7b1fa2
```

## 二、现状与问题边界

### 2.1 当前职责

`05-pages-overlays.css` 当前承载以下能力：

- 首页快捷创建、最近内容、详情页片段、编辑器、分类和标签页面。
- 共享欢迎横幅及与页面正文的相邻关系布局。
- Toast、通用 Modal、更新记录、右键菜单、图片预览等运行时动态挂载组件。
- 提示词库、分类、首页和详情页的多轮视觉实现与后置补丁。
- 部分深色模式、媒体查询、容器查询和抗挤压规则。

`06-responsive-overrides.css` 是整个 PC 样式链的最终覆盖层，仍可能覆盖 `05-pages-overlays.css` 中的页面与弹层规则。

### 2.2 已识别的结构风险

| 风险 | 具体表现 | 当前风险等级 |
|---|---|---|
| 同名选择器多轮覆盖 | 分类、快捷操作、页面卡片等存在基础规则和后置新拟态规则 | 中高 |
| 深色规则分散 | 图片预览、页面面板和欢迎横幅的深色适配不都位于基础规则附近 | 中等 |
| 响应式规则分散 | 页面局部断点、容器查询与最终抗挤压规则跨越多个区段 | 中等 |
| 全局弹层依赖 | Toast、Modal、右键菜单和图片预览不能按路由延迟加载 | 中等 |
| 定位与协作成本 | 单文件较大，新增规则易被追加到末尾并形成新补丁 | 中高 |

### 2.3 不在本计划内

- 不拆分 `05-pages-overlays.css`。
- 不调整 `pc.css` 的导入顺序。
- 不合并、删除或重命名选择器。
- 不修改颜色、尺寸、动效、断点、主题 Token 或页面视觉设计。
- 不将全局弹层改为页面级按需 CSS 加载。

## 三、治理目标

- 在不产生样式行为变化的前提下，阻止 `05-pages-overlays.css` 无序增长。
- 降低新增页面样式误触后置覆盖、深色适配和响应式规则的概率。
- 将后续第二轮拆分的触发条件、文件边界和验证方式提前固定。
- 保持 `pc.css` 作为桌面端唯一稳定入口，保持 `06-responsive-overrides.css` 的最终覆盖位置。

## 四、当前维护规则

### 4.1 新增规则落位

新增 CSS 必须先定位既有同类规则并插入相邻区段，不得默认追加到文件末尾。

| 变更类型 | 首选落点 | 必查覆盖 |
|---|---|---|
| 首页、详情、编辑器、分类页面基础样式 | 对应页面已有基础区段 | 后续页面补丁、深色模式、局部断点 |
| 欢迎横幅与页面顶部布局 | 共享欢迎横幅区段 | 相邻兄弟选择器、页面变体、减少动态效果 |
| Toast、Modal、右键菜单、图片预览 | 全局弹层区段 | `z-index`、焦点、动画和深色模式 |
| 提示词库基础样式 | 提示词库主体区段 | 后续布局压缩、容器查询和 `06` 覆盖 |
| 必须覆盖旧实现的规则 | 最接近的既有后置补丁区段 | 被覆盖选择器、覆盖原因、浅色/深色/窄窗口表现 |

### 4.2 后置覆盖最小记录

新增后置覆盖时，在紧邻规则前以现有文件注释风格标记以下信息：

- 覆盖目标选择器。
- 触发场景，例如深色模式、窄窗口、侧栏收起或页面变体。
- 不可移动原因，例如必须晚于某基础规则或兼容历史 DOM。

不为历史规则补写大规模注释；仅约束本计划生效后的新增覆盖，避免引入无关格式噪音。

### 4.3 禁止事项

- 禁止为“整洁”目的将跨段选择器合并为单一定义。
- 禁止将深色模式、低动效或媒体查询集中迁移到文件某一处。
- 禁止改变 Toast、图片预览、右键菜单、Modal 的加载方式或层级顺序。
- 禁止绕过 `pc.css` 直接在业务 JavaScript 中导入 `05` 或未来子文件。
- 禁止在未核对后置覆盖的情况下删除“看似重复”的样式规则。

## 五、回归要求

每次改动下列区域时，必须执行对应最小回归。

| 改动区域 | 必测页面或能力 | 必测场景 |
|---|---|---|
| 分类、标签、快捷操作 | 分类页、创建弹窗、颜色选择 | 浅色、深色、窄窗口、键盘焦点 |
| 编辑器、比例、分段控件 | 编辑器、新建与编辑状态 | 窄窗口、减少动态效果 |
| 提示词库、筛选、预览 | 列表、预览、分页、右键菜单 | 容器宽度变化、深色、焦点状态 |
| 欢迎横幅 | 首页、提示词库、编辑器、分类、设置 | 页面相邻间距、深色、减少动态效果 |
| Toast、Modal、更新记录 | 动态反馈、确认弹窗、更新记录 | 层级、焦点归还、Escape、深色 |
| 图片预览 | 预览打开、缩放、关闭、下载入口 | 深色、窄窗口、与 Toast/Modal 的层级 |

所有涉及 CSS 文本断言的结构变动，必须同步验证 `src/js/pc-css-test-utils.js` 的读取顺序与 `pc.css` 入口顺序一致。

## 六、拆分触发条件

满足任一条件后，创建独立实施任务并执行第二轮无行为拆分：

| 编号 | 触发条件 | 阈值或证据 |
|---|---|---|
| T1 | 文件规模持续增长 | 超过 9,000 行 |
| T2 | 样式回归重复出现 | 连续两个迭代发生跨页面覆盖、深色或响应式回归 |
| T3 | 协作冲突影响交付 | 同一文件在连续两个迭代出现合并冲突 |
| T4 | 选择器覆盖失控 | 一个核心选择器出现超过三轮定义，且无法在一次审查中判断最终来源 |
| T5 | 新增大型页面 | 单次预计新增 800 行以上页面或弹层样式 |

未满足触发条件时，继续执行本计划的维护规则，不提前进行结构搬迁。

## 七、触发后的第二轮拆分预案

### 7.1 目标文件

触发后将 `05-pages-overlays.css` 替换为下列连续区段文件：

```text
src/css/pc/
├─ 05a-legacy-page-primitives.css
├─ 05b-category-base.css
├─ 05c-global-overlays.css
├─ 05d-welcome-library-base.css
└─ 05e-page-late-overrides.css
```

### 7.2 顺序契约

`pc.css` 的加载顺序固定为：

```text
01 → 02 → 03 → 04 → 05a → 05b → 05c → 05d → 05e → 06
```

`06-responsive-overrides.css` 必须始终最后导入。

### 7.3 连续边界

| 文件 | 原 `05-pages-overlays.css` 行段 | 内容 |
|---|---:|---|
| `05a-legacy-page-primitives.css` | 1–564 | 旧通用页面片段、编辑器基础、颜色选择器和分类弹窗基础 |
| `05b-category-base.css` | 565–1115 | 分类与标签初始实现及紧邻媒体查询、低动效规则 |
| `05c-global-overlays.css` | 1116–1890 | Toast、Modal、更新记录、右键菜单、图片预览及其动画/低动效规则 |
| `05d-welcome-library-base.css` | 1891–3169 | 共享欢迎横幅、编辑器/提示词库基础实现及局部断点 |
| `05e-page-late-overrides.css` | 3170–文件末尾 | 文件夹弹窗、首页/详情/分类后续实现、深色、响应式和抗挤压补丁 |

边界以源文件当前内容为基准；实施前必须重新核对行号，只能在完整规则之间切分。

### 7.4 实施安全闸门

1. 记录拆分前的文件字节内容、行数、构建与测试结果。
2. 仅按连续原始行段剪切，不复制、不删除、不格式化、不重排。
3. 拼接 `05a` 至 `05e` 的内容，与拆分前 `05-pages-overlays.css` 逐字节比对。
4. 更新 `pc.css` 与 `pc-css-test-utils.js`，二者顺序必须一致。
5. 执行全量测试、生产构建和第五节规定的人工视觉回归。
6. 只有回归稳定后，才能在另一个独立计划中开始选择器去重或补丁收敛。
