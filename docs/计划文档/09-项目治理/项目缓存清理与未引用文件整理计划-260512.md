### 清理判断原则

1. 只删除可再生成、无业务含义、无交付价值的缓存和构建输出。
2. 不默认删除源码、计划文档、测试记录、设计原稿和发布产物。
3. 对未跟踪文件先做引用审计：文件名、目录名、运行时路径、文档路径至少查一次。
4. 对 `build/` 只清理被忽略的输出和日志，保留受 Git 跟踪的 `build/app.spec`、`build/app_main.py`、`build/icon.ico`、`build/installer.nsi`。
5. 对 `installer-shell/preview` 分层处理：浏览器 profile 和日志可删；截图证据需先核对测试记录是否引用。
6. 清理动作若改变目录结构或忽略规则，必须同步 `docs/apps-code-map.md` 与最近模块文档。

## 实施计划

### 阶段 1：建立清理基线

- 记录清理前 `git status --short --ignored`。
- 记录未跟踪文件清单与忽略文件清单。
- 记录重点目录体积，作为清理后对比。
- 确认当前没有需要保留的运行中服务占用临时目录。

### 阶段 2：先清理低风险缓存

优先清理这些可再生成内容：

- `.codex-logs/`
- 根目录 `vite-*.log`、`vite-*.err.log`
- `.pytest_cache/`
- `**/__pycache__/` 与 `*.pyc`
- `build/app.log`
- `build/stderr.log`
- `build/debug_output.txt`
- `build/build/`
- `build/dist/`
- `build/installer-shell/`
- `dist/`
- `android/.gradle/`
- `android/build/`
- `android/app/build/`
- `installer-shell/src-tauri/target/`

暂不默认清理这些内容：

- `.venv/`：本地 Python 环境，删除会影响继续开发。
- `node_modules/`、`installer-shell/node_modules/`：依赖目录，体积较大但会影响本地运行速度。
- `android-sdk/`：本机 Android SDK，删除后打包链路会中断。
- `python/data/`：含用户本地数据和同步状态，不能按缓存处理。
- `releases/`：发布交付产物，虽然被忽略，但不能作为普通缓存删除。

### 阶段 3：审计未跟踪内容

对以下内容执行引用审计，再决定保留、删除、移动或加入忽略规则：

- `.vscode/`：默认不纳入提交，可直接加入忽略或删除本地配置。
- `installer-shell/preview/`：按 `chrome-profile*`、`edge-metrics-profile*`、日志、截图证据拆分处理。
- `UI设计稿/安装壳UI/`：当前设计规则文档明确说明原始设计稿目录，需保留为设计来源，不直接作为运行时资源引用。
- `UI设计稿/图标/打勾勾.png`：需检查是否已被运行时资源替代，若无引用且无计划引用再删除。
- 新增 `docs/计划文档/` 与 `docs/测试记录/`：按安装器壳任务链保留，避免丢失实施闭环。
- `scripts/build_installer_shell_package.py`：已被 `scripts/README.md` 和计划文档引用，应保留。

### 阶段 4：补齐忽略规则

清理后检查 `.gitignore` 是否覆盖新增缓存源，优先考虑补充：

```gitignore
.vscode/
installer-shell/preview/
```

截图证据默认不提交到仓库；需要复核时本地重新生成。

### 阶段 5：验证与收口

- 重新执行 `git status --short --ignored`，确认变更项显著减少。
- 执行 `git diff --check`，确认文档和忽略规则没有格式问题。
- 若仅删除缓存，不需要运行完整功能测试。
- 若 `.gitignore`、文档或脚本被调整，执行对应轻量验证：
  - `node --check installer-shell/src/main.js`
  - `Get-ChildItem installer-shell/src/pages -Filter *.js | ForEach-Object { node --check $_.FullName }`
  - `cargo check --manifest-path installer-shell/src-tauri/Cargo.toml`
- 同步更新 `docs/apps-code-map.md` 和最近模块文档。
