## Phase 0：依赖安全修复（P0，最高优先）

**目标**：消除 `npm audit` 报告的 2 个 high + 2 个 moderate 漏洞，重点修复 vite 8.0.10 高危区间。

- 步骤：
  1. `npm audit fix`（或手动升级 `vite` 到 `>=8.0.16`、随带 `undici`/`brace-expansion`/`tar` 补丁）。
  2. 确认 `package.json` 与 `package-lock.json` 中 vite 版本已升到补丁版。
  3. 重跑 `npm run build` 与 `npm run test`，确认无回归。
  4. 重跑 `npm audit` 确认无 high/moderate。
- 目标文件：`package.json`、`package-lock.json`（仅版本号变更，无逻辑改动）。
- 预期收益：关闭 Windows 平台 vite `launch-editor` NTLM 泄露与 `server.fs.deny` 绕过等高危漏洞。
- 风险：补丁升级可能微调构建行为；需在多平台（Windows/Android）回归。
- 验证：`npm audit` 无 high/moderate；`npm run build` 成功；`npm run test` 129+ 用例通过；Android 构建不受影响。

## Phase 1：图片资产优化（P1，体积优先）

**目标**：将 `dist` 从 ~9.7 MB 降至 ~2–3 MB，显著提升 Web 首屏与 Android APK 体积。

- 步骤：
  1. 扫描 `src/assets` 中 >200 KB 的 PNG（tip-mascot、detail-image-placeholder、home-folder、保存、home-welcome-bg、tag、greeting-hand、data、empty-mailbox 等）。
  2. 压缩/转格式：PNG 用 `pngquant`/sharp 有损压缩；静态插画转 **WebP/AVIF**（体积可降 60–80%）。
  3. 构建期配置：`vite.config.js` 增加图片资源压缩（或 `vite-plugin-imagemin`），大图按需懒加载。
  4. 兼容性：确认 Capacitor Android WebView 与 Tauri WebView 支持 WebP/AVIF；若不支持则保留 PNG 降级或仅做 PNG 有损压缩。
  5. 移除 `src/assets` 中未被引用的冗余图片。
- 目标文件：`src/assets/*.png`、`vite.config.js`、相关 `<img>`/CSS 背景引用、Android/Tauri 资源引用。
- 预期收益：dist 体积下降 60–80%，APK/IPA 同步受益。
- 风险：格式转换可能带来视觉质量损失；WebP/AVIF 在老旧 WebView 兼容性需确认。
- 验证：重测 `dist` 总体积（目标 <3 MB）；视觉回归（PC/移动端关键插画页）；`npm run build` 通过。

## Phase 2：抽取共享模块（P2，可维护性）

**目标**：消除 PC/移动端约 4800–5800 行平台无关重复，下沉到 `src/js/shared/`，两端退化为平台适配壳。

- 步骤（按模块分批，每批独立验证）：
  1. 新建 `src/js/shared/`，先落地低风险纯函数模块：`formatBytes`、`generateId`、`escapeHtml`、`escapeAttr`、`formatRelativeTime`（合并自 `pc-utils.js` 与 `mobile-*.js`）。
  2. `shared/router-core.js`：合并 `pc-router.js`/`mobile-router.js`（差异仅 `view` 标记）。
  3. `shared/icon-assets.js`：合并 `pc-icon-assets.js`/`mobile-icon-assets.js`。
  4. `shared/import-core.js`：统一 `handleBackupImport`/`handleChatGptVaultImport`/`getJsonImportErrorMessage`/`readJsonFile`/`openJsonImportPicker`。
  5. `shared/list-render.js`：统一 `getFilteredItems`/`renderList`/`getTags`/缩略图懒加载。
  6. `shared/menu-actions.js`：统一 `getPromptSetMenuItems`/`showRenameDialog`/`showMoveDialog`/`handleCopy`/`showDeleteConfirm`（UI 外壳差异以回调注入）。
  7. `shared/editor-core.js`：统一 `applyPromptImageToolImport`/`buildPromptToolImportedImages`/图片校验。
  8. 两端 `pc-*.js`/`mobile-*.js` 改为 `import` 共享层，仅保留平台外壳与事件绑定；删除已下沉的重复实现。
- 目标文件：新增 `src/js/shared/*`；修改 `pc-editor.js`/`mobile-editor.js`、`pc-library.js`/`mobile-library.js`、`pc-settings.js`/`mobile-settings.js`、`pc-router.js`/`mobile-router.js`、`pc-menu-actions.js`/`mobile-menu-actions.js`、`pc-home.js`/`mobile-home.js`、`pc-detail.js`/`mobile-detail.js`、`pc-utils.js`/`mobile-utils.js`、`pc-icon-assets.js`/`mobile-icon-assets.js`。
- 预期收益：减少 ~5000 行重复；修复两端行为漂移（如导入分流不一致）；新功能只需改一处。
- 风险：抽取时易引入回归；函数签名差异需逐一核对；测试需同步更新 import 路径。
- 验证：每批完成后 `npm run test` 通过；PC 与移动端关键路径回归（编辑器保存、导入、列表、菜单动作、路由切换）；`npm run build` 通过。

## Phase 3：版本稳定性与 CI（P3）

**目标**：避免意外升级，保证可复现构建。

- 步骤：
  1. `package.json` 生产依赖去掉 `^`，锁定精确版本（构建/测试工具可保留 `^` 或锁定）。
  2. CI 流水线增加：`npm ci`（强制 lock 一致）→ `npm audit`（失败阈值 high）→ `npm run build` → `npm run test:all`。
  3. 评估 `png-to-ico`/`sharp` 是否改为 `optionalDependencies`（仅桌面/Android 构建需要）。
- 目标文件：`package.json`、CI 配置（如 `.github/workflows/*` 或对应构建脚本）。
- 预期收益：本地与 CI 构建可复现；杜绝 `^` 浮动引入的意外破坏性升级。
- 风险：去 `^` 后升级需手动改版本，发布流程略重；需团队共识。
- 验证：CI 在干净环境 `npm ci` 成功；`npm audit` 绿灯；构建产物一致。

## Phase 4（可选）：构建配置增强

- `vite.config.js` 增加 `build.rollupOptions.output.manualChunks` 分包（PC/Mobile/共享层分离，利于缓存）。
- 开启 brotli 预压缩（`vite-plugin-compression` 或构建后压缩）。
- 控制 sourcemap 生成（发布构建关闭，避免泄露源码）。
- 说明：当前 JS 体积极小，Phase 4 收益有限，建议在 Phase 1 图片优化落地后视实际再决定。
