# 项目优化诊断报告-260707

> 创建日期：2026-07-07
> 项目：prompt-image-tool v2.3.6（跨平台：Windows Tauri / Android Capacitor / Web）
> 范围：方向 A（PC/移动端重复代码去重）+ 方向 B（依赖与构建稳定性）
> 方法：静态代码分析 + 依赖审计（`npm audit`）+ 构建产物测量。本轮**零源码改动**，仅产出诊断结论。
> 交付：本文档（诊断）+ `项目优化计划-260707.md`（后续实施蓝图）

---

## 1. 概述

本轮诊断聚焦两个方向，结论摘要如下：

- **方向 A（去重）**：`src/js` 中 PC 端与移动端存在 11 对成对模块，合计约 **10580 行**，估算 **45%–55%（约 4800–5800 行）为平台无关逻辑**，当前散落在两端重复实现，是明确的技术债。
- **方向 B（依赖与构建）**：
  - 依赖版本整体已为稳定主版本（vite8 / ts6 / vitest4 / capacitor8 / tauri2），但 **vite 8.0.10 命中 8.0.0–8.0.15 高危漏洞区间**（2 个 high + 2 个 moderate），需升补丁版而非降级。
  - 构建产物 **9.7 MB**，瓶颈几乎全在**静态 PNG 图片（约 9.5 MB）**，JS/CSS 体积极小（gzip 后 <100 KB）。
- 综合优先级：**P0 安全修复 → P1 图片体积 → P2 去重 → P3 版本稳定性**。

### 2.2 平台无关 vs 平台相关拆分（要点）

- **app**：无关=路由注册、`createPage` 骨架、`handleRouteChange`；相关=PC 侧栏/存储环动画 vs Mobile 底部导航/返回键/Capacitor。
- **editor**：无关=导入字段映射、图片大小/类型校验、保存时版本组装、`escapeAttr`；相关=PC 模态/侧栏布局 vs Mobile actionSheet/直读 DOM。
- **library**：无关=过滤/搜索/排序、`getTags` 解析、缩略图懒加载、收藏切换；相关=PC 表格+预览分栏+分页 vs Mobile 卡片列表。
- **category**：无关=`loadCategoryData`、`renameTag`/`clearTagFromAll`/`clearAllTags`；相关=PC 拖拽排序/彩色弹窗 vs Mobile 对话框/原生 `prompt()`。
- **settings**：无关=备份导入导出流程、ChatGPT Vault 导入、`getJsonImportErrorMessage`；相关=PC 网络/同步展示 vs Mobile 局域网扫描/进度。
- **router**：整体均可共享，仅 `view` 标记与栈导出差异。
- **menu-actions**：整体均可共享，仅弹窗 vs actionSheet 的 UI 外壳不同。

- **icon-assets**：`pcIcon`/`mobileIcon` 同构，仅图标映射与 class 命名不同。

## 3. 方向 B：依赖与构建稳定性诊断

### 3.1 依赖漏洞扫描（`npm audit`）

当前 `package-lock.json` 实测存在 **4 个漏洞（2 high / 2 moderate）**：

| 包 | 区间 | 严重度 | 漏洞 | 修复 |
| --- | --- | --- | --- | --- |
| **vite** | 8.0.0 – 8.0.15 | **high** | launch-editor NTLMv2 哈希泄露（UNC 路径）；`server.fs.deny` Windows 备用路径绕过 | `npm audit fix`（升到 >=8.0.16） |
| **undici** | 7.0.0 – 7.27.2 | **high** | TLS 证书校验绕过（SOCKS5）、Set-Cookie 注入、WebSocket DoS 等 7 项 | 随 vite 升级修复 |
| **brace-expansion** | 5.0.2 – 5.0.5 | moderate | 大数值范围绕过 `max` DoS 防护 | `npm audit fix` |
| **tar** | <=7.5.15 | moderate | PAX size 覆盖导致 tar 解析差分（文件走私） | `npm audit fix` |

> **结论**：虽然 vite 8 是稳定主版本，但 **8.0.10 命中高危区间**，属于"稳定版但有已知 CVE"。修复方式应为**升级补丁版本（>=8.0.16）**，而非降级大版本。

### 3.2 版本稳定性结论

- `vite@8.0.10`、`typescript@6.0.3`、`vitest@4.1.5`、`@capacitor/core@8.3.1`、`@tauri-apps/cli@2.11.0` 在 2026-07 均为已发布的**稳定主版本发行版**，"锁定稳定版"目标基本达成。
- 但存在上游 CVE（见 3.1），需将 vite 升到补丁版。**不要盲目降级**主版本。
- 建议：生产依赖固定**精确版本**（去掉 `^` 浮动）；CI 增加 `npm ci` + `npm audit` 校验，避免本地 `^` 解析引入意外升级。

### 3.3 构建产物体积基线（`vite build` 实测）

- 总体积：**9.7 MB（10,183,299 字节）/ 47 文件**，构建耗时 **277 ms**。
- 关键发现：**JS 与 CSS 极小**：
  - `pc-app.js` 154 KB（gzip 47.78 KB）、`mobile-app.js` 144.81 KB（gzip 35.59 KB）
  - `pc-app.css` 149 KB（gzip 24.17 KB）、`mobile-app.css` 72.46 KB（gzip 10.33 KB）
  - 其余 JS 模块均在 20 KB 以下（gzip <6 KB）。
- **体积瓶颈在静态 PNG 图片**：约 9 个 800 KB–1.6 MB 的插画/占位图合计约 9.5 MB：

| 图片 | 大小 |
| --- | --- |
| tip-mascot.png | 1.57 MB |
| detail-image-placeholder.png | 1.09 MB |
| home-folder.png | 1.04 MB |
| 保存.png | 1.02 MB |
| home-welcome-bg.png | 950 KB |
| tag.png | 924 KB |
| greeting-hand.png | 875 KB |
| data.png | 845 KB |
| empty-mailbox.png | 810 KB |

> 优化低垂果实明确：**图片资产压缩/转 WebP/AVIF**，预计可将 dist 从 ~9.7 MB 降至 2–3 MB（降幅 60–80%），对 Web 首次加载与 Android APK 体积收益显著。JS 分包/压缩收益极小（已足够精简），不应作为重点。

### 3.4 构建配置与依赖健康

- `vite.config.js` 未显式配置 `build.minify`（默认 esbuild，已开启）、未配置 `rollup` 分包/`manualChunks`、未开启 brotli 预压缩。对当前体积极小影响有限，可在图片优化后视需补充。
- `devDependencies` 含 `png-to-ico`、`sharp`（构建期图标处理）。属构建期依赖，合理；可评估是否改为 `optionalDependencies` 以减小 `npm install` 体积。
- **lock 一致性未校验**：未见 CI 中 `npm ci` 强制锁文件一致。建议加入，防止 `package-lock.json` 与 `node_modules` 漂移。

## 4. 综合结论与优化优先级

| 优先级 | 方向 | 事项 | 预期收益 |
| --- | --- | --- | --- |
| **P0** | B-安全 | `npm audit fix` 升 vite>=8.0.16 等，消除 2 high + 2 moderate | 关闭 Windows 平台 vite 高危漏洞 |
| **P1** | B-体积 | 图片资产压缩/转 WebP/AVIF + 懒加载 | dist 从 ~9.7 MB 降至 ~2–3 MB |
| **P2** | A-去重 | 抽取 `src/js/shared/` 层，下沉 ~5000 行平台无关逻辑 | 降低维护成本，修复两端行为漂移 |
| **P3** | B-稳定性 | 去 `^` 锁精确版本 + CI `npm ci`/`npm audit` 校验 | 可复现构建，避免意外升级 |

> 详细实施步骤、目标文件、风险与验证方式见 `项目优化计划-260707.md`。
