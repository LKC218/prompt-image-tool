# 仓库优化整理计划

## 背景

当前项目本地目录体积约 4.85GB，其中主要占用来自可再生成的构建缓存：

- `src-tauri/target` 约 3.96GB
- `node_modules` 约 264MB
- `android/app/build` 约 253MB
- `build/dist` 约 38MB
- `android/.gradle` 约 19.6MB

Git 历史对象约 144MB，当前跟踪文件约 61.92MB。历史大对象主要来自 APK、ZIP 安装包和设计稿图片。

## 目标

- 清理本地可再生成缓存，降低工作目录体积。
- 完善忽略规则，防止构建产物、安装包和运行时数据继续进入 Git。
- 将已跟踪的安装包和运行时数据从 Git 索引移除，同时保留本地文件。
- 补齐项目入口说明和目录职责说明，降低后续维护成本。

## 后续可选项

如需继续压缩远端仓库体积，可单独制定 Git 历史瘦身方案，使用历史重写工具清理旧 APK、ZIP 对象。该操作需要强推，会影响协作者，必须单独确认后执行。
