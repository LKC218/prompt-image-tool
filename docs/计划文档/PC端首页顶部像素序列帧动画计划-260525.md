# PC端首页顶部像素序列帧动画计划

> 版本：v1.0.0  
> 创建日期：2026-05-25  
> 范围：PC 端首页顶部欢迎横幅红框区域，基于工程师提供的像素柯基序列帧图增加轻量循环动画

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现项目级 `SKILL.md`，本轮遵循用户提供的 `AGENTS.md` 指令、仓库文档同步规范和既有计划文档格式。
- 本次使用 `planning-with-files`，用于把计划沉淀到 `docs/计划文档/` 并同步 `docs/apps-code-map.md`。
- 本次使用 `ui-ux-pro-max`，用于校准动画、响应式稳定性和 `prefers-reduced-motion` 可访问性约束。
- 已确认红框区域属于 PC 端首页顶部欢迎横幅，结构由 `src/js/pc-home.js` 调用 `src/js/pc-welcome-banner.js` 渲染。
- 已确认横幅样式集中在 `src/css/pc.css`，首页专属类名为 `.pc-welcome-banner-home`，右侧原有柯基插画为 `.pc-welcome-mascot`。
- 已确认当前项目为 Vite + 原生 JavaScript + 原生 CSS，不需要引入动画库；序列帧动画适合用本地 PNG 资源加 CSS `steps()` 完成。

### 1.2 当前现状

- 首页欢迎横幅左侧是标题和副标题，右侧是静态柯基插画，中间红框区域目前为空白装饰区。
- `renderPcWelcomeBanner()` 是共享横幅函数，首页、提示词库、新建编辑、分类与标签、设置页都在复用，因此不能把首页专属动画硬编码到所有页面。
- 首页横幅高度约 176px，红框区域可承载一条 80px 到 110px 高的横向像素动画，但不能压住标题、副标题、搜索框或右侧插画。
- 工程师提供的图 2 是横向序列帧图，视觉上约 8 帧，适合处理成单张横向精灵图后用 `background-position` 播放。
- 需要先确认图 2 PNG 是否真的带透明通道；如果棋盘格是图片内容而不是透明背景，需要先生成透明版本，否则放到浅蓝横幅上会出现灰白格。

### 1.3 设计目标

- 在图 1 红框区域增加一只像素柯基从左到右轻快走动的装饰动画，强化首页的可爱、创作陪伴感。
- 动画只作为首页氛围装饰，不抢搜索、统计卡和最近使用列表的注意力。
- 序列帧保持像素质感，使用 `image-rendering: pixelated`，不做模糊缩放。
- 动画区域使用绝对定位和固定安全边界，不改变横幅原有 Flex 布局，不造成首屏内容位移。
- 开启系统减弱动态后停止循环动画，保留静态第一帧或直接弱化隐藏。

## 二、实施计划

### Phase 1：素材验收与运行时资源落点

- 检查工程师提供的图片：
  - 路径：`C:\Users\Administrator\Downloads\ChatGPT Image 2026年5月25日 01_40_15.png`
  - 确认尺寸、帧数、透明通道和有效角色边界。
- 将可运行资源复制或导出到 `src/assets/pc/`，建议命名：
  - `src/assets/pc/home-pixel-dog-sprite.png`
- 实施时将运行时精灵图压缩为 576px × 108px，每帧 72px × 108px，避免首页装饰资源过大。
- 如果原图带透明通道：
  - 使用 `sharp` 或图像处理脚本裁剪出角色序列帧有效区域。
  - 输出等宽等高的横向精灵图，例如 `8 × 96px` 或 `8 × 112px`。
- 如果原图不带透明通道：
  - 先生成透明背景版本，再进入裁剪和等宽帧处理。
  - 不把带棋盘格背景的原图直接作为运行时资源。
- 素材处理脚本可临时放在 `scripts/`，若只是一次性处理可不接入运行时；如后续还会反复替换序列帧，再沉淀为脚本说明。

### Phase 2：欢迎横幅结构扩展

- 修改 `src/js/pc-welcome-banner.js`，给 `renderPcWelcomeBanner(options)` 增加一个通用装饰插槽，例如：
  - `decorationsHtml`
  - 或 `centerHtml`
- 装饰插槽插入 `.pc-welcome-section` 内，并由调用方决定是否传入。
- 默认值为空，确保提示词库、新建编辑、分类与标签、设置页视觉不变。
- 修改 `src/js/pc-home.js`，仅首页传入像素动画结构：
  - 外层：`.pc-welcome-pixel-stage`
  - 运动层：`.pc-welcome-pixel-runner`
  - 帧动画层：`.pc-welcome-pixel-sprite`
- 像素动画元素设置 `aria-hidden="true"`，不进入键盘焦点，不影响辅助技术阅读顺序。

### Phase 3：CSS 序列帧与路径动画

- 在 `src/css/pc.css` 的首页欢迎横幅区域增加专属样式：
  - `.pc-welcome-banner-home .pc-welcome-pixel-stage`
  - `.pc-welcome-pixel-runner`
  - `.pc-welcome-pixel-sprite`
- 红框区域定位建议：
  - `position: absolute`
  - 左侧避开标题区，右侧避开原有静态柯基插画。
  - 顶部位于横幅中上区域，约 34px 到 48px。
  - 高度约 96px 到 112px。
- 帧动画建议：
  - `.pc-welcome-pixel-sprite` 使用 `background-image: url(...)`。
  - `background-size` 按帧数拉满，例如 8 帧为 `800% 100%`。
  - `animation: pc-pixel-dog-frames 0.72s steps(8) infinite`。
  - `@keyframes pc-pixel-dog-frames` 只改变 `background-position-x`。
- 横向移动建议：
  - `.pc-welcome-pixel-runner` 在 stage 内从左侧外缘移动到右侧外缘。
  - `animation: pc-pixel-dog-cross 6s linear infinite`。
  - 运动速度保持慢节奏，避免和页面操作抢注意力。
- 分层原因：
  - sprite 内层负责换帧。
  - runner 外层负责位移。
  - 避免同一个元素同时抢占 `transform` 或 `background-position` 导致后续维护困难。

### Phase 4：响应式与低动效兜底

- 主内容宽度不足时隐藏动画或缩短 stage，优先保护标题、搜索框和右侧插画。
- 建议在现有 PC 容器查询或媒体查询中加入兜底：
  - 宽度充足：展示完整横向走动。
  - 宽度紧张：隐藏像素动画，只保留原欢迎横幅。
- 开启 `prefers-reduced-motion: reduce` 时：
  - 关闭 `animation`。
  - 让 `.pc-welcome-pixel-runner` 静止在红框中心，或将整个 stage 隐藏。
  - 不播放循环位移、缩放、投影或关键帧。
- 动画层设置 `pointer-events: none`，不影响搜索框、导航、横幅按钮等交互。

## 五、实现建议

- 第一版建议只做一只像素柯基横向移动，不叠加粒子、扫光、弹跳文字或复杂视差。
- 序列帧帧率建议控制在 10fps 到 14fps 左右，路径移动周期建议 5.5s 到 7s。
- 像素角色高度建议约 84px 到 104px，过大会压迫横幅文字，过小会像噪点。
- 如果图 2 后续需要替换为更干净的透明帧，保持输出文件名不变即可减少代码改动。
