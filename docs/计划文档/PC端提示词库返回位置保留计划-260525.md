# PC端提示词库返回位置保留计划-260525

> 范围：PC 端提示词库页面从详情页返回时，保留上次查看的页码、选中项和列表滚动位置。

## 问题定位

当前提示词库页在 `src/js/pc-library.js` 里使用模块级状态保存 `currentPage`、`pageSize`、`selectedPromptId`，但在 `unmount()` 时会直接重置：

- `selectedPromptId = ''`
- `currentPage = 1`
- `pageSize = 20`

同时，表格主体是独立滚动容器 `.pc-library-table-scroll`，所以“恢复页码”和“恢复滚动位置”是两个独立问题。

## 目标

1. 从提示词库进入详情页后，再返回时恢复上次的页码。
2. 恢复列表里上次查看的选中项。
3. 恢复提示词库表格滚动位置。
4. 不影响搜索、筛选、收藏、右键菜单和双击进入详情的现有行为。

## 实施思路

### 1. 增加提示词库视图快照

在 `src/js/pc-library.js` 内新增一个轻量状态对象，用于记录：

- 当前筛选条件
- 当前搜索关键字
- 当前页码
- 每页数量
- 当前选中提示词 ID
- 表格滚动位置 `scrollTop` / `scrollLeft`

### 2. 离开页面前先保存

在以下时机保存快照：

- 点击进入详情页前
- `unmount()` 执行时

这样无论是正常跳转还是浏览器历史返回，都能拿到最近一次状态。

### 3. 返回页面后再恢复

在 `mount()` 和 `renderLibraryContent()` 渲染完成后恢复状态：

- 先恢复 `currentPage`、`pageSize`、`selectedPromptId`
- 再在 `requestAnimationFrame()` 后恢复滚动位置

避免 DOM 还没渲染完就回填滚动，导致位置失效。

### 4. 收敛重置逻辑

`unmount()` 不再无条件清空全部列表状态，只保留真正需要清理的临时数据。

### 5. 补最小测试

补一组围绕状态保存/恢复的测试，重点验证：

- 进入详情前记录页码和滚动位置
- 返回后恢复到同一页
- 返回后表格滚动位置不被重置
- 搜索和筛选状态不丢失

## 风险点

- 如果用户在详情页之外切换了筛选条件，返回时需要确认是“恢复原状态”还是“保留最新筛选”；当前计划优先恢复离开时的原状态。
- 滚动容器必须在内容渲染后再回填，否则会出现恢复失败。
