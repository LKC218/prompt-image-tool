# 提示词库列表区排版视觉一致性优化计划-260510

> 范围：PC 端提示词库红框区域，包含表格卡片、选中行、横向滚动条、分页区，以及表格与右侧预览面板的视觉关系。

## 思考与分析

### Skill 嗅探结果

- 项目根目录未发现额外 `SKILL.md` 或 `AGENTS.md` 文件，本次遵循对话中提供的项目级 AI 指令集与开发规范。

- 已使用 `ui-ux-pro-max` 技能参考表格、分页与响应式布局风险点：表格需要横向滚动兜底，异步图片需稳定尺寸，分页控件之间要保留足够间距。

### 当前问题定位

从截图红框看，当前列表区的主要问题不是单个元素错误，而是表格、选中行和分页之间的视觉节奏不一致：

- 表格卡片顶部、表头、选中行、横向滚动条、分页区都在同一大容器内，但分区边界过于生硬，选中行的蓝色描边和滚动条同时出现时，视觉重量偏重。
- 当前表格使用稳定最小宽度和横向滚动兜底，能防止列名竖排，但在只有一条数据时，列表区显得过空，行内容与表头之间的垂直对齐不够紧凑。
- 缩略图、创建时间、最近使用、收藏按钮之间的视觉密度差异较大，导致用户视线从左到右移动时节奏不连续。
- 分页区采用三栏布局，但在数据很少时仍占据完整高度，和上方单行数据之间形成较大的空白断层。
- 截图中右侧预览面板只露出边缘，说明默认窗口下表格卡片与预览布局已经接近临界状态，需要优先保护表格区可读性，避免再增加横向负担。

### 代码与文档落点

- 页面结构由 `src/js/pc-library.js` 的 `renderLibraryContent()`、`renderTableRow()` 和 `renderPagination()` 输出。
- 表格和分页样式集中在 `src/css/pc.css` 的 `pc-library-*` 类名下，重点涉及 `.pc-library-table-card`、`.pc-library-table-scroll`、`.pc-library-table`、`.pc-library-row-active`、`.pc-library-pagination`。
- 现有模块文档为 `docs/UI计划/PC端/02-提示词库.md`，已说明提示词库表格列名、横向滚动与 16:9 响应式策略。
- 若进入实现阶段，需要同步 `docs/apps-code-map.md` 与 `docs/UI计划/PC端/02-提示词库.md`。

### 设计目标

- 让红框区域像一个完整的“列表工作台”，而不是表格、滚动条、分页三个独立拼块。
- 保持表格列名单行可读，继续保留横向滚动兜底，但弱化滚动条与选中描边同时出现时的视觉冲突。
- 提升单条数据场景的平衡感：表头、行内容、分页的高度比例更稳定，空白更克制。
- 保持缩略图、时间、收藏、操作的对齐规则一致，不改变现有数据结构和交互逻辑。
- 不影响移动端，不新增依赖，不重写提示词库筛选、收藏、分页、预览和右键菜单逻辑。

## 实施计划

### Phase 1：红框列表区视觉基线

1. 调整表格卡片整体分区：
   - 保持 `.pc-library-table-card` 作为唯一外层卡片。
   - 使用更柔和的边框、内阴影或浅色分割线统一表头、列表、分页之间的层级。
   - 避免在表格行和卡片外框之间叠加过多高饱和描边。
2. 收敛红框内垂直节奏：
   - 表头高度维持在 `48px` 左右。
   - 数据行高度建议控制在 `108px` 到 `120px`，保证缩略图完整但不显空。
   - 分页区高度建议从固定厚重感改为内容驱动，保持 `64px` 到 `72px` 的稳定区间。
3. 统一边距规则：
   - 表头、行单元格、分页左右内边距使用同一组变量或同等数值。
   - 横向滚动条与分页之间增加轻量分隔，避免滚动条贴住分页区。

### Phase 2：表格列与单行内容对齐

1. 优化列宽权重：
   - 提示词列保留最大权重，但不要过度挤压时间列。
   - 标签列在无标签时应保持空态稳定，不让空列显得像断层。
   - 收藏与操作列继续使用固定宽度和居中对齐。
2. 优化缩略图和文本组合：
   - 缩略图保持 `16:9`，可微调到更协调的宽度，例如 `120px × 68px` 或 `124px × 70px`。
   - 提示词标题、摘要与缩略图垂直居中，摘要最多两行，避免行高被长文本撑开。
3. 优化时间信息：
   - 创建时间继续允许两行显示，但应与“最近使用”的基线更统一。
   - 时间字体重量略低于标题，高于辅助摘要，形成清晰层级。

### Phase 3：选中态与滚动条减冲突

1. 调整选中态：
   - 将 `.pc-library-row-active` 从强蓝色整圈描边，优化为浅蓝背景 + 左侧强调条或内侧细描边。
   - 避免选中态边框压住表头分割线和滚动条区域。
2. 优化 hover 与 active：
   - hover 只做浅背景和轻微色彩变化，不改变行高、边框宽度或布局尺寸。
   - active 状态和 hover 状态共用同一套色彩语义，避免蓝色、灰色、白色层级割裂。
3. 滚动条处理：
   - 保留横向滚动能力。
   - 通过 `scrollbar-color`、滚动条高度或滚动条上下留白降低视觉突兀感。
   - 若浏览器支持有限，至少保证滚动条不贴住选中行蓝色边界。

### Phase 4：分页区重排与低数据量状态

1. 三段分页对齐：
   - 左侧总数与每页数量形成一组。
   - 中间页码保持视觉中心。
   - 右侧跳转输入和“页”形成紧凑组。
2. 低数据量适配：
   - 只有一页时，上一页和下一页按钮保持禁用态，但视觉上降低权重。
   - 总数、每页选择、跳转输入仍保留，避免功能突然消失造成认知跳动。
3. 窄宽度兜底：
   - 继续使用容器查询在主内容区不足时让分页分行。
   - 分行后每组控件左对齐，控件之间保持最小 `8px` 到 `10px` 间距。

### Phase 5：默认 16:9 与侧栏状态复核

1. 默认窗口 `1366 x 768`：
   - 展开侧栏时，红框区域不出现表格外溢到页面外的问题。
   - 收起侧栏时，表格区变宽但不让行内容显得松散。
2. 常见桌面尺寸：
   - `1440 x 900` 下表格与预览关系稳定。
   - `1920 x 1080` 下内容最大宽度受控，不把表格列拉得过散。
3. 小窗口兜底：
   - `1024 x 576` 下表格保留横向滚动，分页允许分行，不出现按钮文字竖排。
