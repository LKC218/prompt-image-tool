# PC首页侧边栏收拉与自适应缩放计划-260510

> 范围：PC 端应用外壳、首页主内容区、侧边栏收拉交互、文档同步与验证

## 思考与分析

### Skill 嗅探结果

- 项目根目录未发现额外 `SKILL.md`，本次以 `AGENTS.md` 约束、现有 PC 端原生 JavaScript 与原生 CSS 架构为准。
- 已使用 `planning-with-files` 技能，将本次需求沉淀为持久 Markdown 计划文档。
- 已使用 `frontend-design` 技能约束本次设计方向：复用现有可爱治愈、轻量工具型风格，不引入新 UI 库，不重做全局设计系统。
- 已使用 `ui-ux-pro-max` 技能参考响应式与可访问性风险点：收起侧边栏后仍需保留可识别导航、焦点状态、合理对比度和稳定布局尺寸。

### 当前问题定位

- PC 端应用壳层由 `src/js/pc-app.js` 渲染，左侧侧边栏和主内容区分别为 `.pc-sidebar` 与 `.pc-main`。
- 侧边栏宽度由 `src/css/pc.css` 中 `--pc-sidebar-width: 280px` 控制，当前没有展开/收起状态。
- 主内容区已设置 `container-type: inline-size` 与 `container-name: pc-main`，首页已有容器查询兜底；因此侧栏收起后，首页可通过主内容区宽度自然获得更多展示空间。
- 首页布局集中在 `pc-home-*` 类名，已具备统计卡、最近使用、收藏分类、快速创建等响应式规则；本次不应重构首页数据与业务交互。
- 当前侧边栏包含 Logo、产品名、副标题、导航文字、吉祥物、本地数据卡片。原安全徽章区域已改为收拉按钮区，避免顶部 Logo 行拥挤。

### 设计目标

- 在 PC 首页左侧导航栏增加收拉操作，用户可以在完整侧边栏与窄图标栏之间切换。
- 收起态隐藏字体与非必要装饰信息，仅保留应用图标、导航图标、收拉按钮和必要的可访问标签。
- 侧边栏宽度变化后，首页主内容区自适应放大或收缩，利用已有 `pc-main` 容器查询减少额外页面逻辑。
- 收拉状态持久化到 `localStorage`，再次打开 PC 端保持用户上次选择。
- 不影响移动端，不引入新依赖，不改变路由、数据存储、导入导出和首页核心交互。

### 设计原则

- 展开态保持现有视觉，避免破坏已经完成的 16:9 与首页排版优化。
- 收起态宽度建议控制在 `76px` 到 `88px`，导航项使用固定方形点击区，图标居中。
- 收起态隐藏文字时使用视觉隐藏或 `aria-label` 保留语义，不让屏幕阅读器丢失导航含义。
- 收拉按钮放在原安全徽章区域，展开态使用满宽胶囊按钮并显示文案，收起态退化为紧凑图标按钮，并提供 `title`、`aria-label`、`aria-expanded`。
- 动效只做宽度、透明度与轻微位移，时长控制在 `180ms` 到 `240ms`，并尊重 `prefers-reduced-motion`。

## 实施计划

### Phase 1：状态与结构

1. 在 `src/js/pc-app.js` 增加侧边栏收起状态：
   - 新增状态变量，例如 `isSidebarCollapsed`。
   - 初始值从 `localStorage.getItem('pc-sidebar-collapsed')` 读取。
   - 在 `mount()` 时将状态同步到 `.pc-app`，例如添加 `pc-sidebar-collapsed` 类或 `data-sidebar="collapsed"` 属性。
2. 在 `renderShell()` 中增加收拉按钮：
   - 放在 `.pc-sidebar-header` 内原安全徽章区域。
   - 展开态显示 `收起侧栏` 文案，收起态显示 `展开侧栏` 文案但通过样式隐藏文字。
   - 使用现有图标风格，优先内联简洁 SVG 或复用本地图标，不新增图标库。
   - 按钮需要包含 `aria-label`、`title`、`aria-expanded`。
3. 在 `setupSidebarNav()` 或独立 `setupSidebarToggle()` 中绑定点击事件：
   - 点击后切换状态。
   - 更新根节点类名、按钮文案属性和 `localStorage`。
   - 不触发现有路由跳转逻辑。

### Phase 2：侧边栏收起态样式

1. 在 `src/css/pc.css` 增加宽度变量：
   - `--pc-sidebar-expanded-width: 280px`。
   - `--pc-sidebar-collapsed-width: 84px`。
   - `--pc-sidebar-width` 根据根状态切换。
2. 收起态导航：
   - `.pc-nav-item` 改为居中方形或近方形点击区。
   - 隐藏 `.pc-nav-item span`，保留 `aria-label`。
   - 图标尺寸可从 `20px` 提升到 `22px`，增强识别。
   - 当前页高亮仍使用浅蓝底与蓝色图标。
3. 收起态头部与底部：
   - Logo 保留图标，隐藏产品名和副标题。
   - 吉祥物卡片与本地数据文案收起态隐藏，避免窄栏拥挤。
   - 本地数据环如保留，需要只展示圆环；建议第一阶段隐藏底部复杂卡片，降低风险。
4. 过渡与抗挤压：
   - 给 `.pc-sidebar`、导航文字、收拉按钮区、底部模块设置稳定过渡。
   - 所有隐藏文字使用 `overflow: hidden`、`white-space: nowrap` 和透明度过渡，避免宽度动画中闪烁。
   - 展开态按钮宽度吃满侧栏内容区，减少原徽章区域的大块空白；收起态按钮保持约 `44px` 方形点击区。
   - 在 `prefers-reduced-motion: reduce` 下关闭宽度动画。

### Phase 3：首页自适应缩放

1. 优先复用 `.pc-main` 容器查询：
   - 侧栏收起后主内容区变宽，首页统计卡、最近使用双列、底部双栏自然按容器宽度恢复。
   - 不在 `pc-home.js` 增加侧栏状态判断，避免页面与应用壳层耦合。
2. 补充首页窄宽兜底：
   - 检查 `.pc-home-search-bar`、`.pc-home-stat-grid`、`.pc-home-recent-grid`、`.pc-home-bottom-grid` 在收起/展开切换时是否跳动。
   - 若默认 16:9 下收起侧栏后内容过宽，继续受 `--pc-content-max` 控制，不让阅读线失控。
3. 控制视觉节奏：
   - 收起侧栏时页面内容不做额外动画，只让主内容区因弹性布局自然重排。
   - 避免首页卡片二次淡入，防止用户误以为页面刷新。

### Phase 4：可访问性与键盘行为

1. 收拉按钮：
   - 支持键盘聚焦与回车/空格触发。
   - 展开态 `aria-label="收起侧边栏"`，收起态 `aria-label="展开侧边栏"`。
   - `aria-expanded` 与实际状态一致。
2. 导航项：
   - 收起态每个导航按钮补齐 `aria-label` 和 `title`。
   - 仍通过现有 `.pc-nav-active` 表示当前路由。
3. 焦点样式：
   - 收起态按钮和导航项仍有清晰 `focus-visible`。
   - 禁止只依赖颜色表达状态，保留背景形状或边框差异。

### Phase 5：文档同步

进入实现阶段后同步更新：

- `docs/apps-code-map.md`：补充 `src/js/pc-app.js` 支持侧边栏收拉状态，`src/css/pc.css` 支持展开/收起侧边栏与主内容响应。
- `docs/UI计划/PC端/01-首页仪表盘.md`：补充首页在侧栏展开/收起时的主内容自适应说明。
- 如收拉状态沉淀为全 PC 页面通用壳层规则，再检查 `docs/技术文档/pc-technical-doc.md` 是否需要补充。

## 验收矩阵

| 场景 | 目标 |
|------|------|
| 默认 `1366 x 768` 展开态 | 视觉保持现状，首页无回归 |
| 默认 `1366 x 768` 收起态 | 侧栏仅显示图标，首页主内容区获得更大宽度 |
| `1024 x 576` 展开态 | 首页不出现文字重叠、按钮溢出或横向滚动 |
| `1024 x 576` 收起态 | 图标导航可点击，首页关键模块可读 |
| `1440 x 900` 展开/收起切换 | 宽度切换平滑，主内容不闪烁、不误触发路由 |
| 键盘操作 | 收拉按钮可聚焦可触发，导航项焦点可见 |
| 重启应用 | 侧栏保持上次展开或收起状态 |

## 验收标准
