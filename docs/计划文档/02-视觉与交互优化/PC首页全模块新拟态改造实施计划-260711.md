# PC首页全模块新拟态改造实施计划

> 版本：v1.1.0  
> 创建日期：2026-07-11  
> 范围：PC 首页仪表盘视觉体系、键盘可访问性、响应式验收与文档同步  
> 状态：已实施，自动验收通过；受控浏览器人工交互验收待补充

## 一、目标与边界

### 1.1 改造目标

将 PC 首页统一为“暖白软雕工作台”视觉：页面以暖白为基底，依靠左上高光与右下暖棕投影表达可操作对象的凸起，依靠双向内阴影表达输入槽、图标槽与已激活状态。

改造后，首页应在保持现有信息密度与操作效率的前提下，解决内容区普通卡片、新拟态统计卡、浅色功能卡并存导致的视觉语言不一致问题。

### 1.2 范围

- 欢迎横幅、搜索框、统计卡、最近使用、收藏分类、快速创建六个首页模块。
- 首页内的键盘聚焦、按下反馈、减弱动态与桌面响应式。
- 首页实现与设计文档的同步。

### 1.3 不在本次范围内

- 不修改提示词、分类、图片、收藏、导入等数据逻辑和路由语义。
- 不修改移动端首页；移动端须另行制定触摸优先的新拟态适配计划。
- 不修改全局 `--pc-neu-*` Token 的数值，避免影响已完成的新拟态搜索栏、统计卡、筛选标签、侧边栏和编辑器按钮。
- 不引入第三方 UI 框架、阴影图片或持续性高成本动画。

## 二、现状分析

### 2.1 源码与文档落点

| 类型 | 文件 | 职责 |
|---|---|---|
| 页面结构与事件 | `src/js/pc-home.js` | 渲染首页六个模块，绑定搜索、跳转、收藏、更多菜单、分类、新建、导入事件。 |
| PC 首页样式 | `src/css/pc.css` | 首页布局、搜索框、统计卡、内容面板、列表、分类与快捷创建样式。 |
| 新拟态规范 | `docs/设计文档/新拟态按钮设计规范.md` | 定义 `--pc-neu-*` Token、状态和 Uiverse / CodePen 参考原则。 |
| 首页模块说明 | `docs/UI计划/PC端/01-首页仪表盘.md` | 首页结构、交互、尺寸和验收要求。 |
| 代码导航 | `docs/apps-code-map.md` | 首页样式、逻辑与设计文档入口。 |

### 2.2 已具备的能力

- 搜索框已采用“外凸容器 + 内凹输入槽”结构。
- 统计卡已采用凸起表面、内凹圆形图标槽和语义色顶部描边。
- 全局已具备 `--pc-neu-bg`、`--pc-neu-light`、`--pc-neu-dark`、偏移量、模糊半径、圆角和过渡 Token。
- 首页已具备减弱动态媒体查询、侧边栏收拉响应式和最近使用空状态。

### 2.3 当前问题

1. `pc-home-panel` 使用白色面板、实线边框和普通投影，内部又嵌套最近使用行、分类卡和快捷卡，形成卡片套卡片。
2. 最近使用行、分类卡和快捷创建卡仍使用普通边框、浅色填充或单向投影，与搜索框、统计卡的新拟态光源不一致。
3. 搜索输入框移除了原生聚焦轮廓，但外层没有明确的键盘聚焦环，仅放大镜变色，焦点识别不足。
4. 收藏分类项目由可点击 `div` 渲染，无法通过键盘聚焦或触发；现有 `:focus-visible` 规则对该元素无效。
5. 快速创建中的“新建提示词”和“导入提示词”权重相近，首页主任务不够突出。

## 三、设计决策

### 3.1 视觉层级

| 层级 | 使用位置 | 视觉规则 |
|---|---|---|
| 平面叙事层 | 欢迎横幅、页面背景、标题区、标签文本 | 不使用新拟态外阴影；以插画、排版和颜色提供识别。 |
| 凸起信息层 | 统计卡、最近使用行、分类块、导入入口 | 与 `--pc-neu-bg` 接近，使用统一左上高光与右下投影。 |
| 内凹交互层 | 搜索输入槽、缩略图槽、统计图标槽、图标按钮、已按下状态 | 使用双向 `inset` 阴影，不使用深色描边。 |
| 强操作层 | 新建提示词 | 使用现有高对比品牌 CTA，而不是低对比新拟态，确保主任务可见。 |

### 3.2 参考案例的落地原则

- 采用新拟态规范中 Uiverse 案例的统一光源、凸起和按压凹陷逻辑。
- 采用 CodePen 胶囊案例的多层阴影思路，仅用于紧凑图标按钮和选择态，不迁移其强烈文字阴影或缩放幅度。
- 不复制外部示例代码；所有实现复用项目现有 Token、字体、主题色与无障碍规则。

### 3.3 状态规则

| 状态 | 非主操作卡片 | 图标按钮 | 主 CTA |
|---|---|---|---|
| 默认 | 轻量凸起 | 轻量凸起 | 高对比品牌填充 |
| 悬停 | 阴影略增强，不改变布局尺寸 | 阴影收紧，图标变为主题色 | 保留现有 CTA 动效 |
| 按下 | 内凹阴影，最多下移 1px | 内凹阴影，最多下移 1px | 沿用现有按下反馈 |
| 键盘聚焦 | 2px 品牌色聚焦环 | 2px 品牌色聚焦环 | 保留现有聚焦环 |
| 禁用 | 降低透明度，移除互动阴影 | 同左 | 保留现有禁用逻辑 |

## 四、分阶段实施计划

### Phase 0：基准截图与回归边界

- 启动 PC 首页，在侧边栏展开和收起状态下保存 `1024x768`、`1180x768`、`1366x768`、`1440x900` 截图。
- 记录欢迎横幅、搜索、统计、最近使用、底部双栏和空状态的现有位置。
- 确认搜索、查看全部、详情跳转、收藏切换、更多菜单、分类跳转、新建和导入均可用，作为改造后的回归清单。

### Phase 1：结构与可访问性修正

**修改文件：** `src/js/pc-home.js`

- 将收藏分类项目从 `div.pc-category-card` 改为 `button type="button".pc-category-card`。
- 保留 `data-folder-id` 和现有事件委托，确保点击后仍导航至 `/library` 并携带 `folder` 参数。
- 为分类按钮保留可读名称；文件夹图片继续使用空 `alt`，防止屏幕阅读器重复朗读装饰图。
- 不修改最近使用项的数据排序、图片加载、收藏、更多菜单、导入和搜索逻辑。

### Phase 2：页面容器和搜索焦点

**修改文件：** `src/css/pc.css`

- 将 `.pc-home-panel` 从“白底 + 描边 + 外阴影卡片”调整为无描边的内容分区；最近使用、收藏分类、快速创建只保留必要内边距，不再形成外层浮卡。
- 欢迎横幅保持插画背景和像素动画，不增加拟态外阴影；它是视觉叙事层，不是互动对象。
- 保留 `.pc-home-search-bar` 的外凸、`__outer` 的内凹结构；将其硬编码阴影替换为由现有 `--pc-neu-*` Token 推导的值。
- 在 `.pc-home-search-bar:focus-within` 上增加 `outline` 或等效品牌色聚焦环，并维持放大镜的现有轻量反馈。
- 保持搜索框悬浮位置和首页 `18px` 模块间距，除非截图验证出现新的重叠或首屏压迫。

### Phase 3：统计卡与最近使用

**修改文件：** `src/css/pc.css`

- 统计卡保留现有新拟态结构，不重做；将最小高度与首页设计文档统一为 `96px`，确保数字和标签不拥挤。
- 移除 `.pc-recent-item` 的实线边框，改为暖白凸起行；默认阴影使用比统计卡更轻的 Token 派生值，以适配六条高密度列表。
- 悬停仅增强阴影与主题色提示，不再使用明显上浮；按下使用短暂内凹，避免列表阅读时产生跳动。
- 将 `.pc-recent-thumb` 改为内凹缩略图槽。真实图片继续 `object-fit: cover`，无图占位图标保持可见。
- 将收藏和更多按钮统一为 32px 圆形或圆角新拟态图标按钮；收藏激活继续以黄色填充星标表达，不能只依赖阴影区分状态。
- 最近使用空状态改为与分区同色的静态内容，不增加额外浮卡或厚重阴影。

### Phase 4：收藏分类与快速创建

**修改文件：** `src/js/pc-home.js`、`src/css/pc.css`

- 分类卡取消当前内联整块浅色背景。数据渲染时仅传入分类语义色 CSS 自定义属性，例如 `--pc-home-category-color`。
- 分类卡主体统一为暖白凸起表面；分类语义色只用于文件夹图标、分类名称和顶部短色线，避免六种背景色与主题色竞争。
- 文件夹图标容器使用轻量内凹槽；分类名称保留单行省略与数量显示。
- `新建提示词` 改为唯一强品牌 CTA，优先复用现有 `.pc-create-btn` 的视觉和交互约定；保留当前路由 `/editor/`。
- `导入提示词` 改为中性凸起拟态入口，图标置于内凹槽；保留文件选择和 JSON 导入流程。
- 两个入口仍保持稳定的两列尺寸，在窄桌面自动随现有断点堆叠，不因图标或文字状态改变尺寸。

### Phase 5：交互、响应式与减弱动态

**修改文件：** `src/css/pc.css`，必要时新增或扩展 `src/js/pc-home.test.js`

- 统一首页可交互元素的 `:focus-visible`，覆盖查看全部、搜索框、最近使用行内按钮、分类按钮、快捷创建按钮。
- 分类按钮使用原生键盘 `Enter` / `Space` 触发，不额外手写键盘事件。
- 仅为可操作对象保留 `box-shadow` 与 `transform` 过渡；不对统计数据、标题、标签、插画添加循环拟态动画。
- 继续遵守 `prefers-reduced-motion: reduce`：禁用进入、悬停位移和像素动画的持续运动，静态表面仍保留清晰层级。
- 验证 `1180px` 最近使用单列、`1100px` 统计和分类两列、底部模块堆叠后没有阴影裁切、文字重叠或横向滚动。
