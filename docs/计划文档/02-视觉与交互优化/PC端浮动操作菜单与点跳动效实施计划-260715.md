# PC端浮动操作菜单与点跳动效实施计划

> 创建日期：2026-07-15
> 范围：PC 端提示词集合的更多操作入口、右键操作入口及其共享操作菜单
> 状态：已实施待验收

## 一、目标与边界

将现有带外层卡片和分割线的矩形上下文菜单，重构为由独立悬浮操作按钮与文字标签组成的操作组。点击三点更多按钮时，三个点依次跳动，并与按钮组的逐项展开连续衔接。

本次只改变 PC 端共享菜单的呈现、触发反馈、定位、键盘可用性与关闭行为，不改变已有操作动作、存储逻辑、确认弹窗和移动分类对话框。

### 1.1 保持不变

- `getPromptSetMenuItems()` 继续提供重命名、移动到分类、复制和删除等业务动作与处理函数。
- 点击“移动到分类”继续打开现有分类选择对话框，不引入层级子菜单。
- 删除操作继续进入现有不可撤销确认弹窗。
- 右键菜单继续在指针位置触发；首页、提示词库、详情页和分类页继续复用同一菜单能力。

### 1.2 不在本次范围内

- 不引入 Lottie 文件、播放器依赖或远程动画资源。
- 不改动移动端长按菜单。
- 不修改操作项文案、图标资源或业务权限判断。
- 不将菜单改造成持续常驻的浮动操作按钮。

## 二、现状与问题

### 2.1 现有实现

- `src/js/pc-utils.js` 的 `showContextMenu(x, y, items)` 创建全局唯一的 `#pcContextMenu`，在鼠标坐标附近显示菜单，并通过 Promise 返回选中的 `action`。
- `src/css/pc.css` 的 `.pc-context-menu`、`.pc-context-item` 与 `.pc-context-divider` 组成一张带统一背景、边框和分割线的传统菜单卡片。
- 三点更多按钮与右键入口均调用共享菜单，业务动作由 `src/js/pc-menu-actions.js` 的 `getPromptSetMenuItems()` 维护。

### 2.2 重构动机

- 现有菜单的外层卡片和分割线视觉较重，与当前轻拟态和独立图标按钮体系不一致。
- 三点按钮点击后缺少明确的触发反馈，菜单出现与触发源之间缺乏视觉连续性。
- 参考 Speed Dial 的独立按钮展开节奏适合短操作集合，但其固定悬浮按钮形态不适合直接套用在列表项上下文菜单中。

## 三、交互方案

### 3.1 菜单形态

- 菜单由若干 `.pc-context-action` 独立操作行构成，不再使用整块白色外层卡片和分割线。
- 每行采用“文字标签 + 图标方形按钮”布局；默认文字在左、图标按钮在右。
- 菜单整体以触发元素或右键坐标为锚点，但每一项自身独立承担背景、圆角、阴影与按压反馈。
- 普通操作按重命名、移动到分类、复制的顺序排列；删除始终位于末尾，以危险色进行区分。

### 3.2 三点触发反馈

- 三点更多按钮中的三个点拆为独立元素，按“左、中、右”顺序各跳动一次。
- 单点跳动总时长为 340ms，点间延迟为 95ms；上移幅度 3px，最高点缩放至 1.05，并以平滑减速落位，不使用额外回弹。
- 点击时触发按钮在前 60ms 左右缩放到 0.94，并于约 180ms 内平滑恢复正常尺寸。
- 点跳动开始约 220ms 后，操作按钮组开始显示；此时第一个点已完成主体位移、第二个点正在上行，保证反馈与菜单展开连续衔接。
- 菜单已展开时，触发按钮保留激活态；再次点击仅关闭菜单，不重复播放点跳动效果。

### 3.3 操作组展开与收拢

- 首项从最靠近触发点的位置开始显示，后续项按远离触发点的方向依次出现。
- 单项使用 `opacity` 与 `transform` 动画：从 `translateY(8px) scale(.92)` 过渡到正常位置与尺寸。
- 单项时长为 220ms 至 260ms，相邻项错峰 45ms；关闭时以较短的反向过渡收拢。
- 悬停时按钮上移 2px 并强化阴影；按下时缩放至 0.96。
- 所有动画只使用 `transform` 与 `opacity`，不动画 `top`、`left`、宽度或高度。

### 3.4 打开、关闭与重复触发

- 三点按钮点击：播放一次点跳动，随后打开菜单；快速重复点击需取消上一轮待显示任务，确保同一时刻仅有一个菜单实例。
- 右键触发：直接显示操作组，不播放三点跳动，因为该入口没有三点触发源。
- 点击叶子操作：先关闭操作组，再执行既有动作。
- 再次点击原触发按钮、点击操作组外部、页面滚动、路由切换或按 Escape：关闭操作组。
- 菜单关闭后将焦点归还三点触发按钮；右键触发且无可用触发元素时不强制转移焦点。

### 3.5 键盘与辅助技术

- 菜单容器使用 `role="menu"`，操作项使用原生 `button` 与 `role="menuitem"`。
- 打开后焦点落在首个可用操作项；上下方向键循环切换，Enter 与空格执行，Escape 关闭。
- 三点按钮保留可见的 `focus-visible` 状态，并维护 `aria-expanded` 与 `aria-controls`。
- 删除按钮保留语义化危险操作标识与现有二次确认。

## 四、视觉与定位规范

### 4.1 尺寸与层级

| 元素 | 规范 |
| --- | --- |
| 图标按钮 | 42px × 42px，14px 圆角 |
| 标签字号 | 13px 至 14px，中等字重 |
| 标签与按钮间距 | 10px |
| 操作行间距 | 10px |
| 菜单与锚点间距 | 8px |
| 视口安全边界 | 8px |
| 菜单层级 | `z-index: 9500`，保持既有弹层优先级 |

### 4.2 色彩与阴影

- 图标继续沿用既有语义色：重命名紫色、移动蓝色、复制绿色、删除红色。
- 普通按钮使用暖白表面与柔和双层阴影；删除按钮采用淡红底色、红色标签和红色图标底座。
- 移除菜单外层边框与外层白底，避免形成第二张大卡片。
- 操作组背后使用比内容范围外扩约 `12px` 的局部柔焦层，以暖白半透明表面和 `backdrop-filter: blur(10px)` 弱化重叠背景；不支持模糊时提高表面不透明度作为可读性兜底。柔焦层不接收指针事件，不替代独立操作按钮，也不形成传统菜单卡片。
- 激活、悬停、按下与焦点态复用现有 PC 主题变量，不新增固定品牌主色。

### 4.3 定位策略

- 三点入口默认使操作组沿纵向向下展开；下方空间不足时向上翻转。
- 操作组以图标按钮列靠近触发点的方向排列，标签向可用空间一侧延伸。
- 右键入口根据指针所在半区决定文字标签的延伸方向，优先避免右侧或左侧文字裁切。
- 完成初次测量后，`left` 与 `top` 只用于最终定位；视觉进入与离开只通过 `transform` 和 `opacity` 实现。

### 4.4 减少动态效果

- `prefers-reduced-motion: reduce` 下不播放点跳动、按钮错峰、缩放和位移动画。
- 菜单与按钮直接显示或隐藏，保留焦点、键盘和关闭逻辑。

## 五、代码实施步骤

### Phase 1：确认调用入口与触发元素

1. 检查首页、提示词库、详情页和分类页中三点更多按钮的结构与事件绑定。
2. 将可控的三点文本标记升级为三个独立点元素，确保不影响现有按钮文案、`aria-label` 与事件委托。
3. 为 `showContextMenu()` 增加可选触发元素与触发来源参数，保证旧的右键调用保持兼容。

### Phase 2：重构共享菜单渲染与状态

1. 在 `src/js/pc-utils.js` 中保留单例菜单、Promise 返回值和现有清理机制。
2. 将菜单项渲染为包含标签与图标按钮的独立操作行；忽略既有 `divider` 的视觉渲染，但不要求调用方立即移除该数据。
3. 记录打开来源、触发元素、待展开任务和当前动作监听器；统一处理取消、关闭和焦点归还。
4. 加入 Escape、方向键、Enter 与空格处理，并避免与页面其他全局快捷键重复响应。

### Phase 3：实现三点跳动与菜单衔接

1. 在 `src/css/pc.css` 新增三点跳动关键帧、触发按钮按下态、激活态和低动态覆盖。
2. 点击三点按钮时添加一次性开启状态类，在 `animationend` 后清理，避免下一次打开无法重新播放。
3. 通过一次可取消的延迟任务，在第二个点开始跳动后显示操作组。
4. 菜单关闭、路由切换或重复点击时清除待展开任务和开启状态类。

### Phase 4：实现独立按钮视觉与视口定位

1. 替换 `.pc-context-menu`、`.pc-context-item`、`.pc-context-divider` 的传统卡片视觉为操作组样式。
2. 新增独立操作项、图标按钮、标签、危险态、悬停、按下、焦点和进入/离开状态。
3. 实现基于触发元素的上下翻转定位，以及基于鼠标坐标的左右延伸定位。
4. 在 1366 × 768 默认窗口和窄内容区确认菜单不遮挡主操作、不超出视口。

### Phase 5：接入业务页并同步文档

1. 首先接入首页最近使用项和提示词库列表的三点入口，核验与右键入口的共用行为。
2. 接入详情页、分类页等其余 `getPromptSetMenuItems()` 调用处。
3. 更新 `docs/UI计划/PC端/01-首页仪表盘.md` 与 `docs/UI计划/PC端/02-提示词库.md` 的更多操作说明。
4. 更新 `docs/apps-code-map.md` 的 `pc.css`、`pc-utils.js`、`pc-menu-actions.js` 及业务页导航说明。

### 6.2 人工验收

- 在首页、提示词库、详情页和分类页检查三点入口的跳动节奏与菜单展开连续性。
- 在列表中部、底部、窗口左右边缘分别验证定位与翻转。
- 验证右键入口直接出现操作组，不播放三点动效。
- 验证删除危险态清晰但不影响其他操作可读性。
- 在启用减少动态效果的系统环境中确认无循环或位移动画。
- 运行项目既有构建与测试命令，并记录与本次改动无关的预存失败。

## 八、风险与回退

- 共享菜单覆盖多个页面，重构前先确保 `showContextMenu()` 的旧参数签名可继续工作，避免一次性修改所有调用点。
- 点跳动只允许在三点入口触发；右键入口、键盘触发和程序化调用不得等待动画。
- 若独立按钮组在高密度列表中造成遮挡或定位复杂度过高，可保留新的三点跳动，同时回退为有外层卡片的紧凑菜单布局；业务动作、关闭逻辑与无障碍改进无需回退。
- 不使用外部 Lottie 资源，避免网络可用性、许可证、包体体积与运行时初始化风险。

## 九、完成标准

- 三点点击有一次明确、非循环的左中右点跳动，且按钮组在约 140ms 后开始展开。
- 操作菜单呈独立悬浮按钮与文字标签，不显示传统分割线菜单卡片。
- 右键和三点入口均可调用同一业务动作，旧调用方不发生功能回归。
- 菜单始终位于视口内，支持外部点击、滚动、Escape、键盘执行和焦点归还。
- 减少动态效果下保留完整功能但关闭动态表现。
- 相关自动化验证、构建与人工验收完成后再将本文档状态改为“已完成”。
