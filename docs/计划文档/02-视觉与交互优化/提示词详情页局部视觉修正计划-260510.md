# 提示词详情页局部视觉修正计划-260510

## 思考与分析

### Skill 嗅探结果

- 根目录未发现项目级 `SKILL.md`。
- 本次任务是针对 PC 端提示词详情页进行局部视觉修正计划制定，已使用 `frontend-design` 技能约束布局、视觉层级、响应式和可访问性。
- 已使用 `ui-ux-pro-max` 技能检索图片缩放、柔和卡片界面相关建议。关键结论：图片应随容器缩放，避免固定尺寸溢出；有意义图片需保留可访问替代文本；柔和马卡龙风格应保持清晰对比和可读层级。
- 已使用 `planning-with-files` 技能，将本次 5 项问题拆成独立计划文档，避免混入上一份大重构计划。

### 当前问题定位

本次问题均集中在 `src/js/pc-detail.js` 和 `src/css/pc.css` 的 `pc-detail-*` 详情页实现：

1. 顶部天空背景目前受 `.pc-detail-screen` 内边距和 `.pc-detail-sky` 绝对定位影响，视觉上没有铺满整页宽度。
2. 右侧“信息概览”字段过多，当前包含风格/模型、比例、生成引擎、标签、创建时间，导致卡片高度偏大，并挤压第一屏右侧信息密度。
3. 无封面图片占位仍使用字符图标，需要替换为指定图片资源：`UI设计稿/图标/插画设计/图片.png`。
4. 本地安全提示卡文案需要改成“开始编写提示词”，并保留参考图中的简洁提示卡气质。
5. 封面图片当前使用 `object-fit: cover`，会裁切内容；需要改为按比例缩小完整显示图片内容，即 `object-fit: contain`，避免预览区域放大裁切。

### 设计目标

- 只做局部修正，不重排详情页主体结构，不改变路由、数据存储和事件绑定。
- 顶部背景横向铺满窗口，同时保留云朵和星点装饰层。
- 信息概览只保留“标签、比例、创建时间”三项，减少右侧栏高度，使第一屏更接近参考图窗口高度。
- 空封面占位使用指定插画图片，不再使用字符占位图标。
- 安全提示卡文案调整为“开始编写提示词”，视觉上保持轻量、不抢主内容。
- 封面预览完整显示图片全貌，容器内可留白，但不裁切、不放大到丢失内容。

## 实施计划

### Phase 1：顶部背景铺满

1. 调整 `.pc-detail-screen` 与 `.pc-detail-sky` 的关系：
   - 保留页面内容左右内边距。
   - 让 `.pc-detail-sky` 使用 `left: 50%`、`width: 100vw`、`transform: translateX(-50%)` 或等价方案铺满视口宽度。
2. 检查云朵装饰在 1366px、1440px 和宽屏下是否仍覆盖顶部主体后方。
3. 避免给整个详情页加横向滚动，重点检查 `100vw` 与滚动条宽度带来的溢出风险。

### Phase 2：信息概览精简并保持窗口高度

1. 修改 `renderInfoCard()`：
   - 删除“风格/模型”字段。
   - 删除“生成引擎”字段。
   - 保留字段顺序为：标签、比例、创建时间。
2. 清理不再需要的临时变量：
   - `styleModel`
   - `engine`
3. 评估 `getStyleModel()` 与 `getGenerateEngine()` 是否仍被使用：
   - 若仅详情页使用且无引用，直接删除，保持 KISS。
4. 调整 `.pc-detail-info-card` 或 `.pc-detail-info-item` 的内边距：
   - 保持信息概览卡片视觉稳定。
   - 不为了撑满而加无意义空白。
   - 右侧栏整体高度应更紧凑，第一屏更接近截图目标。

### Phase 3：空封面占位图标替换

1. 将 `UI设计稿/图标/插画设计/图片.png` 复制或整理到正式运行资源目录：
   - 推荐目标：`src/assets/pc/detail-image-placeholder.png`。
   - 原 `UI设计稿/` 目录只作为设计稿和参考，不直接作为运行时资源引用。
2. 在 `src/js/pc-detail.js` 中通过 Vite import 引入该图片：
   - 避免生产构建后资源路径失效。
3. 修改 `renderCoverImage()` 的无图占位：
   - 将字符图标替换为 `<img>`。
   - 保留“暂无封面图片”文本或按视觉需要弱化。
4. 新增或调整 CSS：
   - `.pc-detail-cover-placeholder-icon`
   - 固定图标尺寸，避免图片原始尺寸撑开布局。

### Phase 4：提示卡文案调整

1. 修改 `renderLocalSafetyCard()` 文案：
   - 主文案改成“开始编写提示词”。
2. 保留或弱化副文案：
   - 若完全按用户要求，只显示“开始编写提示词”。
   - 若需要保持卡片平衡，可保留短副文案，但优先以用户指定文本为核心。
3. 若文字变短导致卡片右侧插画空间过空，微调 `.pc-detail-safety-card` 内边距和插画位置。

### Phase 5：封面图完整预览

1. 将 `.pc-detail-cover img, .pc-detail-cover-img-wrap img` 的 `object-fit` 从 `cover` 改为 `contain`。
2. 为封面容器增加柔和底色：
   - 避免图片未铺满时露出的空白显得突兀。
   - 可使用浅米色、浅蓝粉渐变或棋盘感极弱的中性背景。
3. 保持点击图片打开原图逻辑不变。
4. 检查多图切换：
   - 切换时仍完整显示。
   - 计数器和圆点不遮挡关键图片内容。

### Phase 6：文档同步

1. 更新 `docs/apps-code-map.md`：
   - 若新增 `src/assets/pc/detail-image-placeholder.png`，补充 PC 详情页占位图资源说明。
   - 维持该文档只做导航，不写完成记录。
2. 更新就近模块说明文档：
   - `docs/UI计划/PC端/03-提示词详情.md`
   - `docs/UI计划/PC端/03-提示词详情-可爱治愈风.md`
3. 更新本计划文档任务状态与验证结果。

## 风险与注意事项

- 顶部背景使用 `100vw` 时可能因滚动条宽度产生 1 至 2px 横向溢出，实现时需要检查 `overflow-x`。
- 将封面改为 `contain` 后，部分竖图或窄图会出现留白，需要用柔和背景承接，不能显得像图片加载失败。
- 指定图片资源不应直接从 `UI设计稿/` 引用，应进入 `src/assets/` 后由构建工具管理。
- 信息概览字段减少后，右侧栏视觉重心会变轻，需要保持版本记录和提示卡间距统一。
