# PC端文字字符图标替换计划-260512

## 思考与分析

### Skill 嗅探结果

- 已使用 `planning-with-files` 技能：本任务是多步骤 UI 资产补齐与前端替换，需要先沉淀可执行计划文档，再进入实现。
- 已使用 `frontend-design` 技能：本任务涉及 PC 端按钮、菜单、状态、提示词详情和编辑页的图标体系统一，需要保持现有本地图标接入方式、可访问性和稳定尺寸。

### 当前问题

PC 端仍存在用 Emoji、数学符号、箭头字符、勾选字符和加号字符充当图标的情况。它们会带来以下风险：

- 不同系统字体渲染不一致，PC 安装版和浏览器预览可能出现粗细、颜色、垂直对齐差异。
- Emoji 会受系统彩色字体影响，难以通过 CSS 统一颜色。
- 纯字符图标缺少统一尺寸盒，容易在按钮、标签、弹窗、详情卡片中造成轻微跳动。
- 现有文档已要求 PC 端图标优先本地化，继续保留字符图标会破坏统一设计规则。

### 外部图标来源

- iconfont 搜索入口：`https://www.iconfont.cn/search/index?searchType=icon&q=%E4%BF%9D%E5%AD%98&page=1&fromCollection=-1`
- 使用策略：从 iconfont 补齐缺失语义图标，但不使用远程脚本、在线字体或 CDN。所有图标下载为 SVG 后本地化放入仓库，保留来源和授权说明。
- 若 iconfont 页面需要登录或动态下载受限，优先人工导出 SVG；实现层仍按本地 SVG 模块导入，不依赖 iconfont 运行时。

## 实施计划

### Phase 1：资产获取与本地化

1. 打开 iconfont 搜索页，以“保存”为起点补齐同风格线性 SVG 图标。
2. 统一选择线性、圆角、单色、可通过 CSS `filter` 或 `currentColor` 控制的图标。
3. 下载 SVG 后放入 `src/assets/icons/pc/`。
4. 新增 `src/assets/icons/pc/iconfont-license.md`，记录 iconfont 页面地址、下载日期和图标来源说明。
5. 若图标 SVG 内含固定颜色，优先清理为 `fill="currentColor"` 或 `stroke="currentColor"`；复杂路径保留原始结构但移除多余尺寸噪声。

### Phase 2：建立 PC 图标工具

1. 在需要替换的模块内复用现有 `iconImg()` 模式，避免引入新依赖。
2. 对 SVG 按模块导入，生成 `<img class="pc-icon-img" alt="" aria-hidden="true">`。
3. 对纯图标按钮补齐 `aria-label` 或保留已有 `title`。
4. 避免把图标改成远程 URL、iconfont 字体类或运行时脚本。

### Phase 3：逐页替换字符图标

1. 先替换 `pc-detail.js`，因为残留最多，且详情页有顶部按钮、元信息、操作区和菜单四类场景。
2. 再替换 `pc-editor.js`，覆盖返回、添加、删除、分类、正负向提示词和选中状态。
3. 替换 `pc-menu-actions.js` 的编辑菜单项和选中勾。
4. 替换 `pc-library.js` 分页箭头。
5. 替换 `pc-utils.js` 图片查看器复位与关闭按钮。

### Phase 4：样式收口

1. 复用 `.pc-icon-img`、`.pc-btn-icon`、`.pc-inline-icon` 等已有样式。
2. 为详情页补充必要的局部尺寸规则，确保图标盒固定为 16px、18px 或 20px。
3. 删除只服务字符图标的特殊字号、字体权重和 line-height 修补。
4. 检查按钮内文字不被图标挤压，窄宽度下不换行错位。

### Phase 5：文档同步

1. 更新 `docs/apps-code-map.md`：
   - 增加 `src/assets/icons/pc/` 的导航说明。
   - 更新 `pc-detail.js`、`pc-editor.js`、`pc-utils.js` 的图标职责描述。
2. 更新 `docs/技术文档/pc-technical-doc.md`：
   - 说明 PC 端字符图标已统一本地化为 SVG。
   - 说明 iconfont 图标只作为离线资产来源，不作为运行时依赖。
3. 如实施过程中发现“字符图标替换规范”可复用，再沉淀到 `docs/项目开发经验/项目开发经验.md`。

## 风险与处理

| 风险 | 表现 | 处理 |
|---|---|---|
| iconfont 下载需要登录 | 无法直接命令下载 SVG | 浏览器手动加入项目并导出 SVG，代码仍只依赖本地 SVG |
| SVG 风格不统一 | 线宽、圆角、填充差异明显 | 同一批图标优先选择同作者或同风格，必要时换用已有 Lucide SVG |
| 彩色 SVG 难以跟随主题 | hover 或深浅色状态不一致 | 清理为单色 `currentColor` 或通过 CSS filter 统一 |
| 替换后按钮尺寸跳动 | 图标盒尺寸不固定 | 给图标容器设置固定宽高、flex 居中 |
| 扫描误报业务字符 | 正文或用户数据里的字符被命中 | 只扫描源码模板和按钮区域，业务文本不做硬替换 |
