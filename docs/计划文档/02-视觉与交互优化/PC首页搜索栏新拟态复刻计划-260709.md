# PC 首页搜索栏新拟态复刻计划

> 版本：v1.0.0
> 创建日期：2026-07-09
> 范围：PC 端首页搜索栏视觉重构（新拟态 Neumorphism 风格），含结构、样式、轻量交互反馈与文档同步
> 参考原型：CodePen「Neumorphism Search Bar」(tdtrung17693 / BaozOzQ)

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 根目录未发现 `SKILL.md`。
- 本次为「视觉风格复刻」任务，目标是把 CodePen 的新拟态搜索栏效果落到本项目 PC 首页搜索栏。
- 当前 PC 首页结构由 `src/js/pc-home.js` 渲染，样式由 `src/css/pc.css` 承载。
- 就近文档落点：`docs/UI计划/PC端/01-首页仪表盘.md`（第 91 行有搜索栏条目）。
- 文件导航落点：`docs/apps-code-map.md`（第 25 行提及 `pc-home.js`）。

### 1.2 当前搜索栏现状

- **HTML**（`src/js/pc-home.js` 第 51-54 行，模板字符串动态生成）：单层结构
  `.pc-home-search-bar#pcHomeSearch` > `.pc-search-icon`(SVG 放大镜) + `.pc-search-input`。
- **CSS**：`src/css/pc.css` 存在**两处** `.pc-home-search-bar` 定义：
  - 第 **4829** 行：基础版（旧，含 `margin-bottom: var(--pc-space-3xl)`）。
  - 第 **6008** 行：增强版（**实际生效**，有 `min-height:62px; margin:-18px 0 2px`，带柔和外投影）。
  - 因 CSS 后者覆盖前者，实际以 6008 区块为准，但两处并存易混淆，需清理。
- **JS**（`src/js/pc-home.js` 第 289 行 `setupHomeEvents`）：
  - `input` 事件：实时过滤首页内容。
  - `keydown Enter`：当有关键词时 `navigate('/library', { search })` 同步跳转。
- **图标**：`.pc-home-inline-icon` 控制 20×20 尺寸（`src/css/pc.css` 第 5992 行）；放大镜为 SVG 描边图标（`ICONS.search`）。

### 1.3 原型效果拆解（CodePen）

| 原型要点 | 适配策略 |
|----------|----------|
| 双向阴影凸起外框（`.finder`） | 复刻：右下暖灰暗影 + 左上纯亮边，背景取比页面略亮的实色制造凸起 |
| `inset` 内凹输入区（`.finder__outer`） | 复刻：新增一层 `__outer` 包裹，用 `inset` 双向阴影制造凹槽 |
| 伪元素画放大镜 | **不采用**：项目已是 SVG 描边放大镜，更清晰，保留 SVG |
| focus 图标上浮 + 弹性缓动 | 复刻：作用于 `.pc-search-icon`，用 `cubic-bezier(0.68,-0.55,0.265,1.55)` 过冲回弹 |
| submit → spinner 旋转加载 | 轻量化：项目 Enter 为同步跳转，改为跳转前 320ms 旋转反馈（`is-searching`） |

### 1.4 关键约束

- `.pc-search-icon`、`.pc-search-input`、`.pc-home-inline-icon` 是**全局类**（library/category 搜索框共用），所有改动必须 scope 在 `.pc-home-search-bar` 选择器下，避免污染其它页面。
- 上浮（`transform` 挂 `span`）与旋转（`transform` 挂 `svg`）分离，避免 transform 冲突。
- 保留现有业务行为：实时过滤、回车跳转、无障碍焦点可达。

## 二、实施计划

### Phase 1：HTML 结构改为双层（方案 A 忠实复刻）

修改 `src/js/pc-home.js` 第 51-54 行，复刻 CodePen 的 `finder / __outer / __inner` 三层嵌套：

```html
<div class="pc-home-search-bar" id="pcHomeSearch">
    <div class="pc-home-search-bar__outer">
        <div class="pc-home-search-bar__inner">
            <span class="pc-search-icon pc-home-inline-icon">${ICONS.search}</span>
            <input type="text" class="pc-search-input" placeholder="搜索提示词、标签、分类..." id="pcHomeSearchInput">
        </div>
    </div>
</div>
```

### Phase 2：清理旧样式并落地新拟态（src/css/pc.css）

1. **删除**第 4829-4843 行旧 `.pc-home-search-bar` 定义（避免与 6008 区块冲突）。
2. 在 **6008 区块**重写外框为新拟态凸起：

```css
.pc-home-search-bar {
    min-height: 62px;
    display: flex;
    align-items: center;
    margin: -18px 0 2px;
    padding: 8px;                 /* 给 outer 预留凸起边框厚度 */
    position: relative;
    z-index: 2;
    background: var(--pc-surface);            /* #FFFFFF，比 page #FFFDF8 略亮 → 凸起 */
    border: 1px solid rgba(255, 255, 255, 0.6);
    border-radius: 22px;
    box-shadow:
        9px 9px 18px rgba(120, 90, 50, 0.14),     /* 暗影 右下 */
        -9px -9px 18px rgba(255, 255, 255, 0.95); /* 亮边 左上 */
    transition: box-shadow 0.25s ease, border-color 0.2s ease;
}
.pc-home-search-bar:focus-within {
    border-color: color-mix(in srgb, var(--pc-accent) 40%, transparent);
}
```

3. 新增 **中层凹槽** 与 **内层行**（紧接外框定义之后）：

```css
.pc-home-search-bar__outer {
    display: flex;
    width: 100%;
    align-items: center;
    padding: 14px 20px;
    border-radius: 16px;
    background: var(--pc-surface);   /* 与外层同色，靠 inset 显凹凸 */
    box-shadow:
        inset 8px 8px 16px -6px rgba(120, 90, 50, 0.16),
        inset -8px -8px 16px -6px rgba(255, 255, 255, 0.95);
}
.pc-home-search-bar__inner {
    display: flex;
    align-items: center;
    gap: 14px;
    flex: 1;
    min-width: 0;
}
```

4. 图标弹性上浮（focus-within 时）：

```css
.pc-home-search-bar .pc-search-icon {
    color: #5E6675;
    transition: transform 0.45s cubic-bezier(0.68, -0.55, 0.265, 1.55), color 0.2s ease;
}
.pc-home-search-bar:focus-within .pc-search-icon {
    color: var(--pc-accent);
    transform: translateY(-4px) scale(1.05);
}
```

5. 轻量旋转反馈（跳转前）：

```css
.pc-home-search-bar.is-searching .pc-search-icon svg {
    animation: pc-home-search-spin 0.45s linear infinite;
}
@keyframes pc-home-search-spin {
    to { transform: rotate(360deg); }
}
```

### Phase 3：无障碍降级

在 `src/css/pc.css` 现有 `@media (prefers-reduced-motion: reduce)` 区块（第 4809 行附近）追加：

```css
.pc-home-search-bar .pc-search-icon,
.pc-home-search-bar .pc-search-icon svg {
    transition: none;
    animation: none;
}
```

### Phase 4：JS 轻量旋转反馈（src/js/pc-home.js）

修改 `setupHomeEvents` 的 `keydown Enter` 分支，跳转前打 `is-searching` 并延时 320ms 跳转，让旋转可见：

```js
searchInput.addEventListener('keydown', (e) => {
    if (e.key === 'Enter' && homeSearchKeyword) {
        const bar = pageEl.querySelector('#pcHomeSearch');
        if (bar) bar.classList.add('is-searching');
        setTimeout(() => navigate('/library', { search: homeSearchKeyword }), 320);
    }
});
```

## 五、待确认问题

- 旋转反馈时长 320ms 为轻量值；若工程师希望更明显可上调至 450-500ms（但会延迟跳转感知）。
- 新拟态背景目前取 `var(--pc-surface)`(#FFFFFF)；若首页背景在深色主题下切换，需后续补充暗色新拟态阴影变量（当前项目以浅色为主，本次不处理暗色分支）。
