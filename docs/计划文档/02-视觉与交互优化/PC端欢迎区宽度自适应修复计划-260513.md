### 当前问题定位

截图中的问题出现在 PC 端提示词库页面顶部欢迎横幅。相关代码位置如下：

- 欢迎横幅组件：`src/js/pc-welcome-banner.js`
- 首页调用：`src/js/pc-home.js`
- 提示词库调用：`src/js/pc-library.js`
- 样式集中位置：`src/css/pc.css`
- 背景图片资源：`src/assets/pc/home-bg.png`
- 右侧吉祥物资源：`src/assets/mobile/mascots/corgi-home.png`

当前横幅基础样式为 `.pc-welcome-banner`，已经通过负外边距抵消页面内边距：

```css
margin: calc(var(--pc-page-pad-y) * -1) calc(var(--pc-page-pad-x) * -1) 0;
```

但页面主体 `.pc-page` 仍受 `max-width: var(--pc-content-max)` 与 `margin: 0 auto` 控制。左侧导航收起后，主内容可视区域变宽，而 `.pc-page` 最大宽度仍固定在 `1360px`，导致红框范围内的欢迎背景只覆盖内容容器宽度，无法铺满主内容可视宽度。

根因不是背景图尺寸不够，也不是 `background-size: cover` 失效，而是“横幅背景层”和“页面内容安全区”被放在同一个最大宽度容器里。继续只调 `background-size` 或换更宽图片，无法从根上解决留白问题。

### 设计目标

- 欢迎区背景图片横向铺满 PC 主内容可视宽度，左侧贴近主内容边界，右侧延伸到滚动条前的可用区域。
- 横幅内部文字、按钮和吉祥物仍保持舒适的内容安全区，不在超宽屏上无限拉开。
- 保持现有可爱治愈视觉风格，不更换图片资产，不重构页面路由。
- 优先以 CSS 修复，避免改动数据层和页面业务逻辑。
- 同时覆盖首页、提示词库、编辑页、分类页、设置页等复用欢迎横幅的 PC 页面，避免只修截图单页。

## 实施计划

### Phase 1：确认布局边界

1. 复核 `.pc-main`、`.pc-page`、`.pc-welcome-banner` 三层宽度关系。
2. 明确当前主内容宽度来源：
   - `.pc-main` 负责可滚动主区域。
   - `.pc-page` 负责页面内容最大宽度和居中。
   - `.pc-welcome-banner` 当前只是 `.pc-page` 内部的横幅。
3. 确认提示词库页面的欢迎区来自 `renderPcWelcomeBanner({ className: 'pc-welcome-banner-library' })`，不是独立写死的页面横幅。

### Phase 2：拆分背景层和内容安全区

推荐方案是保留 `.pc-page` 的内容最大宽度，但让欢迎横幅背景横向突破 `.pc-page`：

1. 给 `.pc-welcome-banner` 建立“视口铺满”宽度策略：
   - 使用 `width: calc(100cqw - var(--pc-page-pad-x) * 2)` 或等效主容器宽度算法。
   - 若容器单位兼容性不稳定，则退回到基于 `.pc-main` 宽度的负边距方案。
2. 横幅背景层铺满主内容可视宽度：
   - 继续使用 `background-size: cover`。
   - 按页面类型微调 `background-position`，避免右侧吉祥物被裁切。
3. 横幅内部 `.pc-welcome-section` 设置最大宽度：
   - 使用 `max-width: var(--pc-content-max)` 或独立变量。
   - 居中显示，保持标题和吉祥物的位置稳定。

### Phase 3：搜索栏与操作按钮同步适配

截图中提示词库页的搜索栏和“新建提示词”按钮位于横幅下方。欢迎区铺满后，需要避免下方控件视觉错位：

1. 保持搜索栏仍在内容安全区内，不跟随横幅背景无限拉伸。
2. 检查 `.pc-library-search-row` 在左侧栏展开、收起状态下的对齐。
3. 确认按钮在宽屏时不会贴近滚动条，窄屏时不会与搜索栏挤压。

### Phase 4：响应式断点补齐

1. 在主内容较窄时，横幅回到普通容器宽度，避免负边距造成横向滚动。
2. 在左侧栏收起状态下，横幅背景应随 `.pc-main` 增宽自然铺满。
3. 在 `1024 x 576`、`1366 x 768`、`1920 x 1080` 三档下检查：
   - 横幅背景无左右明显留白。
   - 标题不贴边。
   - 右侧吉祥物不被滚动条遮挡。
   - 搜索栏和主按钮不重叠。

### Phase 5：文档与验证

如果进入实现阶段并修改 PC 通用欢迎横幅规则，需要同步：

- `docs/apps-code-map.md`
- `docs/UI计划/PC端/01-首页仪表盘.md`
- `docs/UI计划/PC端/02-提示词库.md`

若最终只是微调 `pc.css` 的通用横幅宽度，不需要新增模块说明文档；若抽象出新的横幅布局规则，则补充到 PC 端 UI 设计文档。

## 风险与注意事项

- 不建议直接取消 `.pc-page` 的 `max-width`，否则表格、详情页和编辑页会在大屏上过度拉伸。
- 不建议只修改背景图资源尺寸，根因在容器宽度，不在图片像素。
- 不建议把搜索栏也强行铺满全宽，搜索输入过长会降低可读性，并破坏提示词库工具区节奏。
- 若使用容器查询单位，需要确认当前构建链路和目标 WebView 支持情况；不稳定时使用传统负边距兜底。
