# PC端默认16比9与自适应排版优化计划-260510

## 思考与分析

### Skill 嗅探结果

- 已检查项目内 `.github/skills/frontend-design/SKILL.md`，本次任务属于 PC 端 UI/UX 布局优化，应优先遵循项目内前端设计规范。
- 已使用全局 `frontend-design` 技能确认本次优化目标：保持现有产品风格，提升桌面端布局比例、阅读层级、响应式稳定性和交互质感。
- 已使用 `ui-ux-pro-max` 技能参考响应式布局风险点：避免内容横向溢出、表格压缩、固定元素遮挡、动态内容导致布局跳动。
- 已使用 `planning-with-files` 技能，将本次计划沉淀为持久 Markdown 文档。

### 当前问题定位

PC 端应用由 Tauri 窗口承载，优化前默认窗口为 `1200 x 800`，最小窗口为 `800 x 600`。现有 PC UI 采用固定左侧栏与右侧主内容区结构：

- 左侧栏固定 `280px`。
- 主内容页统一使用左右 `48px` 内边距。
- 多数页面断点按整个视口宽度计算，而不是按扣除侧边栏后的主内容容器宽度计算。

因此在优化前默认 `1200 x 800` 窗口下，实际可用于页面内容的宽度约为 `824px`。如果继续按视口宽度判断是否进入双栏、单栏、表格横向滚动，就容易出现以下问题：

- 页面看似达到桌面断点，实际主内容区已经不足以承载双栏。
- 提示词库表格和右侧预览面板互相挤压。
- 详情页主图高度按 `vw` 计算，实际容器变窄后图片比例失衡。
- 编辑页双栏区域在中等窗口中被压缩，表单、标签、比例按钮容易换行错乱。
- 卡片内部图标、标题、按钮、统计数字在窄内容区中互相抢空间。

### 优化目标

- 将 PC 端默认桌面体验固定为 `16:9` 视觉比例，默认窗口确认采用 `1366 x 768`，兼顾 16:9 与当前信息密度。
- 建立“主内容容器优先”的响应式策略，避免仅按视口宽度判断布局。
- 所有关键卡片、表格、按钮、标题、标签都具备抗挤压能力，不出现文字重叠、逐字竖排、按钮文字溢出。
- 保持现有可爱治愈、柔和工具型风格，不重写数据层、不引入新 UI 库、不影响移动端。
- 先做 PC 端全局布局兜底，再逐页治理首页、提示词库、详情页、编辑页、分类页、设置页。

## 实施计划

### Phase 1：窗口比例与应用外壳基线

1. 调整 Tauri PC 端默认窗口：
   - 默认窗口确认采用：`1366 x 768`。
   - 保持 `resizable: true`。
   - 最小窗口允许从 `800 x 600` 提升到 `1024 x 576`，维持 16:9，同时避免 PC UI 被压到移动端尺度。
2. 建立 PC 端视觉安全区：
   - 左侧栏仍保留固定宽度，但需要评估是否从 `280px` 缩到 `256px`。
   - 页面左右内边距从固定 `48px` 改为响应式变量，例如 `clamp(24px, 3vw, 48px)`。
   - 主内容区设置统一最大宽度，并在大屏下居中，避免 2K、4K 屏过宽导致阅读线过长。
3. 明确 PC 端不自动切移动端：
   - Tauri 环境继续使用 PC UI。
   - 窄窗口下只做 PC 单栏退化，不混入移动端组件。

### Phase 2：响应式断点从视口转向主内容区

1. 梳理当前断点：
   - `1440px`：提示词库预览面板收窄。
   - `1280px` / `1279px`：设置页、详情页局部调整。
   - `1180px`：提示词库、首页、分类页开始折叠。
   - `1023px` / `959px`：详情页、编辑页进入单栏。
2. 新增主内容区宽度判断思路：
   - 优先使用 CSS 容器查询；若兼容性或改造成本不合适，则通过更保守的视口断点兜底。
   - 双栏页面折叠判断应按“主内容区可用宽度”设计，而不是按窗口总宽度设计。
3. 推荐页面级断点：
   - 主内容区 `>= 1120px`：允许复杂双栏。
   - 主内容区 `900px - 1119px`：使用窄双栏或重点页提前单栏。
   - 主内容区 `< 900px`：统一单栏，卡片纵向排列。

### Phase 3：全局抗挤压排版规则

1. 卡片基础规则：
   - 所有网格项使用 `minmax(0, 1fr)`。
   - 卡片标题、描述、标签容器设置 `min-width: 0`。
   - 重复卡片设置稳定 `min-height`，避免异步内容或长文本导致跳动。
2. 文本防错规则：
   - 单行标题使用 `overflow: hidden`、`text-overflow: ellipsis`、`white-space: nowrap`。
   - 多行正文使用 `overflow-wrap: anywhere` 或 `word-break: break-word`。
   - 中文表头、按钮、短标签默认不逐字换行。
3. 按钮防错规则：
   - 图标按钮优先使用固定宽高。
   - 文本按钮设置 `white-space: nowrap`，窄屏时允许整组换行，不允许单个按钮内文字竖排。
   - 主操作按钮在窄内容区可从横排改成两列或纵排。
4. 图片比例规则：
   - 列表缩略图、预览图、详情页封面统一用明确 `aspect-ratio`。
   - 避免用 `vw` 单独控制内容图片高度。
   - 详情页封面默认使用 `16 / 9`，优先由容器宽度推导高度，确保默认 16:9 窗口下图片区域比例稳定。

### Phase 4：首页适配优化

1. 首页统计卡片：
   - 默认 4 列。
   - 主内容区不足时提前降为 2 列。
   - 数字、图标、标题不互相挤压，长数字允许缩小或截断说明文本。
2. 最近使用列表：
   - 默认双列。
   - 中等窗口提前改为单列。
   - 缩略图保持固定比例，信息区使用省略策略。
3. 底部分类与快捷创建：
   - 保持两块区域平衡。
   - 主内容区不足时纵向排列，避免两个面板并排压缩。

### Phase 5：提示词库适配优化

1. 工作区布局：
   - 大屏：表格 + 右侧预览面板。
   - 默认 16:9 窗口：优先保护表格可读性，右侧预览面板提前下沉。
   - 窄窗口：预览面板下沉，表格独占宽度。
2. 表格兜底：
   - 表格设置稳定最小宽度。
   - 外层使用横向滚动兜底。
   - 表头始终单行可读，不出现逐字竖排。
3. 分页区：
   - 页码、跳转、每页数量在中窄宽度下允许分行。
   - 不允许分页控件挤压表格列。

### Phase 6：提示词详情页适配优化

1. 详情页主体：
   - 大屏保持左主右辅双栏。
   - 默认 16:9 下确保主栏宽度优先，不让右栏压缩主图和提示词正文。
   - 主内容区不足时右栏下沉。
2. 封面图：
   - 取消单纯依赖 `vw` 的高度计算。
   - 建议使用 `aspect-ratio: 16 / 9` 加最大高度限制。
   - 保证无图占位、图片轮播按钮、计数器不重叠。
3. 标题与元信息：
   - 标题允许多行，但不能压住收藏与操作按钮。
   - 元信息条在窄区域改为单列或两列。

### Phase 7：编辑页适配优化

1. 表单布局：
   - 大屏保留主表单 + 侧栏。
   - 中等窗口提前单栏，避免右侧“分类、比例、插画”压缩主表单。
2. 图片上传区：
   - 缩略图网格使用 `auto-fill + minmax`。
   - 上传按钮与删除按钮固定尺寸。
3. 比例按钮：
   - 保持统一宽高。
   - 空间不足时整组换行，不改变按钮内部文本排版。

### Phase 8：分类与设置页适配优化

1. 分类与标签页：
   - 表格列在中窄宽度下隐藏次要列或切为更紧凑的两列结构。
   - 快捷操作区优先保持按钮文本可读。
2. 设置页：
   - 外观、本地存储、备份、同步模块按主内容宽度折叠。
   - 操作卡片从 4 列逐步降为 2 列、1 列。
   - IP 地址、按钮组允许换行，避免同步地址挤出容器。

## 验收矩阵

| 场景 | 分辨率 | 目标 |
|------|--------|------|
| 最小推荐 PC 窗口 | `1024 x 576` | 不出现文本重叠、卡片互相挤压、按钮竖排 |
| 默认 PC 窗口 | `1366 x 768` | 主要页面第一屏比例舒适，提示词库和详情页可读 |
| 常见笔记本 | `1440 x 900` | 双栏页面层级清晰，卡片间距稳定 |
| 标准桌面 | `1920 x 1080` | 主内容不过度拉伸，阅读线长度合理 |
| 高分屏 | `2560 x 1440` | 页面居中且最大宽度受控，不显得松散 |

## 风险与注意事项

- 直接提升最小窗口尺寸会改变用户可缩放下限；已确认接受 `1024 x 576` 作为 PC 端最小有效体验。
- 如果只改 Tauri 窗口，不改 CSS 断点，默认比例会改善，但中窄窗口下仍会出现卡片挤压。
- 如果只改 CSS 断点，不改默认窗口，首次启动仍可能不是理想 16:9。
- 表格横向滚动是兜底策略，不应替代合理列宽和预览面板折叠。
- 详情页封面从 `vw` 控高改为容器比例后，个别历史图片的展示裁切方式需要复核。
- 本计划只覆盖 PC 端 UI，不应影响移动端 `mobile.css` 与 `mobile-*.js`。

## 已确认决策

- 默认窗口采用 `1366 x 768`。
- PC 端最小窗口允许提升到 `1024 x 576`。
- 提示词库默认窗口下，右侧预览面板提前下沉，以保护表格可读性。
- 详情页封面默认使用 `16:9`。
