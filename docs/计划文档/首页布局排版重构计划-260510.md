# 首页布局排版重构计划

> 版本：v1.0.0  
> 创建日期：2026-05-10  
> 范围：PC 端首页布局、视觉层级、交互动效与文档同步

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现 `SKILL.md`。
- 本次使用 `frontend-design`、`ui-ux-pro-max`、`planning-with-files`。
- 当前项目为 Vite + 原生 JavaScript + 原生 CSS，PC 首页主要由 `src/js/pc-home.js` 输出结构，由 `src/css/pc.css` 承载样式，并复用 `src/js/pc-welcome-banner.js` 顶部欢迎横幅。
- 当前首页已有欢迎横幅、搜索、统计、最近使用、收藏分类、快速创建等模块，但整体仍偏纵向堆叠；参考图强调“横幅视觉入口 + 悬浮搜索框 + 四张图标统计卡 + 双列最近使用列表 + 底部双栏功能区 + 本地安全提示条”的桌面首屏结构。
- 设计资源已具备：`src/assets/pc/home-bg.png`、`src/assets/mobile/mascots/corgi-home.png`、本地图标资源和提示词缩略图数据，可优先复用，避免引入新依赖。

### 1.2 设计目标

- 严格参考图片布局排版，形成轻盈、可爱、治愈的桌面仪表盘首页。
- 保留现有首页功能：搜索、统计、最近使用、收藏切换、更多菜单、分类跳转、新建提示词、导入提示词。
- 用本地图标或现有资源替代页面中的装饰性 emoji，提升图标一致性。
- 首页首屏信息密度接近参考图：顶部欢迎区不占满屏，搜索框压在横幅下沿，统计卡横向排布，最近使用区采用左右两列。
- 确保 1024px、1280px、1440px 桌面宽度下文字不重叠、不溢出、无横向滚动。
- 动效只用于轻量进入、悬停和按钮反馈，并尊重 `prefers-reduced-motion`。

### 1.3 参考图拆解

- 顶部欢迎区：浅蓝天空背景，左侧问候文案，右侧柯基插画，整体高度约 190px 到 220px。
- 搜索区：白色长条搜索框覆盖在横幅和内容区之间，左右留白大，右侧有扫描或聚焦类图标按钮。
- 统计区：四张浅色卡片横向排列，每张卡片左侧为图标盒，右侧为标题与数字；色系分别为蓝、黄、紫、粉。
- 最近使用：一个大白色面板，标题左、查看全部右；列表为两列三行，每项包含缩略图、标题、标签、时间、收藏星标和更多按钮。
- 底部功能区：左侧收藏分类四宫格，右侧快速创建两张横向操作卡。
- 安全提示条：底部绿色低饱和提示条，强调本地运行和数据安全。

## 二、实施计划

### Phase 1：结构重排

- 重构 `src/js/pc-home.js` 的 `render()`，新增首页专用外层容器，便于控制参考图式整体节奏。
- 页面顺序调整为：欢迎横幅、悬浮搜索、统计卡、最近使用面板、底部双栏区、安全提示条。
- 最近使用列表保持取前 6 条，但渲染为双列三行；窄桌面宽度自动退化为单列。
- 收藏分类限制展示前 4 个，与参考图四宫格一致；保留点击分类跳转提示词库的行为。
- 快速创建保留“新建提示词”和“导入提示词”两个入口，卡片尺寸与参考图一致。

### Phase 2：数据与缩略图增强

- 统计卡继续读取提示词总数、分类数、标签数、收藏数。
- 最近使用项优先显示集合第一张图片缩略图；无图时使用现有默认缩略图或本地图片图标兜底。
- 标签继续使用 `TAG_COLORS`，控制最多 3 个，避免撑破卡片。
- 搜索逻辑保持当前输入实时过滤，回车进入提示词库搜索结果。

### Phase 3：视觉样式

- 在 `src/css/pc.css` 中新增或收敛首页专用类名，建议统一使用 `pc-home-*` 前缀，避免影响提示词库、分类页和设置页。
- 欢迎横幅复用 `renderPcWelcomeBanner()`，必要时传入首页专用 `className` 扩展尺寸、插画位置和背景裁切。
- 搜索框使用白色高亮表面、柔和阴影和 16px 到 18px 圆角，形成悬浮层级。
- 统计卡使用淡色背景、细边框、图标盒、数字强对比；悬停只做轻微上移和阴影增强。
- 最近使用面板和底部双栏区使用白色面板、轻描边和统一间距，不做卡片套卡片。
- 安全提示条使用浅绿背景、锁形图标、单行文案和右侧箭头。

### Phase 4：交互与动效

- 页面加载时对统计卡、最近使用项和底部功能卡做一次 160ms 到 240ms 的轻量淡入上移。
- 列表项、分类卡、快速创建卡使用固定尺寸和 `transform` 悬停反馈，避免布局抖动。
- 星标和更多按钮保持可点击区域至少 32px，增加 `focus-visible`。
- 对 `prefers-reduced-motion: reduce` 用户关闭进入动效和位移动效。

### Phase 5：文档与验证

- 更新 `docs/apps-code-map.md`，补充首页文件职责和样式落点。
- 更新 `docs/UI计划/PC端/01-首页仪表盘.md`，让文档与新参考图布局一致。
- 如本次实施中沉淀出可复用规则，再更新 `docs/项目开发经验/项目开发经验.md`；若只是局部页面重排，则不新增经验文档。
- 运行 `npm run test`。
- 启动 Vite 开发服务，检查首页在 1024px、1280px、1440px 桌面宽度下无重叠、无溢出、交互无回归。

## 五、待确认问题

- 是否需要做到像参考图一样隐藏左侧导航，只呈现纯首页画布？建议保留现有 PC 端导航框架，只重排首页内容区，避免影响全局路由体验。
- 搜索框右侧参考图的扫描图标是否需要实际功能？建议第一阶段只作为“进入提示词库搜索”或“清空/聚焦”类轻交互，不新增识别功能。
- 最近使用缩略图是否必须全部显示真实图片？建议优先读取已有图片，缺图时用本地默认缩略图兜底。
