# 设置页布局排版重构计划

> 版本：v1.0.0  
> 创建日期：2026-05-10  
> 范围：PC 端设置页布局、视觉层级、交互动效与文档同步

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现 `SKILL.md`。
- 本次使用 `frontend-design`、`ui-ux-pro-max`、`planning-with-files`。
- 当前项目为 Vite + 原生 JavaScript + 原生 CSS，PC 设置页主要由 `src/js/pc-settings.js` 和 `src/css/pc.css` 承载。
- 参考图强调“标题 + 大横幅 + 双列卡片 + 全宽备份区 + 底部安全提示”的桌面排版，当前设置页仍偏纵向堆叠，需要重排信息架构。

### 1.2 设计目标

- 严格贴近参考图的页面节奏：轻量标题、浅蓝插画横幅、两列核心设置卡、横向备份操作卡、绿色安全提示条。
- 保留现有主题色、导入导出、同步地址复制与刷新、存储统计等逻辑，不改变备份数据协议。
- 用现有 SVG 图标和吉祥物资源替代装饰性 emoji，降低视觉噪声。
- 保证键盘可达、焦点清晰、动效可关闭。

## 二、实施计划

### Phase 1：页面结构重排

- 重构 `pc-settings.js` 的 `render()`，输出设置页专用容器。
- 页面顺序固定为：标题、设置欢迎横幅、外观设置与本地存储双列区、数据备份与恢复区、局域网同步区、底部安全提示条。
- 把原有“关于”信息压缩为本地存储卡片中的辅助信息，避免参考图式页面过长。

### Phase 2：视觉与交互动效

- 在 `pc.css` 中补充设置页专用类名，不影响其他页面的通用卡片。
- 横幅使用浅蓝背景、云朵氛围、柯基插画和气泡文案。
- 设置卡片使用柔和边框、轻阴影和 16px 圆角，按钮 hover 只做轻量位移与阴影。
- 增加 `prefers-reduced-motion` 降级规则。

### Phase 3：数据展示与降级

- 数据大小继续读取 `storage.estimateStorageSize()`。
- 存储空间优先读取 `navigator.storage.estimate()`，不可用时展示“浏览器未提供”。
- 环形图使用 CSS `conic-gradient`，通过 `--ring-percent` 更新比例。
- 同步区继续展示本机同步地址、复制地址和刷新入口。

### Phase 4：文档与验证

- 更新 `docs/apps-code-map.md` 的 PC 设置页文件导航。
- 更新 `docs/UI计划/PC端/06-设置与本地存储.md` 的布局说明。
- 运行 `npm run test`。
- 启动 Vite 开发服务，检查页面无构建错误，并进行桌面宽度视觉验收。
