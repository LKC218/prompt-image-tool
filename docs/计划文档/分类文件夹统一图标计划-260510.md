# 分类文件夹统一图标计划

## 思考与分析

- 场景：PC 端“分类与标签”页面的分类行图标需要统一为设计稿文件夹图标。
- 资源来源：`UI设计稿/图标/插画设计/文件夹.png`，属于项目内设计资产。
- 接入方式：从设计稿生成 256px 运行时 PNG `src/assets/icons/category-folder.png`，由 `src/js/pc-category.js` 统一导入，避免运行时引用设计稿目录。
- 视觉策略：分类行继续保留已有分类色背景，图标本体统一为同一个 PNG；标签页和快捷操作图标不参与本轮调整。

## 实施计划

1. 将设计稿 PNG 本地化为 256px 运行时资源 `src/assets/icons/category-folder.png`。
2. 移除分类行按关键词选择 Lucide 图标的逻辑。
3. 分类列表和分类空状态统一使用 `category-folder.png`。
4. 调整分类 PNG 的展示尺寸，确保插画图标在 42px 容器内清晰可见。
5. 同步 `docs/apps-code-map.md` 与 `docs/UI计划/PC端/05-分类与标签.md`。
6. 运行构建与测试，确认资源能被 Vite 正确打包。
