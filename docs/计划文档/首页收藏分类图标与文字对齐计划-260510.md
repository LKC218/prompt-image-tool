# 首页收藏分类图标与文字对齐计划

> 版本：v1.0.0  
> 创建日期：2026-05-10  
> 范围：PC 端首页收藏分类卡片图标资源、图标文字居中对齐、文档同步与验证

## 一、思考与分析

### 1.1 技能与上下文嗅探

- 项目根目录未发现 `SKILL.md`。
- 本次使用 `frontend-design`、`ui-ux-pro-max`、`planning-with-files`。
- 当前 PC 首页收藏分类由 `src/js/pc-home.js` 的 `renderCategoryGrid()` 渲染，卡片结构为 `.pc-home-category-card`，图标为内联 SVG：`ICONS.folder`。
- 当前样式集中在 `src/css/pc.css` 的 `.pc-home-category-*` 区域，卡片已使用 `flex-direction: column` 和 `align-items: center`，但图片中可见图标、分类名和数量的视觉中心仍需进一步统一。
- 目标图片资源存在：`D:\Work\git\prompt-image-tool\UI设计稿\图标\插画设计\文件夹.png`，尺寸为 `1254 × 1254`，格式为 `RGBA`。
- 当前 `vite.config.js` 设置 `root: 'src'`，因此运行时代码不建议直接引用 `src` 外的 `UI设计稿` 路径；实施时应将目标图片从设计稿目录复制到 `src/assets/` 下，再由 `pc-home.js` 导入。

### 1.2 问题定位

- 图标问题：收藏分类卡片当前使用线性 SVG 文件夹，和工程师指定的插画设计图标不一致。
- 对齐问题：分类名与数字虽然在同一列，但不同卡片的文字宽度、数字行和图标视觉重心不完全一致，容易显得没有居中。
- 尺寸问题：目标图片为大尺寸 PNG，不能直接按原尺寸展示，需要通过 CSS 固定显示盒尺寸，避免图片影响布局。
- 构建问题：若直接写绝对路径或 `UI设计稿` 外部路径，会影响 Vite 构建和跨机器可维护性。

### 1.3 设计目标

- 收藏分类卡片内的图标、分类名、数量三者以卡片中心轴对齐。
- 图标替换为指定的设计稿文件夹 PNG 视觉，并通过运行时资源目录稳定接入。
- 保持卡片原有浅色背景、点击跳转、hover 和响应式布局不变。
- 长分类名继续单行省略，不撑破卡片。
- 数量与分类名使用统一居中排版，避免出现参考图中“数字和图标看似偏移”的问题。

## 二、实施计划

### Phase 1：资源接入

- 将 `UI设计稿\图标\插画设计\文件夹.png` 复制到运行时资源目录，建议命名为 `src/assets/pc/home-folder.png` 或 `src/assets/icons/home-folder.png`。
- 在 `src/js/pc-home.js` 顶部导入该 PNG 资源。
- 保留 `ICONS.folder` 供统计卡和空状态继续使用，避免影响其他模块。

### Phase 2：收藏分类结构调整

- 修改 `renderCategoryGrid()` 中收藏分类卡片图标节点：
  - 从 `<span class="pc-home-category-icon ...">${ICONS.folder}</span>` 改为图片节点。
  - 建议结构为 `<span class="pc-home-category-icon"><img src="..." alt="" aria-hidden="true"></span>`。
- 为分类名和数量增加一个文本容器，例如 `.pc-home-category-copy`，统一控制垂直间距和居中。
- 保持 `data-folder-id` 与点击跳转逻辑不变。

### Phase 3：对齐与视觉样式

- 调整 `.pc-home-category-card` 为稳定的居中布局：
  - `align-items: center`
  - `justify-content: center`
  - 固定内部间距
  - 固定图标盒尺寸
- 为 `.pc-home-category-icon img` 设置固定宽高、`object-fit: contain`、`display: block`。
- 为 `.pc-home-category-copy` 设置 `display: flex`、`flex-direction: column`、`align-items: center`、`text-align: center`。
- 分类名保持 `max-width: 100%`、`overflow: hidden`、`text-overflow: ellipsis`、`white-space: nowrap`。
- 数字保持独立行，居中显示，避免受分类名长度影响。

### Phase 4：文档同步

- 更新 `docs/apps-code-map.md` 中首页资源说明，补充首页收藏分类图标资源落点。
- 更新 `docs/UI计划/PC端/01-首页仪表盘.md` 中收藏分类组件说明，注明使用本地 PNG 插画图标。
- 更新本计划任务清单状态。

### Phase 5：验证

- 运行 `npm run test`。
- 运行 `npm run build`。
- 启动或复用 Vite 服务，检查首页收藏分类：
  - 图标显示为目标 PNG。
  - 图标、分类名、数量在卡片中横向居中。
  - 1024px、1280px、1440px 宽度下不溢出、不重叠。
  - 点击分类卡片仍能跳转到提示词库筛选结果。
