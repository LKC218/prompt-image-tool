# PC端UI完全重构计划

> 版本：1.0 | 创建日期：2026-05-06 | 状态：待执行

---

## 一、重构背景与目标

### 1.1 旧版PC端问题

当前PC端UI存在以下严重问题：

- ❌ **无路由系统**，所有功能挤在一个页面（app.js 单文件1600+行）
- ❌ **无首页仪表盘**、无独立提示词库、无分类管理页、无设置页
- ❌ **无标签系统**、无收藏功能
- ❌ **暗色主题为主**，与移动端卡通风格完全不一致
- ❌ **无页面模块化**，难以维护和扩展
- ❌ 侧边栏仅为列表筛选，非导航结构

### 1.2 重构目标

将PC端从旧版单页面模式，完全重构为与移动端对等的6页面路由架构，采用PC端UI计划文档中定义的**左侧导航栏 + 右侧主内容区**布局，保持与移动端一致的功能完整度，同时适配桌面端交互。

**核心原则**：UI设计风格与移动端保持一致（浅色卡通风格、马卡龙配色、柯基吉祥物）。

## 六、PC端特有交互

- **右键菜单** 替代移动端长按
- **键盘快捷键**：Escape关闭弹窗、Ctrl+S保存、Delete删除
- **拖拽排序**：分类拖拽排序
- **表格布局**：提示词库使用表格而非卡片流
- **双栏表单**：编辑页左右分栏
- **侧边栏导航** 替代底部Tab导航
- **窗口控制**：最小化/最大化/关闭（Tauri环境）

## 八、风险与注意事项

1. **共享模块不修改**：`storage.js`、`utils.js`、`image-utils.js`、`tag-utils.js`、`lan-sync.js` 保持不变，PC端和移动端共用
2. **移动端不受影响**：所有 `mobile-*.js` 和 `mobile.css` 不做任何修改
3. **Tauri兼容**：PC端需同时支持浏览器和Tauri桌面环境
4. **数据兼容**：新旧PC端使用同一套数据结构，无需数据迁移
5. **渐进式开发**：每个Phase完成后可独立验证，不必等全部完成
6. **吉祥物资源**：PC端复用移动端 `assets/mobile/mascots/` 下的柯基/兔子插画
