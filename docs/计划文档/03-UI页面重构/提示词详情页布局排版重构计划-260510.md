# 提示词详情页布局排版重构计划-260510

## 思考与分析

### Skill 嗅探结果

- 根目录未发现项目级 `SKILL.md`。
- 本次任务是制定提示词详情页 UI/UX 重排计划，已使用 `frontend-design` 技能确认视觉方向、布局层级、动效和可访问性边界。
- 已使用 `ui-ux-pro-max` 技能检索桌面详情页、柔和马卡龙风格、响应式布局相关规范。
- 已使用 `planning-with-files` 技能，将计划沉淀为持久 Markdown 文档。

### 当前页面定位

当前 PC 端提示词详情页由 `src/js/pc-detail.js` 渲染，页面样式集中在 `src/css/pc.css` 的 `pc-detail-*` 类名段落。现有结构包含：

- 顶部返回、标题、收藏、更多操作。
- 封面图。
- 标题、标签、收藏数。
- 正向提示词和负向提示词卡片。
- 信息概览卡片。
- 版本记录卡片。
- 底部编辑、复制、更多操作栏。

现状问题不是功能缺失，而是桌面端信息层级仍偏单列堆叠，和参考图中的宽屏双栏详情布局不一致。参考图强调的是：

- 顶部云朵天空氛围背景 + 页面标题 + 面包屑 + 右上操作。
- 主体白色大圆角容器。
- 左侧为大图、标题、标签、元信息条、提示词正文和主操作。
- 右侧为信息概览、版本记录、本地安全提示。
- 整体是可爱治愈风，但内容仍保持工具型页面的清晰扫描效率。

### 设计目标

- 严格贴近参考图的桌面排版比例：左侧主内容约 68%，右侧信息栏约 32%。
- 保留现有详情页功能和数据流，不重写存储、不改变路由、不引入新依赖。
- 使用现有马卡龙色彩变量和本地资源，优先复用 `src/assets/mobile/mascots/corgi-settings.png`、`src/assets/mobile/mascots/tip-mascot.png` 等插画资产。
- 将信息概览与版本记录移动到右侧栏，使主阅读流更聚焦。
- 增强封面图和标题区的视觉权重，让第一屏接近参考图的“图片主视觉 + 详情卡片”结构。
- 响应式退化时保持单列可读，不牺牲移动端已有独立详情页。

### 视觉方向

- 风格：可爱治愈风、浅天空蓝顶部氛围、奶油白主体、马卡龙标签、柔和描边和阴影。
- 设计主动作：详情页主体从“顺序堆叠”升级为“沉浸式详情画布”，用顶部背景和双栏卡片形成强首屏。
- 动效：页面进入时采用轻量 `fade-in-up` 分层出现，封面图切换保留淡入淡出；按钮 hover 使用 150ms 至 200ms 的颜色和阴影过渡。

## 实施计划

### Phase 1：结构重排

1. 调整 `src/js/pc-detail.js` 的详情页 DOM 结构：
   - 将当前顶栏升级为参考图式顶部区域。
   - 左侧显示页面标题“提示词详情”和面包屑“首页 > 提示词库 > 当前提示词名称”。
   - 右侧保留收藏和更多操作按钮。
2. 新增主体布局容器：
   - `.pc-detail-hero`
   - `.pc-detail-breadcrumb`
   - `.pc-detail-shell`
   - `.pc-detail-layout`
   - `.pc-detail-main-column`
   - `.pc-detail-side-column`
3. 调整 `renderDetailContent()` 输出顺序：
   - 左栏：封面图、标题标签区、元信息条、正向提示词、负向提示词、底部操作栏。
   - 右栏：信息概览、版本记录、本地安全提示。
4. 保留现有事件绑定 ID：
   - `pcDetailBack`
   - `pcDetailStar`
   - `pcDetailMoreTop`
   - `pcDetailCopyAll`
   - `pcDetailEdit`
   - `pcDetailMore`
   - 图片切换相关 ID。

### Phase 2：封面与标题区复刻

1. 封面图改为参考图中的横向大图比例：
   - 左栏内高度建议 300px 至 360px。
   - 图片使用 `object-fit: cover`，优先满足参考图的大画面沉浸感。
   - 无图片时使用图像占位卡，不使用大段说明文字。
2. 标题区与封面衔接：
   - 标题卡片略微上移覆盖封面底部，形成参考图的白色浮层效果。
   - 标题、标签、收藏数同一水平层级。
   - 标签颜色继续使用现有马卡龙映射。
3. 元信息条：
   - 在标题区下方加入三段信息条：创建时间、标签、创建者。
   - 当前数据没有创建者字段时，显示固定“提示词管家”。

### Phase 3：提示词卡片优化

1. 正向提示词卡片：
   - 使用浅绿色背景内层，强化“可用内容”的正向语义。
   - 复制按钮保持绿色系，位置对齐参考图右上角。
2. 负向提示词卡片：
   - 使用浅粉色背景内层，强化规避内容语义。
   - 复制按钮保持粉色系。
3. 文本排版：
   - `line-height` 保持 1.75 至 1.85。
   - 使用 `white-space: pre-wrap` 和 `word-break: break-word`，避免长提示词撑破布局。
4. 空状态：
   - 正负向提示词为空时保留轻量占位，不扩张卡片高度。

### Phase 4：右侧信息栏

1. 信息概览卡片：
   - 从四列网格改为右侧竖向表格结构，贴近参考图。
   - 字段顺序为：风格/模型、比例、生成引擎、标签、创建时间。
   - 当前没有生成引擎字段时显示“无（通用提示词）”。
2. 版本记录卡片：
   - 右侧栏显示最近 2 至 3 个版本。
   - 当前版本增加绿色胶囊标识。
   - “查看全部”保留开发中提示或后续接版本管理。
3. 本地安全提示卡：
   - 使用盾牌图标、本地保存文案和柯基插画。
   - 插画优先复用现有本地资源，避免新增网络依赖。

### Phase 5：背景与装饰层

1. 顶部背景：
   - 使用 CSS 渐变和轻量伪元素模拟浅蓝天空、云朵和星点。
   - 不引入大体积背景图，避免加载负担。
2. 主体容器：
   - 白色大圆角容器，内边距 32px。
   - 边框使用 `rgba(233, 226, 218, 0.7)` 或现有 `--pc-border`。
   - 阴影保持柔和，避免高对比重阴影。
3. 图标策略：
   - 优先使用已有本地 SVG 图标资源，例如 `copy.svg`、`edit.svg`、`more-horizontal.svg`、`tag.svg`。
   - 若保留字符图标，限制在装饰性元素，关键操作按钮应尽量本地图标化。

### Phase 6：响应式适配

1. 大屏：
   - `min-width >= 1280px` 使用双栏，推荐列宽 `minmax(0, 1fr) 380px`。
   - 主体最大宽度建议 1360px，贴近参考图宽屏比例。
2. 中屏：
   - `1024px` 至 `1279px` 右侧栏缩为 340px，减少左右留白。
3. 窄屏：
   - `< 1024px` 退化为单列。
   - 右侧信息栏下沉到提示词卡片之后、底部操作之前或之后，按阅读优先级决定。
4. 极窄场景：
   - 保证标题、标签、按钮不互相挤压。
   - 操作按钮允许换行或改为两列布局。

### Phase 7：文档同步

1. 更新 `docs/apps-code-map.md`：
   - 补充 `src/js/pc-detail.js` 承载 PC 详情页双栏详情画布、右侧信息栏和版本区。
   - 补充 `src/css/pc.css` 的 `pc-detail-*` 样式覆盖详情页顶部氛围、双栏布局和响应式兜底。
2. 更新就近模块说明文档：

## 风险与注意事项

- 当前页面已有可用功能，重排时必须保留 ID 与事件绑定，避免“视觉完成但交互丢失”。
- 封面图从 `contain` 改为 `cover` 会更接近参考图，但可能裁切图片边缘；可通过点击查看原图兜底。
- 右侧栏新增“生成引擎”和“创建者”属于展示增强，若当前数据缺字段，必须使用稳定默认值，不能阻断渲染。
- 顶部背景装饰应通过 CSS 实现，避免新增大图依赖导致包体增大。
- 参考图里圆角偏大，但项目规范已有 `--pc-radius-lg: 24px`，实现时应优先复用现有变量，避免局部风格失控。
