# 《提示词管家》移动端 UI 设计复刻文档

> 用途：给 AI / 设计师 / UI 生成工具复刻这一套移动端界面。  
> 目标：还原一套「本地提示词管理器」App 的完整 UI，包括首页、提示词库、提示词详情、新建编辑、分类标签、设置与本地存储 6 个界面。

---

## 1. 产品定位

**产品名称：** 提示词管家  
**产品类型：** 本地提示词管理器 / AI 生图提示词收藏与分类工具  
**核心卖点：** 本地优先、隐私安全、提示词分类管理、快速复制、标签收藏、可导入导出 JSON。  
**用户气质：** 创作者、AI 生图用户、提示词管理用户、视觉设计师、内容创作者。  
**整体情绪：** 温暖、可爱、治愈、轻量、干净、专业但不冰冷。  

---

## 2. 整体视觉关键词

- 移动端 App UI
- iOS 风格界面
- 可爱治愈风
- 本地提示词管理器
- Pastel UI / 柔和马卡龙配色
- 圆角卡片
- 轻拟物柔和阴影
- 干净留白
- Corgi 吉祥物 / 柯基小狗角色
- 简体中文 UI
- 创作者工具
- 本地数据安全
- 轻量化工具型 App

## 3. 画布与设备规范

| 项目 | 规范 |
|---|---|
| 画面比例 | 9:19 或接近 iPhone 竖屏比例 |
| 推荐尺寸 | 863 × 1822 px / 1080 × 2280 px / 1290 × 2796 px |
| UI 类型 | 单张移动端 App 界面截图 / 高保真 UI Mockup |
| 安全区 | 顶部保留 iOS 状态栏，底部保留 Home Indicator 或底部导航栏 |
| 背景 | 暖白 / 奶油白，不要纯灰，不要大面积深色 |
| 生成方式 | 每张图只生成一个完整手机界面，不要拼贴海报，不要六宫格，不要外部装饰背景 |

## 4. 全局设计风格

### 4.1 色彩系统

整体以 **奶油白 + 马卡龙粉 + 浅蓝 + 浅绿 + 浅黄 + 淡紫** 为主。

| 用途 | 推荐颜色 | 说明 |
|---|---|---|
| 主背景 | `#FFFDF8` / `#FFF8EC` | 温暖奶油白背景 |
| 主色粉 | `#FF6F9F` / `#FF7EA8` | 保存按钮、悬浮加号、主题色 |
| 主蓝色 | `#2D9CFF` / `#6EC6FF` | 首页选中态、复制按钮、链接文字 |
| 柔和绿 | `#72D879` / `#CFF7D7` | 正向提示词、标签、备份 |
| 柔和黄 | `#FFE8A3` / `#FFD15C` | 收藏星标、编辑按钮、提示卡片 |
| 柔和紫 | `#B99CFF` / `#EFE5FF` | 标签、导出 JSON、分类按钮 |
| 浅粉卡片 | `#FFF0F5` / `#FFE3EB` | 分类卡片、负向提示词 |
| 正文黑 | `#161A22` | 标题、重要文字 |
| 次级灰 | `#8B8F99` | 说明、时间、辅助信息 |
| 描边色 | `#EFE8DE` / `#EAEAEA` | 输入框、卡片边框 |

### 4.2 字体与文字层级

| 层级 | 字号建议 | 字重 | 用途 |
|---|---:|---|---|
| 页面大标题 | 30–36 px | Bold / Semibold | 首页问候、页面标题 |
| 模块标题 | 24–28 px | Bold | 最近使用、分类管理、本地存储 |
| 卡片标题 | 20–24 px | Semibold | 提示词标题、分类名称 |
| 正文 | 16–18 px | Regular | 提示词内容、说明文字 |
| 辅助文字 | 13–15 px | Regular | 时间、数量、提示说明 |
| 标签文字 | 13–15 px | Medium | 标签胶囊 |

字体风格建议：现代圆润中文无衬线字体，如 **PingFang SC、HarmonyOS Sans、MiSans、思源黑体**。

### 4.3 圆角与阴影

| 元素 | 圆角 | 阴影 |
|---|---:|---|
| 主卡片 | 20–28 px | 轻柔投影，透明度低 |
| 输入框 | 18–24 px | 微弱内外阴影 |
| 标签胶囊 | 999 px | 无明显阴影 |
| 底部悬浮按钮 | 圆形 | 明显但柔和的粉色投影 |
| 分类小图标底 | 14–18 px | 无或极弱阴影 |

阴影建议：`0 8px 24px rgba(120, 90, 50, 0.08)`，整体要轻，不要厚重。

## 5. 全局组件规范

### 5.1 状态栏

- 顶部必须保留 iOS 状态栏。
- 左侧时间：`9:41`
- 右侧：信号、Wi-Fi、电池图标。
- 状态栏颜色为黑色，背景保持透明或奶油白。

### 5.2 顶部导航栏

常见元素：

- 返回箭头：位于左上角，线性图标。
- 页面标题：居中或左对齐，根据页面类型决定。
- 搜索图标 / 更多图标 / 星标图标：位于右侧。
- 主操作按钮：如 `保存`，使用粉色圆角胶囊按钮。

### 5.3 底部导航栏

首页和列表页使用底部导航：

| 导航项 | 图标 | 状态 |
|---|---|---|
| 首页 | 房子图标 | 首页高亮蓝色 |
| 提示词库 | 文档 / 星标图标 | 提示词库高亮粉色 |
| 中央加号 | 粉色圆形按钮 | 全局新建入口 |
| 分类 | 九宫格图标 | 普通灰色 |
| 设置 | 齿轮图标 | 普通灰色 |

中央加号按钮要更大，悬浮在底部导航栏上方，带粉色渐变和柔和阴影。

### 5.4 标签胶囊

- 圆角胶囊形。
- 背景使用浅色，如浅绿、浅粉、浅紫、浅蓝、浅黄。
- 文字颜色比背景深一档。
- 标签示例：`场景`、`日系`、`樱花`、`春天`、`科幻`、`夜景`、`霓虹`、`可爱`、`治愈`。

### 5.5 卡片组件

所有卡片使用：

- 白色或浅色背景。
- 20–28 px 圆角。
- 细描边或轻投影。
- 卡片之间留白充足。
- 不要拥挤，不要大量复杂纹理。

# 10. AI 复刻执行建议

## 10.1 生成顺序

建议按以下顺序生成，保证视觉一致：

1. 先生成首页，确定全局风格、柯基形象、色彩系统。
2. 再生成提示词库，统一列表卡片和底部导航。
3. 再生成详情页，统一封面图、提示词卡片和操作按钮。
4. 再生成新建编辑页，统一输入框和表单样式。
5. 再生成分类标签页，统一管理列表和功能按钮。
6. 最后生成设置页，统一安全横幅和本地存储卡片。

## 10.2 保持一致性的关键点

- 柯基吉祥物不要每张都变成不同品种，保持橙白柯基、大眼睛、粉色腮红。
- 所有卡片都使用大圆角和轻阴影。
- 所有页面背景保持奶油白。
- 所有中文文字尽量真实、清楚、不要乱码。
- 不要将界面变成网页后台或企业管理系统。
- 不要过度科幻、不要大面积暗色、不要赛博朋克化。
- 列表页和首页保留底部导航栏，详情页、编辑页、分类页、设置页可不放底部导航，按原界面结构保持。
